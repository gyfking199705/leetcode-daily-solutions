LeetCode #139 - 单词拆分

Word Break · 动态规划 · 中等难度

📋 题目描述

给定一个非空字符串 s 和一个包含非空单词的列表 wordDict，判断字符串 s 是否可以被空格拆分为一个或多个在字典中出现的单词。

Medium 动态规划 字符串 哈希表 记忆化搜索
示例 1

输入: s = "leetcode", wordDict = ["leet", "code"]

输出: true

解释: 返回 true 因为 "leetcode" 可以被拆分成 "leet" + "code"

示例 2

输入: s = "applepenapple", wordDict = ["apple", "pen"]

输出: true

解释: 返回 true 因为 "applepenapple" 可以被拆分成 "apple" + "pen" + "apple"

示例 3

输入: s = "catsandog", wordDict = ["cats", "dog", "sand", "and", "cat"]

输出: false

💡 核心思路

这是一个经典的动态规划问题。定义 dp[i] 表示：字符串的前 i 个字符（即 s[0:i]）是否可以被拆分成字典中的单词。

状态转移方程

dp[i] = OR { dp[j] AND s[j:i] ∈ wordDict } for all j < i

即：对于每个位置 i，检查是否存在一个位置 j，使得前 j 个字符可以被拆分，且从 j 到 i 的子串在字典中。

优化技巧

  • 将 wordDict 转换为 set，实现 O(1) 查找
  • 利用单词最大长度限制搜索范围，减少不必要的检查
  • dp[0] = true 作为边界条件（空字符串可以被拆分）

🔍 解题技巧总结

  • 前缀思维：判断s能否拆分 = 检查所有前缀是否可拆，且剩余部分在字典中
  • 边界处理：dp[0] = True 是关键，表示空字符串天然可拆分
  • 哈希优化：将list转为set，查询从O(m)降到O(1)
  • 剪枝：利用单词最大长度限制内层循环范围
  • 记忆化：DFS版本需要memo字典避免重复计算
