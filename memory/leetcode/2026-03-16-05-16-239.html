滑动窗口最大值

LeetCode #239 · 单调队列经典应用

Hard 单调队列 滑动窗口 双端队列

📋 题目描述

给你一个整数数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 k 个数字。滑动窗口每次只向右移动一位。

返回 滑动窗口中的最大值。

示例 1：
输入： nums = [1,3,-1,-3,5,3,6,7], k = 3
输出： [3,3,5,5,6,7]
解释：
滑动窗口位置 最大值
--------------- -----
[1 3 -1] -3 5 3 6 7 3
1 [3 -1 -3] 5 3 6 7 3
1 3 [-1 -3 5] 3 6 7 5
1 3 -1 [-3 5 3] 6 7 5
1 3 -1 -3 [5 3 6] 7 6
1 3 -1 -3 5 [3 6 7] 7
示例 2：
输入： nums = [1], k = 1
输出： [1]
!
核心挑战： 如何在窗口滑动时，快速获取当前窗口的最大值？
暴力解法需要 O(k) 时间求最大值，总复杂度 O(n×k)。需要更优解！

💡 解题思路

维护一个单调递减的双端队列，队列中存储的是元素的索引：

  • 队列从头到尾对应的值是递减的
  • 队头元素始终是当前窗口的最大值
  • 队列中只保留当前窗口内的有效元素
1 移除无效元素：窗口右移时，移除队头不在窗口范围内的索引
2 维护单调性：新元素入队前，从队尾弹出所有比它小的元素（它们不可能成为最大值）
3 记录答案：窗口形成后，队头即为当前窗口最大值

1
元素淘汰规律：如果 a 在 b 的左边，且 a ≤ b，那么 a 永远不可能成为窗口最大值（因为 b 比 a 大，且 b 后过期）
2
队列性质：队列中的元素值严格递减，因此队头始终是当前窗口最大值
3
时间复杂度：每个元素最多入队、出队各一次，总复杂度 O(n)

📝 代码实现

Python3 单调队列解法 
from collections import deque

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        # 单调递减队列，存储索引
        q = deque()
        result = []
        
        for i, num in enumerate(nums):
            # Step 1: 移除队尾小于当前值的元素（维护单调性）
            while q and nums[q[-1]] < num:
                q.pop()
            
            # Step 2: 当前索引入队
            q.append(i)
            
            # Step 3: 移除队头不在窗口范围内的元素
            if q[0] <= i - k:
                q.popleft()
            
            # Step 4: 窗口形成后，记录最大值（队头）
            if i >= k - 1:
                result.append(nums[q[0]])
        
        return result
Python3 暴力解法（对比） 
class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        # 暴力解法：每次遍历窗口求最大值
        result = []
        for i in range(len(nums) - k + 1):
            window_max = max(nums[i:i + k])  # O(k)
            result.append(window_max)
        return result
        # 时间复杂度: O(n×k) - 会超时！

📊 复杂度分析

解法 时间复杂度 空间复杂度 说明
暴力解法 O(n × k) O(1) 每次窗口遍历求max，超时
优先队列 (堆) O(n × logk) O(k) 需要延迟删除无效元素
单调队列 ⭐ O(n) O(k) 每个元素最多入队出队一次
✓
最优解分析：
单调队列之所以能达到 O(n)，是因为每个元素最多被处理两次（入队一次，出队一次）。 这是一种典型的均摊分析（Amortized Analysis）场景。
