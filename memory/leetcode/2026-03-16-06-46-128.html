🎯 LeetCode #128 - 最长连续序列

Medium 哈希表 · 数组 · 序列优化

📋 题目描述

给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

示例 1：

输入：nums = [100, 4, 200, 1, 3, 2]

输出：4

解释：最长连续序列为 [1, 2, 3, 4]，长度为 4

示例 2：

输入：nums = [0, 3, 7, 2, 5, 8, 4, 6, 0, 1]

输出：9

解释：最长连续序列为 [0, 1, 2, 3, 4, 5, 6, 7, 8]，长度为 9

约束条件：
  • 0 ≤ nums.length ≤ 10⁵
  • -10⁹ ≤ nums[i] ≤ 10⁹

💡 解题思路

🔑 核心思想：只从序列起点开始计数

一个数 num 是某个连续序列的起点，当且仅当 num - 1 不在数组中。

步骤 1：将所有数字存入哈希集合

▼

使用哈希集合（HashSet）存储所有数字，实现 O(1) 的查询效率。这一步的目的是去重 + 快速查找。

对于示例 [100, 4, 200, 1, 3, 2]，集合为：{100, 4, 200, 1, 3, 2}

步骤 2：识别序列起点

▼

遍历集合中的每个数字 num，检查 num - 1 是否在集合中：

  • 如果 num - 1 存在 → num 不是起点，跳过
  • 如果 num - 1 不存在 → num 是序列起点，开始计数

这个技巧保证每个序列只被遍历一次。

步骤 3：从起点向后扩展计数

▼

对于每个序列起点 num，不断检查 num + 1, num + 2... 是否在集合中，直到中断，记录序列长度。

更新全局最大长度。

📝 代码实现

Python3 


1class Solution:


2    def longestConsecutive(self, nums: List[int]) -> int:


3        # 将所有数字存入哈希集合，去重 + O(1) 查询


4        num_set = set(nums)


5        longest = 0


6        


7        for num in num_set:


8            # 只从序列起点开始计数（关键优化）


9            if num - 1 not in num_set:


10                current_num = num


11                current_streak = 1


12                


13                # 向后扩展，统计序列长度


14                while current_num + 1 in num_set:


15                    current_num += 1


16                    current_streak += 1


17                


18                # 更新最大长度


19                longest = max(longest, current_streak)


20        


21        return longest
💡 代码关键点解析
  • 第 4 行：使用 set() 去重并保证 O(1) 查询
  • 第 9 行：核心判断！只有序列起点才进入内层循环，避免重复计算
  • 第 14 行：while 循环向后扩展，寻找连续的数
  • 第 19 行：实时更新全局最大长度

📊 复杂度分析

度量 值 说明
时间复杂度 O(n) 外层遍历 O(n) + 内层 while 每个元素最多被访问一次
空间复杂度 O(n) 哈希集合存储所有元素

时间复杂度证明：

看似有两层循环，但由于「只从序列起点开始计数」的优化，每个数字最多被访问两次：

  • 一次在外层循环中被检查是否为起点
  • 一次在内层 while 循环中被计入某个序列

因此总时间复杂度为 O(n)。
