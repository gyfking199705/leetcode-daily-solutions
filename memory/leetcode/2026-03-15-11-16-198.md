## 🎯 LeetCode 精讲 #198 - 打家劫舍

---

### 📋 题目描述

**来源**: LeetCode 第198题  
**难度**: Medium  
**链接**: https://leetcode.cn/problems/house-robber/

#### 题目原文
> 你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，**如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警**。
> 
> 给定一个代表每个房屋存放金额的非负整数数组，计算你 **不触动警报装置的情况下** ，今晚能够偷窃到的最高金额。

#### 示例

**示例 1:**
```
输入: [1,2,3,1]
输出: 4
解释: 偷窃 1 号房屋 (金额 = 1) ，然后偷窃 3 号房屋 (金额 = 3)。
     偷窃到的最高金额 = 1 + 3 = 4 。
```

**示例 2:**
```
输入: [2,7,9,3,1]
输出: 12
解释: 偷窃 1 号房屋 (金额 = 2), 偷窃 3 号房屋 (金额 = 9)，接着偷窃 5 号房屋 (金额 = 1)。
     偷窃到的最高金额 = 2 + 9 + 1 = 12 。
```

#### 约束条件
- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 400`

---

### 💡 解题思路

#### 核心思想：动态规划

这道题是**动态规划**的经典入门题，展示了DP最核心的思想：**状态定义** + **状态转移方程**。

#### 第一步：定义状态

设 `dp[i]` 表示：**偷窃前 i 间房屋能获得的最高金额**（第 i 间可选可不选）

#### 第二步：状态转移分析

对于第 `i` 间房屋，小偷有两个选择：

| 选择 | 条件 | 获得金额 |
|------|------|----------|
| **偷第 i 间** | 不能偷第 i-1 间 | `nums[i] + dp[i-2]` |
| **不偷第 i 间** | 可以偷第 i-1 间 | `dp[i-1]` |

**为什么偷第 i 间时要加 dp[i-2]？**
- 因为相邻房屋不能同时偷，如果偷了第 i 间，第 i-1 间就不能偷
- 所以只能从第 i-2 间及之前的最佳状态转移过来

#### 第三步：状态转移方程

```
dp[i] = max(dp[i-1], nums[i] + dp[i-2])
        ↑偷i-1不偷i    ↑偷i，加上i-2的最佳值
```

#### 第四步：初始化与边界

- `dp[0] = nums[0]`（只有一间房，只能偷它）
- `dp[1] = max(nums[0], nums[1])`（两间房，偷金额大的那间）

### 📝 代码实现

#### 方法一：标准动态规划（空间复杂度 O(n)）

```python
from typing import List

class Solution:
    def rob(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 0:
            return 0
        if n == 1:
            return nums[0]
        
        # dp[i] 表示偷到第i间房屋时的最大金额
        dp = [0] * n
        dp[0] = nums[0]
        dp[1] = max(nums[0], nums[1])
        
        for i in range(2, n):
            # 状态转移：要么不偷第i间（继承dp[i-1]），要么偷第i间（nums[i] + dp[i-2]）
            dp[i] = max(dp[i-1], nums[i] + dp[i-2])
        
        return dp[n-1]
```

#### 方法二：空间优化版（空间复杂度 O(1)）

观察状态转移方程 `dp[i] = max(dp[i-1], nums[i] + dp[i-2])`，发现每次只依赖前两个状态，可以用滚动变量优化：

```python
from typing import List

class Solution:
    def rob(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 0:
            return 0
        if n == 1:
            return nums[0]
        
        # 只用两个变量记录前两个状态
        prev2 = nums[0]  # dp[i-2]
        prev1 = max(nums[0], nums[1])  # dp[i-1]
        
        for i in range(2, n):
            # 计算当前状态
            curr = max(prev1, nums[i] + prev2)
            # 滚动更新
            prev2 = prev1
            prev1 = curr
        
        return prev1
```

#### 方法三：最简写法

```python
from typing import List

class Solution:
    def rob(self, nums: List[int]) -> int:
        prev2, prev1 = 0, 0  # dp[i-2], dp[i-1]
        
        for num in nums:
            # 当前最优 = max(不偷当前, 偷当前+前前最优)
            curr = max(prev1, num + prev2)
            prev2 = prev1
            prev1 = curr
        
        return prev1
```

### 🔍 详细示例分析

以 `nums = [2, 7, 9, 3, 1]` 为例：

```
房屋:    0    1    2    3    4
金额:    2    7    9    3    1

初始化: prev2 = 0, prev1 = 0

i=0 (金额2):
  curr = max(0, 2+0) = 2
  prev2 = 0, prev1 = 2
  → 偷第0间，获得2

i=1 (金额7):
  curr = max(2, 7+0) = 7
  prev2 = 2, prev1 = 7
  → 不偷第0间，偷第1间，获得7

i=2 (金额9):
  curr = max(7, 9+2) = 11
  prev2 = 7, prev1 = 11
  → 偷第0间和第2间，获得2+9=11

i=3 (金额3):
  curr = max(11, 3+7) = 11
  prev2 = 11, prev1 = 11
  → 保持原方案，不偷第3间

i=4 (金额1):
  curr = max(11, 1+11) = 12
  prev2 = 11, prev1 = 12
  → 偷第0、2、4间，获得2+9+1=12

最终结果: 12 ✓
```

### 🎓 关键要点总结

#### 1. DP三要素

| 要素 | 内容 |
|------|------|
| **状态定义** | `dp[i]` = 前i间房屋能偷到的最大金额 |
| **转移方程** | `dp[i] = max(dp[i-1], nums[i] + dp[i-2])` |
| **初始状态** | `dp[0]=nums[0], dp[1]=max(nums[0],nums[1])` |

#### 2. 为什么这样定义状态？

- 问题问的是"最高金额"，所以状态定义为"最大金额"
- 约束条件是"不能偷相邻"，所以转移时要考虑间隔一位

#### 3. 空间优化技巧

- 如果状态转移只依赖固定的前几个状态，可以用滚动变量代替数组
- 本题只依赖 `dp[i-1]` 和 `dp[i-2]`，所以用两个变量即可

### 🎯 面试高频考点

1. **为什么选择 max(dp[i-1], nums[i] + dp[i-2])？**
   - 这是"选或不选"的经典DP思想
   - 不选第i间：继承dp[i-1]
   - 选第i间：必须放弃i-1，所以是nums[i] + dp[i-2]

2. **空间复杂度能否进一步优化？**
   - 已经最优 O(1)，因为必须遍历一遍数组

3. **如果房屋是环形排列怎么办？**
   - 参考打家劫舍II：分两种情况（偷第一家/不偷第一家）分别计算

4. **如何输出具体的偷窃方案（哪些房屋被偷）？**
   - 需要额外记录选择路径，或者用回溯法从dp数组反推

### 💡 举一反三：状态机DP

打家劫舍本质是一个简单的**状态机**：

```
状态0: 上一间没被偷 → 当前可选偷/不偷
状态1: 上一间被偷了 → 当前只能不偷

转移:
  状态0 → 状态0 (当前不偷)
  状态0 → 状态1 (当前偷)
  状态1 → 状态0 (当前不偷)
  状态1 → 状态1 (不允许，相邻不能偷)
```

这种视角有助于理解更复杂的DP问题。
