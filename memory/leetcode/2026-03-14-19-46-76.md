# 🎯 LeetCode 精讲 #76 - 最小覆盖子串

## 📋 题目描述

**题目**: 最小覆盖子串

**难度**: 🔴 Hard

**链接**: https://leetcode.cn/problems/minimum-window-substring/

### 题目内容

给你一个字符串 `s` 、一个字符串 `t` 。返回 `s` 中涵盖 `t` 所有字符的最小子串。如果不存在涵盖 `t` 所有字符的子串，则返回空字符串 `""` 。

**注意**：
- 对于 `t` 中重复字符，我们寻找的子字符串中该字符数量必须不少于 `t` 中该字符数量。
- 如果 `s` 中存在这样的子串，我们保证它是唯一的答案。

### 示例

**示例 1：**
```
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。
```

**示例 2：**
```
输入：s = "a", t = "a"
输出："a"
解释：整个字符串 s 是最小覆盖子串。
```

**示例 3：**
```
输入：s = "a", t = "aa"
输出：""
解释：t 中两个字符 'a' 均应包含在 s 的子串中，因此没有符合条件的子串，返回空字符串。
```

### 约束条件
- `1 <= s.length, t.length <= 10^5`
- `s` 和 `t` 由英文字母组成

## 💡 解题思路

### 核心思想：滑动窗口 + 哈希表

这道题是**滑动窗口算法**的经典应用。我们需要在字符串 `s` 中找到一个窗口，使得窗口内包含 `t` 中所有字符（包括重复次数）。

### 解题步骤

#### 第一步：统计 `t` 中字符需求

使用哈希表 `need` 记录 `t` 中每个字符需要的数量。

例如：`t = "ABC"` → `need = {'A': 1, 'B': 1, 'C': 1}`

例如：`t = "AAB"` → `need = {'A': 2, 'B': 1}`

#### 第二步：滑动窗口遍历 `s`

设置两个指针 `left` 和 `right`，表示当前窗口 `[left, right]`：

1. **`right` 右移扩展窗口**：将 `s[right]` 加入窗口，更新当前窗口中字符的计数
2. **检查窗口是否满足条件**：窗口内包含 `t` 中所有字符且数量足够
3. **若满足条件，尝试收缩窗口**：`left` 右移，尽可能缩小窗口，同时更新最小窗口记录

#### 第三步：关键点——如何判断窗口满足条件

我们不能每次都用哈希表比较（太慢），而是用一个变量 `valid` 记录**满足数量要求的字符种类数**。

- 当 `valid == len(need)` 时，说明窗口满足条件

具体逻辑：
- 当 `window[c] == need[c]` 时，`valid += 1`（该字符刚好满足）
- 注意：如果 `window[c] > need[c]`，`valid` 不变（因为早就满足了）

#### 第四步：收缩窗口

当窗口满足条件时，尝试右移 `left`：
- 如果 `left` 指向的字符在窗口中数量还够（移除后仍 >= need），则可以移除
- 如果刚好等于 `need`，移除后就不满足了，停止收缩

### 图解过程

以 `s = "ADOBECODEBANC", t = "ABC"` 为例：

```
初始化: left=0, right=0, 窗口=""

Step 1: 扩展至 "ADOBEC"
        包含 A,B,C → 满足条件！
        记录长度 6

Step 2: 尝试收缩
        移除 A → "DOBEC" 仍包含 B,C，但缺 A → 停止

Step 3: 继续扩展...
        ...
        
Step N: 窗口 "BANC"
        包含 A,B,C → 满足条件！
        长度 4，更新最小值
```

## 📝 参考代码

```python
from collections import defaultdict

class Solution:
    def minWindow(self, s: str, t: str) -> str:
        # 特殊情况处理
        if not s or not t or len(s) < len(t):
            return ""
        
        # need: 记录 t 中每个字符需要的数量
        need = defaultdict(int)
        for c in t:
            need[c] += 1
        
        # window: 记录当前窗口中每个字符的实际数量
        window = defaultdict(int)
        
        # 初始化指针和变量
        left = 0          # 窗口左边界
        right = 0         # 窗口右边界
        valid = 0         # 满足数量要求的字符种类数
        
        # 记录最小窗口的起始位置和长度
        start = 0         # 最小窗口的起始索引
        min_len = float('inf')  # 最小窗口长度
        
        # 右指针向右扩展窗口
        while right < len(s):
            # c 是移入窗口的字符
            c = s[right]
            right += 1
            
            # 更新窗口数据（只关心 need 中的字符）
            if c in need:
                window[c] += 1
                # 当该字符数量刚好满足需求时，valid 增加
                if window[c] == need[c]:
                    valid += 1
            
            # 当窗口满足条件时，尝试收缩左边界
            while valid == len(need):
                # 更新最小窗口记录
                current_len = right - left
                if current_len < min_len:
                    min_len = current_len
                    start = left
                
                # d 是移出窗口的字符
                d = s[left]
                left += 1
                
                # 更新窗口数据
                if d in need:
                    # 当该字符数量刚好等于需求时，移除后就不满足了
                    if window[d] == need[d]:
                        valid -= 1
                    window[d] -= 1
        
        # 返回结果
        return "" if min_len == float('inf') else s[start:start + min_len]
```

### 代码详解

| 变量 | 含义 |
|------|------|
| `need` | t 中各字符需要的数量 |
| `window` | 当前窗口中各字符的实际数量 |
| `valid` | 满足数量要求的字符种类数 |
| `left/right` | 窗口左右指针 |
| `start/min_len` | 记录最小窗口的位置和长度 |

### 关键逻辑

1. **为什么 `window[c] == need[c]` 时才增加 `valid`**？
   - 因为我们只关心"刚好满足"的那个时刻
   - 超过需求数量时，valid 已经算过了，不需要重复计算

2. **为什么收缩时要判断 `window[d] == need[d]`**？
   - 如果当前数量刚好等于需求，移除后就不满足了
   - 如果当前数量大于需求，移除后仍满足，可以继续收缩

## ⏱️ 复杂度分析

### 时间复杂度: **O(|s| + |t|)**

- 统计 `t` 的字符：O(|t|)
- 滑动窗口遍历 `s`：`left` 和 `right` 都最多移动 |s| 次
- 每个字符最多被访问两次（进入和离开窗口各一次）
- 总体：O(|s| + |t|)

### 空间复杂度: **O(|Σ|)**

- `need` 和 `window` 哈希表的大小取决于字符集大小
- 由于题目说明只包含英文字母，实际为 O(52) = O(1)
- 一般情况：O(|Σ|)，其中 Σ 是字符集

## 🔑 关键要点

### 1. 滑动窗口模板

这道题是滑动窗口的**标准模板题**，核心框架：

```python
left, right = 0, 0
while right < len(s):
    # 扩大窗口
    window.add(s[right])
    right += 1
    
    # 满足条件时收缩窗口
    while 满足条件:
        # 更新结果
        
        # 收缩窗口
        window.remove(s[left])
        left += 1
```

### 2. 双哈希表技巧

- `need`：目标需求（不变的基准）
- `window`：当前状态（动态变化）
- 通过比较两者来判断是否满足条件

### 3. `valid` 变量的妙用

- 避免每次 O(|Σ|) 地比较两个哈希表
- 用 O(1) 的变量追踪满足条件的字符种类数

### 4. 什么时候用滑动窗口

- 找**满足某种条件**的**连续子数组/子串**
- 条件通常是关于"包含某些元素"或"元素和/积满足要求"
