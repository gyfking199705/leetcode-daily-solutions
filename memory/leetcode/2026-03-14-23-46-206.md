## 🎯 LeetCode 精讲 #206 - 反转链表

### 📋 题目描述

给你单链表的头节点 `head`，请你反转链表，并返回反转后的链表。

**示例 1：**
```
输入：head = [1,2,3,4,5]
输出：[5,4,3,2,1]
```

**示例 2：**
```
输入：head = [1,2]
输出：[2,1]
```

**示例 3：**
```
输入：head = []
输出：[]
```

**提示：**
- 链表中节点的数目范围是 `[0, 5000]`
- `-5000 <= Node.val <= 5000`

---

### 💡 解题思路

链表反转的核心是**改变每个节点的 next 指针指向**。

**方法一：迭代法（三指针）**

想象有一排人手拉手，你要让他们调转方向。你需要：
1. 用 `prev` 记录前一个节点（初始为 null，因为原头节点会变成尾节点）
2. 用 `curr` 遍历当前节点
3. 用 `next_temp` 暂存下一个节点（防止断链后丢失）

**过程演示：**
```
初始: 1 → 2 → 3 → 4 → 5 → null
      ↑
     head

Step 1: null ← 1    2 → 3 → 4 → 5 → null
                ↑
               curr

Step 2: null ← 1 ← 2    3 → 4 → 5 → null
                     ↑
                    curr

最终: null ← 1 ← 2 ← 3 ← 4 ← 5
                          ↑
                         head (新的头)
```

**方法二：递归法**

递归的核心思想：假设后面的链表已经反转好了，我只需要把当前节点接到后面去。

```
假设: 1 → 2 → 3 → 4 → 5
         ↑_________| (后面已经反转成 5→4→3→2)

现在需要: 把 2 的 next 指向 1，把 1 的 next 置空
```

递归的妙处是代码简洁，但对栈深度有要求（O(n)）。

### 📝 参考代码

**方法一：迭代法（推荐，空间 O(1)）**

```python
# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def reverseList(self, head: ListNode) -> ListNode:
        prev = None      # 前一个节点，初始为 null
        curr = head      # 当前节点，从头开始
        
        while curr:
            next_temp = curr.next   # 暂存下一个节点
            curr.next = prev        # 反转：当前节点指向前一个
            prev = curr             # prev 前移
            curr = next_temp        # curr 前移
        
        return prev   # 最后 prev 指向新的头节点
```

**方法二：递归法（理解递归思想）**

```python
class Solution:
    def reverseList(self, head: ListNode) -> ListNode:
        # 递归终止条件：空链表或只剩一个节点
        if not head or not head.next:
            return head
        
        # 递归反转后面的链表
        new_head = self.reverseList(head.next)
        
        # 现在的状态：head → (已反转的部分)
        # 需要：让 head.next 指向自己，自己指向 null
        head.next.next = head   # 下一个节点的 next 指向当前
        head.next = None        # 当前节点变成尾节点
        
        return new_head
```

**方法三：头插法（另一种迭代思路）**

```python
class Solution:
    def reverseList(self, head: ListNode) -> ListNode:
        dummy = ListNode(0)   # 虚拟头节点
        # 不断把当前节点插到 dummy 后面
        while head:
            next_node = head.next      # 保存下一个
            head.next = dummy.next     # 当前节点指向 dummy 的下一个
            dummy.next = head          # dummy 指向当前（头插）
            head = next_node           # 移动到下一个
        return dummy.next
```

---

### ⏱️ 复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 迭代法 | O(n) | O(1) | 最优，只遍历一次 |
| 递归法 | O(n) | O(n) | 栈空间消耗，有栈溢出风险 |
| 头插法 | O(n) | O(1) | 与迭代法等价，思路不同 |

---

### 🔑 关键要点

1. **保存 next 再修改** —— 链表操作的第一原则，先保存再断链
2. **三指针技巧** —— `prev`, `curr`, `next` 是链表遍历的标配
3. **虚拟头节点** —— 头插法时常用，简化边界处理
4. **递归的理解** —— 相信子问题已解决，专注当前这一步

**易错点：**
- 忘记保存 `curr.next` 就直接修改，导致链表断裂
- 递归法忘记把 `head.next` 置空，形成环
- 返回的是 `prev` 而不是 `curr`（循环结束后 `curr` 是 null）
