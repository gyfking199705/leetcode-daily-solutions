## 📝 题目描述

给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。

请你找出并返回这两个正序数组的 **中位数** 。

算法的时间复杂度应该为 `O(log (m+n))` 。

### 示例 1

```
输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并数组 = [1,2,3] ，中位数 2
```

### 示例 2

```
输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并数组 = [1,2,3,4] ，中位数 (2 + 3) / 2 = 2.5
```

### 约束条件
- `nums1.length == m`
- `nums2.length == n`
- `0 <= m <= 1000`
- `0 <= n <= 1000`
- `1 <= m + n <= 2000`
- `-10^6 <= nums1[i], nums2[i] <= 10^6`

---

## 💡 解题思路

### 核心思想

这道题要求 `O(log(m+n))` 的时间复杂度，立刻想到**二分查找**。

#### 关键观察

中位数的本质是什么？
- 对于一个有序数组，中位数将数组分成两部分，左边部分的元素都小于等于右边部分
- 左半部分的元素个数 = 右半部分的元素个数（或差1）

**核心转化**：
在合并后的数组中，中位数的位置是确定的。设总长度为 `total = m + n`：
- 如果 `total` 是奇数，中位数是第 `(total+1)/2` 个元素
- 如果 `total` 是偶数，中位数是第 `total/2` 和第 `total/2 + 1` 个元素的平均值

#### 二分查找策略

我们要在较短的数组 `nums1` 中进行二分，设：
- 在 `nums1` 的 `i` 位置切分，左半部分有 `i` 个元素
- 在 `nums2` 的 `j` 位置切分，左半部分有 `j` 个元素

需要满足：
1. `i + j = (m + n + 1) / 2` （确保左边部分的元素数量正确）
2. `nums1[i-1] <= nums2[j]` 且 `nums2[j-1] <= nums1[i]` （确保左边都小于右边）

通过二分查找找到满足条件的 `i` 即可。

## 🖥️ 代码实现

### Python 解法

```python
class Solution:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        # 确保 nums1 是较短的数组，减少二分范围
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1
        
        m, n = len(nums1), len(nums2)
        total = m + n
        half = (total + 1) // 2  # 左半部分需要的元素个数
        
        # 在 nums1 中进行二分查找
        left, right = 0, m
        
        while left <= right:
            # i: nums1 中左半部分的元素个数
            # j: nums2 中左半部分的元素个数
            i = (left + right) // 2
            j = half - i
            
            # 处理边界情况
            nums1_left = float('-inf') if i == 0 else nums1[i - 1]
            nums1_right = float('inf') if i == m else nums1[i]
            nums2_left = float('-inf') if j == 0 else nums2[j - 1]
            nums2_right = float('inf') if j == n else nums2[j]
            
            # 检查是否找到正确的分割点
            if nums1_left <= nums2_right and nums2_left <= nums1_right:
                # 找到正确分割点
                if total % 2 == 1:
                    # 奇数情况，取左边最大值
                    return max(nums1_left, nums2_left)
                else:
                    # 偶数情况，取左边最大值和右边最小值的平均
                    left_max = max(nums1_left, nums2_left)
                    right_min = min(nums1_right, nums2_right)
                    return (left_max + right_min) / 2
            
            elif nums1_left > nums2_right:
                # nums1 的左半部分太大，需要左移
                right = i - 1
            else:
                # nums1 的左半部分太小，需要右移
                left = i + 1
        
        return 0.0  # 理论上不会到达这里
```

### 逐行解析

```python
# 1. 确保 nums1 是较短的数组
if len(nums1) > len(nums2):
    nums1, nums2 = nums2, nums1
```
**为什么？** 我们在 `nums1` 上做二分，让 `nums1` 更短可以减少二分次数，同时避免一些边界问题。

```python
# 2. 计算左半部分需要的元素个数
half = (total + 1) // 2
```
**为什么 `+1`？** 这样无论总数是奇数还是偶数，左半部分的元素个数都等于 `(总数+1)//2`，即奇数时多一个。

```python
# 3. 二分查找范围
left, right = 0, m
```
`i` 表示从 `nums1` 中取多少个元素到左半部分，范围是 `[0, m]`。

```python
# 4. 处理边界
nums1_left = float('-inf') if i == 0 else nums1[i - 1]
nums1_right = float('inf') if i == m else nums1[i]
```
当 `i=0`，表示 `nums1` 的所有元素都在右半部分，左边虚拟为 `-∞`。
当 `i=m`，表示 `nums1` 的所有元素都在左半部分，右边虚拟为 `+∞`。

```python
# 5. 检查分割点是否正确
if nums1_left <= nums2_right and nums2_left <= nums1_right:
```
这是核心条件：左半部分的所有元素都要小于等于右半部分的所有元素。

## 🎯 关键要点总结

### 1. 为什么时间复杂度是 O(log(min(m,n)))？
因为我们只在较短的数组上进行二分查找，所以时间复杂度取决于较短数组的长度。

### 2. 如何处理边界情况？
- 一个数组为空：直接返回另一个数组的中位数
- 所有元素都在一个数组中：使用正负无穷作为虚拟边界

### 3. 为什么使用虚拟边界？
使用 `-inf` 和 `+inf` 可以统一处理所有边界情况，避免写大量 if-else。

### 4. 分割点的核心条件是什么？
```
max(左半部分) <= min(右半部分)
```
即：`nums1[i-1] <= nums2[j]` 且 `nums2[j-1] <= nums1[i]`

## 💭 解题心路

这道题是 LeetCode 上最经典的 Hard 题目之一，看似简单，但要在 `O(log(m+n))` 时间复杂度内解决并不容易。

**常见的错误思路**：
1. ❌ 直接合并数组后排序 → O((m+n)log(m+n))，超时
2. ❌ 使用双指针遍历找第K个 → O(m+n)，不满足对数复杂度要求
3. ❌ 二分查找时边界处理错误 → 导致数组越界或结果错误

**正确思路的关键**：
- 将找中位数转化为找第K小元素
- 利用两个数组已经有序的性质
- 通过二分确定分割点，使得左右两部分元素数量平衡
