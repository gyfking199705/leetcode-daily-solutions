# LeetCode 48 - 旋转矩阵 (Rotate Image)

## 📋 题目描述

**难度：** 中等  
**标签：** 数学、矩阵、原地操作

### 题目原文

给你一幅由 `n × n` 矩阵表示的图像，请你将图像顺时针旋转 90 度。

**你必须在原地旋转图像**，这意味着你需要直接修改输入的二维矩阵。**请不要**使用另一个矩阵来旋转图像。

### 示例

**示例 1：**

```
输入：matrix = [[1,2,3],
                [4,5,6],
                [7,8,9]]
                
输出：[[7,4,1],
      [8,5,2],
      [9,6,3]]
```

**示例 2：**

```
输入：matrix = [[5,1,9,11],
                [2,4,8,10],
                [13,3,6,7],
                [15,14,12,16]]
                
输出：[[15,13,2,5],
      [14,3,4,1],
      [12,6,8,9],
      [16,7,10,11]]
```

### 约束条件

- `n == matrix.length == matrix[i].length`
- `1 <= n <= 20`
- `-1000 <= matrix[i][j] <= 1000`

## 🎯 解题思路

### 核心观察

这道题的关键在于理解**矩阵旋转的数学规律**。

让我们观察一个 3×3 矩阵旋转前后位置的变化：

```
旋转前：            旋转后：
[1, 2, 3]          [7, 4, 1]
[4, 5, 6]    →     [8, 5, 2]
[7, 8, 9]          [9, 6, 3]

位置映射：
(0,0)=1 → (0,2)=3 的位置
(0,1)=2 → (1,2)=6 的位置  
(0,2)=3 → (2,2)=9 的位置
(1,0)=4 → (0,1)=2 的位置
...
```

### 方法一：辅助矩阵（不符合原地要求，但帮助理解）

先考虑如果不限制原地修改，新矩阵 `result[i][j]` 应该等于原矩阵的什么位置？

观察发现：
- 第0行 `[1,2,3]` 变成了第2列 `[1,2,3]` 的倒序 → 其实是正序到了第2列从上往下
- 第1行 `[4,5,6]` 变成了第1列 `[4,5,6]` 的倒序
- 第2行 `[7,8,9]` 变成了第0列 `[7,8,9]` 的倒序

**规律：`result[i][j] = matrix[n-1-j][i]`**

或者说：`result[j][n-1-i] = matrix[i][j]`

### 方法二：原地旋转（正解）

既然不能开新矩阵，我们需要找到一种原地交换的方法。

#### 关键技巧：转置 + 翻转

将矩阵旋转 90° 可以分解为两个步骤：
1. **沿主对角线转置**（左上到右下的对角线）
2. **每一行水平翻转**

让我们验证：

```
原始矩阵：         转置后：          每行翻转后：
[1, 2, 3]         [1, 4, 7]        [7, 4, 1]
[4, 5, 6]   →     [2, 5, 8]   →    [8, 5, 2]  ✓
[7, 8, 9]         [3, 6, 9]        [9, 6, 3]
```

完美！这就是顺时针旋转90度的结果！

#### 另一种理解：四个数一组旋转

每个元素会参与一个4个元素的循环：

```
对于位置 (i, j)，它旋转后会到 (j, n-1-i)
而 (j, n-1-i) 会到 (n-1-i, n-1-j)
而 (n-1-i, n-1-j) 会到 (n-1-j, i)
而 (n-1-j, i) 会回到 (i, j)
```

这四个位置形成一个小循环，可以原地交换：
```
temp = top
 top = left
 left = bottom
 bottom = right
 right = temp
```

## 💻 代码实现

### Python 解法一：转置 + 水平翻转（推荐）

```python
def rotate(matrix: list[list[int]]) -> None:
    """
    原地旋转矩阵 90 度（顺时针）
    时间复杂度：O(n²)
    空间复杂度：O(1)
    """
    n = len(matrix)
    
    # 第一步：沿主对角线转置
    # matrix[i][j] <-> matrix[j][i]
    for i in range(n):
        for j in range(i + 1, n):  # 注意 j 从 i+1 开始，避免重复交换
            matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]
    
    # 第二步：每一行水平翻转
    for i in range(n):
        left, right = 0, n - 1
        while left < right:
            matrix[i][left], matrix[i][right] = matrix[i][right], matrix[i][left]
            left += 1
            right -= 1
```

### Python 解法二：直接四数交换

```python
def rotate(matrix: list[list[int]]) -> None:
    """
    原地旋转矩阵 90 度（顺时针）- 直接交换法
    时间复杂度：O(n²)
    空间复杂度：O(1)
    """
    n = len(matrix)
    
    # 只需要处理左上角的 1/4 区域
    # 对于 n×n 矩阵，处理前 (n+1)//2 行，前 n//2 列
    for i in range((n + 1) // 2):      # 行范围
        for j in range(n // 2):         # 列范围
            # 保存左上角的值
            temp = matrix[i][j]
            
            # 左下角 → 左上角
            matrix[i][j] = matrix[n - 1 - j][i]
            
            # 右下角 → 左下角
            matrix[n - 1 - j][i] = matrix[n - 1 - i][n - 1 - j]
            
            # 右上角 → 右下角
            matrix[n - 1 - i][n - 1 - j] = matrix[j][n - 1 - i]
            
            # 左上角(临时保存) → 右上角
            matrix[j][n - 1 - i] = temp
```

## 📊 复杂度分析

### 时间复杂度：O(n²)
- 转置操作：需要遍历矩阵上三角区域，约 n²/2 次交换
- 水平翻转：需要遍历每一行，约 n²/2 次交换
- 总计：O(n²)

### 空间复杂度：O(1)
- 只使用了常数个额外变量进行交换
- 在输入矩阵上直接修改，没有使用额外矩阵

## 🔑 关键要点总结

1. **核心技巧**：顺时针旋转90° = **主对角线转置** + **水平翻转**

2. **转置时注意**：只遍历上三角（`j > i`），避免重复交换

3. **翻转时注意**：每行只需交换前半部分和后半部分

4. **边界情况**：
   - n = 1：直接返回
   - n 为奇数/偶数：算法都适用

5. **逆时针旋转**：转置 + 垂直翻转（或先垂直翻转再转置）

6. **旋转180°**：水平翻转 + 垂直翻转（或转置两次）

## 💡 面试常问

**Q1：如果要求逆时针旋转90度怎么做？**
> 先沿主对角线转置，再对每一列进行垂直翻转（或先垂直翻转再转置）。

**Q2：如果要求旋转180度怎么做？**
> 方案一：顺时针旋转两次；
> 方案二：先水平翻转再垂直翻转；
> 方案三：沿中心点对称交换。

**Q3：空间复杂度要求是 O(1)，为什么要这么做？**
> 图像处理中，大图可能占用大量内存。原地修改可以节省 n² 的额外空间，对于 4K 图像（约 1200万像素）可以节省几十MB内存。
