128. 最长连续序列

Longest Consecutive Sequence · 哈希表 · 面试极高频

📅 2026-04-28 🔢 第243期 ⏱️ 15分钟精读

📋题目描述

给定一个未排序的整数数组 nums，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

要求：设计并实现时间复杂度为 O(n) 的算法。

陷阱警示：O(n) 是关键约束！不能先排序（排序 O(n log n) 不满足要求）。必须用哈希表实现 O(1) 查找。

💡核心思路

一句话概括

把数组丢进哈希集合，对每个数判断它是否是某个连续序列的起点（即 num-1 不在集合中），然后从起点开始向右遍历计数。

为什么只从起点开始？如果不是起点（即 num-1 在集合中），那这个数一定已经被某个更小的起点统计过了，无需重复计算。这是保证 O(n) 的关键——每个数最多被访问两次。

🧮解法详解

哈希集合法 — O(n) 时间 · O(n) 空间

核心洞察：一个数是序列起点的充要条件是 num-1 不在集合中。

1️⃣ 全部入集合 → 2️⃣ 遍历每个数 → 3️⃣ 判断是否是起点 → 4️⃣ 从起点向右扩展计数
def longestConsecutive(self, nums: List[int]) -> int: num_set = set(nums) longest = 0 for num in num_set: # 只从序列起点开始统计（num-1 不在集合中） if num - 1 not in num_set: current_num = num current_streak = 1 # 向右扩展，直到数字不在集合中 while current_num + 1 in num_set: current_num += 1 current_streak += 1 longest = max(longest, current_streak) return longest
复杂度值说明
时间O(n)每个数最多入/查集合2次（作为起点一次，被其他序列扫描一次）
空间O(n)哈希集合存储所有数字
O(n) 正确性证明：外层循环遍历 n 个数，内层 while 只对起点执行。每个非起点数被外层循环遇到时跳过（O(1)）。每个起点序列中的数最多被内层 while 访问一次（因为它只属于一个序列）。所以所有 while 循环总共最多访问 n 个数。总时间 O(n)。

排序法 — O(n log n) 时间 · O(1) 额外空间

不满足题目 O(n) 要求，但作为对比思路有价值。

def longestConsecutive(self, nums: List[int]) -> int: if not nums: return 0 nums.sort() longest = current = 1 for i in range(1, len(nums)): if nums[i] == nums[i-1]: continue # 跳过重复 elif nums[i] == nums[i-1] + 1: current += 1 else: longest = max(longest, current) current = 1 return max(longest, current)
复杂度值说明
时间O(n log n)排序 dominates 总复杂度
空间O(1) / O(n)取决于排序算法实现
面试要点：如果面试官说"先不考虑复杂度"，可以先提排序法思路（展示你有条理的思考过程），然后指出 O(n log n) 不满足要求，引出哈希集合最优解。

🔍关键细节 & 易错点

⚠️ 为什么必须从起点开始？

如果不判断起点，对每个数都尝试向左/右扩展，每个数会被多次访问。例如数组 [1,2,3,4]，处理 1 时访问 1,2,3,4；处理 2 时又访问 2,3,4……总时间变成 O(n²)。

判断 num-1 是否在集合中，确保每个连续序列只被统计一次。

⚠️ 空数组与重复元素

空数组直接返回 0。重复元素不影响——放入集合后自动去重。

⚠️ Python 集合查找的 O(1) 均摊

虽然 Python set 的 in 操作在极少数情况下可能退化（哈希冲突），但在 LeetCode 的测试数据规模下可视为 O(1)。面试中可以说「哈希表均摊 O(1) 查找」。

🎯面试连环追问

1
如果有负数怎么办？
不影响。集合查找对正负数一视同仁，连续序列可以跨越负数（如 [-2,-1,0,1]）。
2
如果要求输出最长连续序列本身（不只是长度）？
在统计长度时同时记录起点和终点，最后返回序列 list(range(start, end+1))。
3
如果有多个相同长度的序列，要求都输出？
用列表收集所有长度为 max_len 的序列，而不是只更新 max。
4
如果数据是流式输入（一个一个来），如何实时维护最长连续序列？
用两个哈希表：一个存所有数字（去重），一个存每个数字所在序列的边界（并查集/区间合并思想）。每次新数字到来，查看 num-1 和 num+1 是否在集合中，若在则合并区间。
5
如果要求空间复杂度也是 O(1) 怎么办？
那只能排序（O(n log n) 时间，O(1) 额外空间）。O(n) 时间和 O(1) 空间在一般模型下无法同时满足（这题的下界证明涉及复杂理论）。
