# LeetCode 25 - K个一组翻转链表

**日期**: 2026-03-15  
**难度**: Hard  
**标签**: 链表、递归、迭代

---

## 📋 题目描述

给你链表的头节点 `head` ，每 `k` 个节点一组进行翻转，请你返回修改后的链表。

`k` 是一个正整数，它的值小于或等于链表的长度。如果节点总数不是 `k` 的整数倍，那么请将最后剩余的节点保持原有顺序。

你不能只是单纯地改变节点内部的值，而是需要实际进行节点交换。

**示例 1:**
```
输入: head = [1,2,3,4,5], k = 2
输出: [2,1,4,3,5]
```

**示例 2:**
```
输入: head = [1,2,3,4,5], k = 3
输出: [3,2,1,4,5]
```

**进阶**: 你可以设计一个只用 `O(1)` 额外内存空间的算法解决此问题吗？

---

## 💡 解题思路

### 核心挑战

1. **分组定位**: 如何确定每组的范围（找到尾节点），并处理不足 `k` 个节点的情况
2. **链表反转**: 如何高效反转每组内的节点，并连接前后部分
3. **指针维护**: 如何正确维护各组之间的连接关系

### 方法一：迭代法（推荐）

**核心思想**: 使用哑节点（dummy node）简化边界处理，通过迭代逐组翻转。

**算法步骤**:
1. 创建哑节点 `dummy`，`dummy.next = head`
2. 使用 `group_prev` 指向当前已处理完组的最后一个节点
3. 对每组，先检查是否有 `k` 个节点，找到第 `k` 个节点 `kth_node`
4. 反转当前组的 `k` 个节点
5. 调整指针连接：前一组 → 翻转后的组头 → ... → 翻转后的组尾 → 下一组
6. 移动 `group_prev` 到下一组的前驱位置，继续处理

### 方法二：递归法

**核心思想**: 先处理第一组，然后递归处理剩余链表。

**算法步骤**:
1. 从 `head` 出发，走 `k-1` 步找到当前组尾节点 `tail`
2. 若不足 `k` 个节点，直接返回 `head`
3. 保存 `tail.next`（剩余链表头）
4. 反转 `head` 到 `tail` 的部分
5. 递归处理剩余链表
6. 将反转后的组尾（原 `head`）连接到递归结果

## 📝 完整代码

### Python 实现（迭代法）

```python
# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def reverseKGroup(self, head: ListNode, k: int) -> ListNode:
        # 创建哑节点简化边界处理
        dummy = ListNode(0, head)
        group_prev = dummy  # 指向当前组的前一个节点
        
        while True:
            # 检查是否还有至少k个节点
            kth_node = group_prev
            for _ in range(k):
                kth_node = kth_node.next
                if not kth_node:
                    return dummy.next  # 不足k个，直接返回结果
            
            # 记录下一组的起始位置
            group_next = kth_node.next
            
            # 反转当前组的k个节点
            # prev: 反转后的头，curr: 当前要处理的节点
            prev = group_prev.next  # 当前组的第一个节点（反转后会成为最后一个）
            curr = prev.next        # 从第二个节点开始反转
            
            for _ in range(k - 1):
                temp = curr.next
                curr.next = prev
                prev = curr
                curr = temp
            
            # 调整组间的连接
            first_node = group_prev.next  # 原来的第一个节点（现在是最后一个）
            first_node.next = group_next  # 连接下一组
            group_prev.next = kth_node    # 连接前一组到反转后的头
            
            # 移动到下一组
            group_prev = first_node
```

### C++ 实现（迭代法）

```cpp
class Solution {
public:
    ListNode* reverseKGroup(ListNode* head, int k) {
        ListNode* dummy = new ListNode(0, head);
        ListNode* group_prev = dummy;
        
        while (true) {
            // 检查是否还有k个节点
            ListNode* kth = group_prev;
            for (int i = 0; i < k; i++) {
                kth = kth->next;
                if (!kth) return dummy->next;
            }
            
            ListNode* group_next = kth->next;
            
            // 反转当前组
            ListNode* prev = group_prev->next;
            ListNode* curr = prev->next;
            
            for (int i = 0; i < k - 1; i++) {
                ListNode* temp = curr->next;
                curr->next = prev;
                prev = curr;
                curr = temp;
            }
            
            // 调整连接
            ListNode* first = group_prev->next;
            first->next = group_next;
            group_prev->next = kth;
            
            group_prev = first;
        }
    }
};
```

### Java 实现（递归法）

```java
class Solution {
    public ListNode reverseKGroup(ListNode head, int k) {
        if (head == null || k == 1) return head;
        
        // 找到当前组的尾节点
        ListNode tail = head;
        for (int i = 0; i < k - 1; i++) {
            tail = tail.next;
            if (tail == null) return head; // 不足k个
        }
        
        // 保存剩余链表
        ListNode rest = tail.next;
        
        // 反转当前组
        ListNode newHead = reverse(head, tail);
        
        // 递归处理剩余部分并连接
        head.next = reverseKGroup(rest, k);
        
        return newHead;
    }
    
    // 反转从head到tail的链表，返回新的头（原tail）
    private ListNode reverse(ListNode head, ListNode tail) {
        ListNode prev = null;
        ListNode curr = head;
        while (curr != tail) {
            ListNode next = curr.next;
            curr.next = prev;
            prev = curr;
            curr = next;
        }
        curr.next = prev;
        return curr;
    }
}
```

## 📊 复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 迭代法 | O(n) | O(1) | ⭐ 推荐，符合进阶要求 |
| 递归法 | O(n) | O(n/k) | 递归栈空间，代码简洁 |

**详细分析**:
- **时间复杂度 O(n)**: 每个节点被访问常数次（一次用于定位，一次用于反转）
- **空间复杂度 O(1)**: 迭代法只使用常数个指针变量
- **递归法空间复杂度**: 递归深度为链表长度除以 k，即 O(n/k)

## 🎯 关键要点

1. **哑节点（Dummy Node）**: 处理头节点可能改变的情况，简化边界判断

2. **指针的四连击**:
   - `group_prev`: 当前组的前一个节点
   - `kth_node`/`tail`: 当前组的最后一个节点
   - `group_next`: 下一组的第一个节点
   - `first_node`: 当前组翻转前的第一个节点（翻转后是最后一个）

3. **反转过程**: 标准的链表反转，但只需要反转 k-1 条边

4. **边界处理**:
   - 链表为空或只有一个节点
   - k = 1（无需翻转）
   - 最后一组不足 k 个节点
