## 第1题：两数之和 (Two Sum)

📅 **日期**: 2026年03月15日  
🏷️ **难度**: Easy  
🎯 **核心考点**: 哈希表、空间换时间

## 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出**和为目标值** `target` 的那**两个**整数，并返回它们的**数组下标**。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

你可以按任意顺序返回答案。

### 示例

**示例 1：**
```
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。
```

**示例 2：**
```
输入：nums = [3,2,4], target = 6
输出：[1,2]
```

**示例 3：**
```
输入：nums = [3,3], target = 6
输出：[0,1]
```

### 约束条件
- `2 <= nums.length <= 10^4`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`
- **只会存在一个有效答案**

---

## 解题思路

### 方法一：暴力枚举（Brute Force）

#### 思路

最直观的想法：枚举数组中的每一个数 `x`，寻找数组中是否存在 `target - x`。

#### 代码实现

**Python:**
```python
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        n = len(nums)
        for i in range(n):
            for j in range(i + 1, n):
                if nums[i] + nums[j] == target:
                    return [i, j]
        return []  # 根据题意，一定有解
```

**Java:**
```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        int n = nums.length;
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                if (nums[i] + nums[j] == target) {
                    return new int[]{i, j};
                }
            }
        }
        return new int[0];
    }
}
```

**C++:**
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        int n = nums.size();
        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                if (nums[i] + nums[j] == target) {

                }
            }
        }

```

#### 复杂度分析

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| **时间复杂度** | O(n²) | 双重循环，最坏情况下需要遍历所有元素对 |
| **空间复杂度** | O(1) | 只使用了常数额外空间 |

#### 优缺点

✅ **优点**：思路简单，易于理解，无需额外空间  
❌ **缺点**：时间复杂度高，大数据量时会超时

### 方法二：哈希表优化（推荐）⭐

#### 思路

**核心洞察**：
> 当我们遍历到数字 `x` 时，我们需要找的是 `target - x`。如果能让这个"查找"操作从 O(n) 降到 O(1)，整体复杂度就能优化到 O(n)。

**哈希表的作用**：
- 键 (key)：数组中的数字
- 值 (value)：该数字对应的下标

**执行流程**：
1. 创建一个空哈希表
2. 遍历数组，对于每个元素 `num`：
   - 先查找 `target - num` 是否在哈希表中
   - 如果在，说明找到了答案，返回两个下标
   - 如果不在，将当前 `num` 和下标存入哈希表

**为什么先查后存？**
> 避免同一个元素被使用两次。如果先存再查，可能会把当前元素当成自己的配对元素。

#### 代码实现

**Python:**
```python
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        hashtable = {}  # 数字 -> 下标的映射
        
        for i, num in enumerate(nums):
            # 先查找配对数字是否已存在
            complement = target - num
            if complement in hashtable:
                return [hashtable[complement], i]
            
            # 再将当前数字存入哈希表
            hashtable[num] = i
        
        return []
```

**Java:**
```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();
        
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            
            // 先查找
            if (map.containsKey(complement)) {
                return new int[]{map.get(complement), i};
            }
            
            // 再存入
            map.put(nums[i], i);
        }
        
        return new int[0];
    }
}
```

**C++:**
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> map;  // 数字 -> 下标
        
        for (int i = 0; i < nums.size(); i++) {
            int complement = target - nums[i];
            
            // 先查找
            auto it = map.find(complement);
            if (it != map.end()) {
                return {it->second, i};
            }
            
            // 再存入
            map[nums[i]] = i;
        }
        
        return {};
    }
};
```

**Go:**
```go
func twoSum(nums []int, target int) []int {
    m := make(map[int]int)
    
    for i, num := range nums {
        complement := target - num
        if idx, ok := m[complement]; ok {
            return []int{idx, i}
        }
        m[num] = i
    }
    
    return []int{}
}
```

**JavaScript:**
```javascript
var twoSum = function(nums, target) {
    const map = new Map();
    
    for (let i = 0; i < nums.length; i++) {
        const complement = target - nums[i];
        
        if (map.has(complement)) {
            return [map.get(complement), i];
        }
        
        map.set(nums[i], i);
    }
    
    return [];
};
```

#### 执行过程模拟

以 `nums = [2, 7, 11, 15], target = 9` 为例：

| 步骤 | i | num | complement | 哈希表状态 | 操作 |
|------|---|-----|-----------|-----------|------|
| 1 | 0 | 2 | 7 | `{}` | 7不在表中，存入{2:0} |
| 2 | 1 | 7 | 2 | `{2:0}` | 2在表中！返回[0,1] |

**仅遍历2个元素就找到答案！**

#### 复杂度分析

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| **时间复杂度** | O(n) | 只需遍历一次数组，哈希表查找/插入平均O(1) |
| **空间复杂度** | O(n) | 最坏情况下需要存储n-1个元素 |

#### 优缺点

✅ **优点**：时间复杂度最优，一次遍历即可  
❌ **缺点**：需要额外空间存储哈希表

## 关键要点总结

### 1. 核心技巧：空间换时间

这道题是**"空间换时间"**的经典案例：
- 暴力法：时间 O(n²)，空间 O(1)
- 哈希表：时间 O(n)，空间 O(n)

面试时通常要求最优解法（哈希表），但暴力解法可以作为保底方案先说出来。

### 2. 哈希表的使用时机

**当你需要频繁查询"某个元素是否出现过"时，考虑哈希表！**

这类问题的共同特征：
- 两数之和：找 `target - x` 是否存在
- 字母异位词：统计字符出现次数
- 数组交集：判断元素是否在另一数组中

### 3. 代码模板

```python
def twoSum(nums, target):
    seen = {}  # 存储已遍历的元素
    for i, num in enumerate(nums):
        # 1. 先查：找配对元素
        if target - num in seen:
            return [seen[target - num], i]
        
        # 2. 再存：记录当前元素
        seen[num] = i
```

### 4. 常见陷阱

⚠️ **先查后存的顺序不能颠倒**  
如果先存再查，当 `target = 2 * num` 时，会错误地把当前元素当成配对元素。

⚠️ **返回值是下标，不是数值**  
题目要求返回下标，不是找到的两个数。

⚠️ **下标顺序不重要**  
返回 `[i, j]` 或 `[j, i]` 都算正确。

## 面试高频追问

### Q1: 如果数组是有序的，还有更好的解法吗？

**答**：可以用**双指针法**，空间复杂度降到 O(1)。

```python
def twoSumSorted(nums, target):
    left, right = 0, len(nums) - 1
    while left < right:
        s = nums[left] + nums[right]
        if s == target:
            return [left, right]
        elif s < target:
            left += 1
        else:
            right -= 1
```

### Q2: 如果要求返回所有满足条件的数对（可能有多个答案）？

**答**：需要修改哈希表逻辑，存储所有出现的下标，或使用排序+双指针去重。

### Q3: 如果数组极大，无法一次性加载到内存？

**答**：考虑外排序 + 双指针，或使用布隆过滤器等近似算法。

## 一句话总结

> **"先查后存"的哈希表技巧：遍历数组时，先查找配对元素是否已存在，再存入当前元素。时间 O(n)，空间 O(n)。**
