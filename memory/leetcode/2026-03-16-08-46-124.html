🌲 二叉树中的最大路径和

Hard

题目描述

二叉树中的 路径 被定义为一条节点序列，序列中每对相邻节点之间都存在一条边。同一个节点在一条路径序列中 至多出现一次。该路径 至少包含一个 节点，且不一定经过根节点。

路径和 是路径中各节点值的总和。

给你一个二叉树的根节点 root，返回其 最大路径和。

💡 核心难点：路径可以起止于任意节点，不一定要经过根节点，且路径只能"拐弯"一次（二叉树特性）。

示例

示例 1：
1
/ \\
2 3

输入：root = [1,2,3]

输出：6

解释：最优路径是 2 → 1 → 3，路径和为 2 + 1 + 3 = 6

示例 2：
-10
/ \\
9 20
      / \\
15 7

输入：root = [-10,9,20,null,null,15,7]

输出：42

解释：最优路径是 15 → 20 → 7，路径和为 15 + 20 + 7 = 42

解题思路

🎯 核心思想：递归 + 后序遍历

▼

这道题的关键在于理解「经过当前节点的最大路径和」与「从当前节点出发的最大路径和」的区别：

经过当前节点的路径：可以连接左子树和右子树（呈倒V型），但只能用于更新答案，不能向上返回。

从当前节点出发的路径：只能选择一个方向（左或右），可以向上返回给父节点继续使用。

🔍 算法流程图解

▼
对于每个节点，我们计算： ┌─────────────────────────────────────────────────────────────┐ │ │ │ 1. left_gain = max(0, dfs(node.left)) ← 左子树贡献 │ │ （负数贡献视为0，表示不选） │ │ │ │ 2. right_gain = max(0, dfs(node.right)) ← 右子树贡献 │ │ （负数贡献视为0，表示不选） │ │ │ │ 3. 经过当前节点的路径和 = node.val + left_gain + right_gain│ │ → 用于更新全局最大答案 │ │ │ │ 4. 返回给父节点的值 = node.val + max(left_gain, right_gain)│ │ → 只能选一边，因为要形成一条直线 │ │ │ └─────────────────────────────────────────────────────────────┘
1 后序遍历到达最左叶子节点
2 计算该节点作为"转折点"的路径和（左+中+右）
3 更新全局最大答案
4 返回"从该节点出发"的最大路径和给父节点
5 递归处理所有节点

⚠️ 关键陷阱

▼
陷阱1：负值处理
如果子树的贡献为负，应该视为0（即不选这条路径）。使用 max(0, gain) 处理。
陷阱2：返回值的限制
返回给父节点的值必须是"单条路径"（node + max(left, right)），不能同时选两边。
陷阱3：节点值全为负
需要初始化答案为负无穷（或第一个节点的值），不能只初始化为0。

 1# Definition for a binary tree node.


 2# class TreeNode:


 3#     def __init__(self, val=0, left=None, right=None):


 4#         self.val = val


 5#         self.left = left


 6#         self.right = right


 7


 8class Solution:


 9    def maxPathSum(self, root: Optional[TreeNode]) -> int:


10        self.max_sum = float('-inf')


11        


12        def dfs(node):


13            if not node:


14                return 0


15            


16            # 递归计算左右子树的最大贡献值


17            # 负数贡献视为0（表示不从该子树走）


18            left_gain = max(0, dfs(node.left))


19            right_gain = max(0, dfs(node.right))


20            


21            # 以当前节点为"转折点"的路径和


22            # 左-中-右，呈倒V型


23            price_newpath = node.val + left_gain + right_gain


24            


25            # 更新全局最大路径和


26            self.max_sum = max(self.max_sum, price_newpath)


27            


28            # 返回"从当前节点出发"的最大路径和


29            # 只能选一边，因为要形成一条直线返回给父节点


30            return node.val + max(left_gain, right_gain)


31        


32        dfs(root)


33        return self.max_sum
C++ 实现 空间复杂度: O(h) 
 1class Solution {


 2public:


 3    int maxSum = INT_MIN;


 4    


 5    int maxGain(TreeNode* node) {


 6        if (!node) return 0;


 7        


 8        // 递归计算左右子树的最大贡献值


 9        int leftGain = max(0, maxGain(node->left));


10        int rightGain = max(0, maxGain(node->right));


11        


12        // 以当前节点为转折点的路径和


13        int priceNewPath = node->val + leftGain + rightGain;


14        


15        // 更新全局最大值


16        maxSum = max(maxSum, priceNewPath);


17        


18        // 返回从当前节点出发的最大路径和


19        return node->val + max(leftGain, rightGain);


20    }


21    


22    int maxPathSum(TreeNode* root) {


23        maxGain(root);


24        return maxSum;


25    }


26};

总结

🎯 解题模板总结：

这是一类「经过当前节点的最大XX」问题的经典模板，核心在于：

  • 区分「经过当前节点」vs「从当前节点出发」
  • 前者可以两边都选，用于更新答案
  • 后者只能选一边，用于返回给父节点
  • 负贡献剪枝：max(0, 子树贡献)
