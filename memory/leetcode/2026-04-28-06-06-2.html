#2 两数相加

2026年4月28日 · 链表模拟 · Medium

链表 模拟 面试高频 微软/字节/Google
📋 题目
给你两个非空的链表，表示两个非负的整数。它们每位数字都是按照逆序的方式存储的，并且每个节点只能存储一位数字。

请你将两个数相加，并以相同形式返回一个表示和的链表。

你可以假设除了数字 0 之外，这两个数都不会以 0 开头。
示例 1：
输入：l1 = [2,4,3], l2 = [5,6,4]
输出：[7,0,8]
解释：342 + 465 = 807

示例 2：
输入：l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
输出：[8,9,9,9,0,0,0,1]
解释：9999999 + 9999 = 10009998
💡 核心洞察
关键观察
链表数字是逆序存储的，这意味着低位在链表头部。恰好与人类竖式加法从低位到高位计算的方向一致——天然适合模拟加法。
一句话解法
同时遍历两条链表，逐位相加并处理进位。若链表长度不等，缺失的位视为 0。最后若还有进位，在末尾追加新节点。

📝 解题思路
1
初始化：创建哑节点 dummy 作为结果链表的头，用 curr 指针指向当前处理位置，carry 记录进位值。
2
逐位相加：while 循环直到两条链表都为空。每次取 l1 和 l2 当前位的值（若为空则取0），求和 sum = val1 + val2 + carry。
3
处理进位：当前位结果 = sum % 10，新进位 = sum // 10。创建新节点接到结果链表末尾。
4
收尾检查：循环结束后若 carry > 0，追加一个值为 1 的新节点（最高位进位）。
5
返回结果：返回 dummy.next（跳过哑节点）。
💻 代码实现
Python3 
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def addTwoNumbers(self, l1: ListNode, l2: ListNode) -> ListNode:
        # 哑节点：简化头节点的边界处理
        dummy = ListNode(0)
        curr = dummy
        carry = 0

        while l1 or l2 or carry:
            val1 = l1.val if l1 else 0
            val2 = l2.val if l2 else 0

            total = val1 + val2 + carry
            carry = total // 10
            digit = total % 10

            # 尾插法构建结果链表
            curr.next = ListNode(digit)
            curr = curr.next

            # 移动指针（若链表已空则保持 None）
            if l1: l1 = l1.next
            if l2: l2 = l2.next

        return dummy.next

空间复杂度*

n = max(len(l1), len(l2))，即较长链表的长度。

🎯 高频追问
Q1: 如果数字是正序存储的呢？
解法一：先翻转两条链表，然后本题解法，最后翻转结果。解法二：用栈分别压入两条链表的值，出栈时相加（此时就是从低位到高位），用头插法构建结果链表。前者翻转 3 次，后者用 O(n) 栈空间。
Q2: 链表长度差很多时如何优化？
无法优化渐进复杂度。但可以提前计算长度，让较短的链表高位补零后再同步遍历，避免每次判断 l1/l2 是否为 None 的分支预测开销（常数级优化）。
Q3: 如果链表节点值范围很大怎么办？
题目限定一位数字。若节点存多位，则每次取节点值参与计算，处理进位时注意进位可能超过一位数，需要拆分后创建多个节点或修改当前节点值。
Q4: 不允许修改输入链表如何解？
上面的解法本身就不修改输入链表，只读取 l1/l2 的值并移动指针。如果要求连指针移动都不做，就只能用递归或栈先将值读出再计算。
⚠️ 常见坑点
✕
忘记最后 carry > 0 的收尾：999 + 1 = 1000，结果比输入多一位，循环结束后必须检查进位。
✕
循环条件只写 l1 or l2：当两条链表同时为空但还有进位时（如 [5] + [5]），会遗漏最后的 1。
✕
链表长度不等时越界访问：必须先判空再取 val，或用三目运算符给缺位补 0。
✕
返回 dummy 而非 dummy.next：哑节点本身不存储有效数据，结果从 dummy.next 开始。
