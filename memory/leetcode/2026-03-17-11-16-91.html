LeetCode #91 - 解码方法

Medium 动态规划 字符串

📋 题目描述

一条包含字母 A-Z 的消息通过以下映射进行了编码：

  • 'A' -> "1"
  • 'B' -> "2"
  • ...
  • 'Z' -> "26"

给定一个只包含数字的非空字符串，请计算解码方法的总数。

示例 1

输入: s = "12"
输出: 2
解释: 它可以解码为 "AB"（1 2）或者 "L"（12）。

示例 2

输入: s = "226"
输出: 3
解释: 它可以解码为 "BZ"（2 26）、"VF"（22 6）或 "BBF"（2 2 6）。

示例 3

输入: s = "0"
输出: 0
解释: 没有字符映射到以 0 开头的有效数字。

💡 解题思路

核心洞察

  • 每个数字可以单独解码（如果该数字不是'0'）
  • 每两个连续数字可以组合解码（如果组合在10-26范围内）
  • 设 dp[i] 表示前 i 个字符的解码方法数
  • 状态转移：dp[i] = (单独解码的方案) + (组合解码的方案)
维度 结果 说明
时间复杂度 O(n) 只需遍历字符串一次
空间复杂度 O(1) 或 O(n) 可优化为只用两个变量

solution.py - 空间优化版
class Solution:
    def numDecodings(self, s: str) -> int:
        # 空字符串或首字符为'0'，无法解码
        if not s or s[0] == '0':
            return 0
        
        n = len(s)
        # dp[i] 表示前i个字符的解码方法数
        # 使用滚动数组优化，只保留前两个状态
        prev2 = 1  # dp[i-2]
        prev1 = 1  # dp[i-1]，空字符串有一种解码方式
        
        for i in range(1, n):
            current = 0
            
            # 情况1：单独解码 s[i]（如果当前数字不是'0'）
            if s[i] != '0':
                current += prev1
            
            # 情况2：组合解码 s[i-1:i+1]（如果组合在10-26之间）
            two_digit = int(s[i-1:i+1])
            if 10 <= two_digit <= 26:
                current += prev2
            
            # 如果当前位置无法解码，返回0
            if current == 0:
                return 0
            
            # 滚动更新
            prev2 = prev1
            prev1 = current
        
        return prev1
