# 🎯 LeetCode 精讲 #215 - 数组中的第K个最大元素

---

### 📋 题目描述

**来源**: LeetCode 第215题  
**难度**: Medium  
**链接**: https://leetcode.cn/problems/kth-largest-element-in-an-array/

#### 题目原文
> 给定整数数组 `nums` 和整数 `k`，请返回数组中第 `k` 个最大的元素。
> 
> 请注意，你需要找的是数组排序后的第 `k` 个最大的元素，而不是第 `k` 个不同的元素。
> 
> 你必须设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

#### 示例

**示例 1:**
```
输入: nums = [3,2,1,5,6,4], k = 2
输出: 5
解释: 排序后数组为 [1,2,3,4,5,6]，第2大的元素是5
```

**示例 2:**
```
输入: nums = [3,2,3,1,2,4,5,5,6], k = 4
输出: 4
解释: 排序后数组为 [1,2,2,3,3,4,5,5,6]，第4大的元素是4
```

#### 约束条件
- `1 <= k <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

---

### 💡 解题思路

这道题有多种解法，从简单到优化逐步分析：

#### 方法一：排序（最直观）

**思路描述**: 直接排序，取倒数第k个元素

```python
class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        nums.sort()
        return nums[-k]  # 第k大 = 倒数第k个
```

**复杂度分析**:
- 时间复杂度: O(n log n) —— 基于比较的排序
- 空间复杂度: O(1) 或 O(log n)（取决于排序实现）

**缺点**: 不符合题目要求的 O(n) 时间复杂度

---

#### 方法二：最小堆（优先队列）

**思路描述**: 
维护一个大小为 k 的最小堆，堆顶就是第k大的元素

**具体步骤**:
1. 创建一个大小为 k 的最小堆
2. 遍历数组，将元素加入堆
3. 当堆的大小超过 k 时，弹出堆顶（最小的元素）
4. 最后堆顶就是第k大的元素

```python
import heapq

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        # 使用最小堆（Python的heapq默认是最小堆）
        min_heap = []
        
        for num in nums:
            heapq.heappush(min_heap, num)
            # 保持堆的大小为k
            if len(min_heap) > k:
                heapq.heappop(min_heap)  # 弹出最小的
        
        # 堆顶就是第k大的元素
        return min_heap[0]
```

**复杂度分析**:
- 时间复杂度: O(n log k) —— 每个元素入堆出堆 O(log k)
- 空间复杂度: O(k) —— 堆的大小

**优点**: 适合数据流场景，只需维护k个元素

#### 方法三：快速选择（Quickselect）⭐ 最优解

**思路描述**: 
快速选择是快速排序的变种，利用分治思想，平均时间复杂度 O(n)

**核心思想**:
1. 随机选择一个枢轴(pivot)
2. 将数组分为三部分：大于pivot、等于pivot、小于pivot
3. 根据第k大元素所在的区间，递归处理

**详细图解**:
```
数组: [3, 2, 1, 5, 6, 4], k = 2 (找第2大)

第1轮:
  随机选pivot = 4
  分区: [5, 6] | [4] | [3, 2, 1]
        大于    等于   小于
  
  第2大在"大于"区间（因为大于pivot的有2个，正好是k=2）
  在 [5, 6] 中找第2大

第2轮:
  选pivot = 5
  分区: [6] | [5] | []
  
  第2大就是pivot = 5 ✓
```

**完整代码实现**:

```python
import random
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        """
        快速选择算法 - 平均O(n)时间复杂度
        """
        return self.quickSelect(nums, 0, len(nums) - 1, k)
    
    def quickSelect(self, nums: List[int], left: int, right: int, k: int) -> int:
        """
        在nums[left:right+1]中找第k大的元素
        """
        if left == right:
            return nums[left]
        
        # 随机选择pivot，避免最坏情况
        pivot_index = random.randint(left, right)
        
        # 将pivot放到最右边
        nums[pivot_index], nums[right] = nums[right], nums[pivot_index]
        
        # 分区: 将大于pivot的放左边，小于的放右边
        store_index = left
        for i in range(left, right):
            if nums[i] > nums[right]:  # 注意：找第k大，所以大的放左边
                nums[store_index], nums[i] = nums[i], nums[store_index]
                store_index += 1
        
        # 将pivot放到正确位置
        nums[store_index], nums[right] = nums[right], nums[store_index]
        
        # 计算pivot是第几大
        # store_index - left + 1 表示在[left, right]区间内pivot的排名
        rank = store_index - left + 1
        
        if rank == k:
            # pivot正好是第k大
            return nums[store_index]
        elif rank > k:
            # 第k大在左边（比pivot大的那些数中）
            return self.quickSelect(nums, left, store_index - 1, k)
        else:
            # 第k大在右边，找第(k - rank)大
            return self.quickSelect(nums, store_index + 1, right, k - rank)
```

**复杂度分析**:
- 时间复杂度: **平均 O(n)**，最坏 O(n²)
  - 每次分区后只需处理一半的数据
  - 期望比较次数: n + n/2 + n/4 + ... = 2n = O(n)
- 空间复杂度: O(log n) —— 递归栈深度

#### 方法四：计数排序（针对数据范围有限的情况）

**思路描述**: 
利用题目约束 `nums[i]` 范围有限 (-10^4 ~ 10^4)，使用计数排序

```python
class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        # 偏移量，处理负数
        OFFSET = 10**4
        # 计数数组: -10^4 ~ 10^4 映射到 0 ~ 20000
        count = [0] * (2 * OFFSET + 1)
        
        # 计数
        for num in nums:
            count[num + OFFSET] += 1
        
        # 从大到小遍历，找第k大
        remain = k
        for i in range(len(count) - 1, -1, -1):
            remain -= count[i]
            if remain <= 0:
                return i - OFFSET
        
        return -1  # 不会执行到这里
```

**复杂度分析**:
- 时间复杂度: O(n + R)，R为数据范围 (这里 R = 20001)
- 空间复杂度: O(R)

**适用场景**: 数据范围远小于n时非常高效

### 🎯 面试高频考点

**Q1: 为什么快速选择是O(n)？**

假设每次都能将数组分成两半：
```
T(n) = n + T(n/2)     (分区O(n) + 递归处理一半)
     = n + n/2 + n/4 + n/8 + ...
     = 2n = O(n)
```

**Q2: 最坏情况为什么是O(n²)？**

如果每次pivot都是最小或最大元素：
```
T(n) = n + T(n-1) = n + (n-1) + (n-2) + ... = O(n²)
```

**解决办法**: 随机选择pivot，或在工程中使用"中位数的中位数"选择pivot

**Q3: 堆解法和快速选择怎么选？**

| 场景 | 推荐 |
|------|------|
| 静态数组，只查询一次 | 快速选择 O(n) |
| 数据流，多次查询 | 最小堆 O(n log k) |
| k很小（如k=1,2,3） | 直接线性扫描 O(nk) |
| k接近n | 找第(n-k+1)小，同样思路 |

**Q4: 如果数组中有重复元素？**

所有解法都适用，题目要求"第k大"而非"第k个不同的数"
