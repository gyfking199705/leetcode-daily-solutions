🎯 LeetCode #160 - 相交链表

📋 题目描述

给你两个单链表的头节点 headA 和 headB，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 null。

两个链表在节点 c1 开始相交：

链表A: a1 → a2 → c1 → c2 → c3 ↗ 链表B: b1 → b2 相交节点 = c1

注意：相交不是值相等，而是节点对象相同（即内存地址相同）。

进阶要求：能否设计一个时间复杂度 O(m+n)、空间复杂度 O(1) 的解决方案？

💡 核心思想：双指针法（浪漫相遇）

两个指针分别从两个链表头出发，走到末尾时跳到对方链表头部继续走。如果存在交点，它们必定在第一次相遇。

pA 路线: A → B → C → D → E → F → C → D → E → F → ... pB 路线: X → Y → C → D → E → F → X → Y → C → D → E → F → ... 假设: A链表长度 = m, B链表长度 = n, 公共长度 = k pA 走的总步数 = m + (n-k) = m+n-k pB 走的总步数 = n + (m-k) = m+n-k 两者在交点相遇！✨

📊 复杂度分析

维度值说明
时间复杂度O(m + n)最坏情况下遍历两个链表各两次
空间复杂度O(1)只使用两个指针，常数级空间
比较次数≤ 2(m+n)每个指针最多走 m+n 步

🔍 易错点 & 注意事项

  • 相交≠值相等：必须比较节点对象（地址），而不是节点的值
  • 无交点的情况：两个指针最终都会变为 null，返回 null
  • 链表长度不等：双指针法自动处理长度差异
  • 边界情况：其中一个链表为空时直接返回 null
