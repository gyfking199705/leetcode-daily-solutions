## 📖 题目描述

你这个学期必须选修 `numCourses` 门课程，记为 `0` 到 `numCourses-1`。

在选修某些课程之前需要一些先修课程。先修课程按数组 `prerequisites` 给出，其中 `prerequisites[i] = [ai, bi]`，表示如果要学习课程 `ai` 则**必须**先学习课程 `bi`。

请你判断是否可能完成所有课程的学习？如果可以，返回 `true`；否则，返回 `false`。

### 示例

**示例 1：**
```
输入：numCourses = 2, prerequisites = [[1,0]]
输出：true
解释：总共有 2 门课程。学习课程 1 之前，你需要完成课程 0。这是可能的。
```

**示例 2：**
```
输入：numCourses = 2, prerequisites = [[1,0],[0,1]]
输出：false
解释：课程 1 依赖课程 0，而课程 0 又依赖课程 1，形成环，无法完成。
```

---

## 🎯 核心思路

**本质问题：有向图是否存在环？**

- 每门课程 = 图的一个节点
- 先修关系 `[ai, bi]` = 一条从 `bi` 指向 `ai` 的有向边（学 `bi` 才能学 `ai`）
- 能修完所有课 ↔ 图是**有向无环图（DAG）**
- 无解 ↔ 图中存在**环**（课程互相依赖，永远无法完成）

> 💡 **关键结论**：有向图存在拓扑排序 ⇔ 图是 DAG（有向无环图）

## 📝 解法一：BFS 拓扑排序（Kahn 算法）

### 算法原理

1. **建图**：构建邻接表 + 统计每个节点的入度
2. **入队**：将所有入度为 0 的节点（无前置依赖的课程）入队
3. **消解**：依次出队，将邻居节点入度减 1，若减到 0 则入队
4. **判断**：统计出队节点数，若等于课程总数则无环，返回 true

### 代码实现（Python）

```python
from collections import deque, defaultdict
from typing import List

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        # 构建邻接表和入度数组
        graph = defaultdict(list)
        in_degree = [0] * numCourses
        
        for ai, bi in prerequisites:
            graph[bi].append(ai)  # bi -> ai 的边
            in_degree[ai] += 1    # ai 的入度加 1
        
        # 将所有入度为 0 的节点入队
        queue = deque()
        for i in range(numCourses):
            if in_degree[i] == 0:
                queue.append(i)
        
        # 拓扑排序
        visited = 0  # 统计已处理的课程数
        while queue:
            cur = queue.popleft()
            visited += 1
            # 将当前课程的所有后续课程入度减 1
            for next_course in graph[cur]:
                in_degree[next_course] -= 1
                if in_degree[next_course] == 0:
                    queue.append(next_course)
        
        return visited == numCourses
```

## 📝 解法二：DFS 染色法（状态标记）

### 算法原理

使用三种状态标记节点：
- `0` = 未搜索（白色）
- `1` = 搜索中（灰色）—— 在递归栈中
- `2` = 已完成（黑色）—— 已回溯

DFS 过程中：
- 遇到状态 `1` 的节点 → 发现环 → 返回 false
- 所有节点都完成搜索且未发现环 → 返回 true

### 代码实现（Python）

```python
from collections import defaultdict
from typing import List

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        # 构建邻接表
        graph = defaultdict(list)
        for ai, bi in prerequisites:
            graph[bi].append(ai)
        
        # 0: 未搜索, 1: 搜索中, 2: 已完成
        state = [0] * numCourses
        
        def dfs(course: int) -> bool:
            if state[course] == 1:  # 遇到搜索中的节点，有环
                return False
            if state[course] == 2:  # 已完成的节点，无需重复搜索
                return True
            
            state[course] = 1  # 标记为搜索中
            for next_course in graph[course]:
                if not dfs(next_course):
                    return False
            state[course] = 2  # 标记为已完成
            return True
        
        # 对每个未搜索的节点进行 DFS
        for i in range(numCourses):
            if state[i] == 0:
                if not dfs(i):
                    return False
        
        return True
```

## 📊 复杂度分析

| 解法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| BFS (Kahn) | O(V + E) | O(V + E) | V=课程数, E=先修关系数 |
| DFS 染色 | O(V + E) | O(V + E) | 递归栈深度可能较大 |

- **时间复杂度**：每个节点和边都只被处理一次
- **空间复杂度**：邻接表 + 状态数组/入度数组

## ⚖️ 两种解法对比

| 对比维度 | BFS (Kahn) | DFS (染色法) |
|---------|-----------|-------------|
| 核心逻辑 | 消解入度，逐步删除 | 跟踪递归栈，检测环 |
| 直观性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 代码量 | 稍多 | 较少 |
| 适用场景 | 需要输出拓扑序、课程数大 | 仅需检测环、代码简洁优先 |
| 注意事项 | 无 | 递归深度过大可能栈溢出 |

**推荐**：面试时优先写 BFS，逻辑更直观，不易出错。

## 🎓 关键要点总结

1. **建图方向**：`[ai, bi]` 表示 `bi → ai` 的边（学 `bi` 才能学 `ai`），不要搞反

2. **环的本质**：课程依赖形成闭环，永远无法满足（如 A→B→C→A）

3. **拓扑排序判定**：
   - 能拓扑排序 = 无环 = 可以完成所有课程
   - 不能拓扑排序 = 有环 = 无法完成

4. **实战技巧**：
   - BFS 解法中，用队列维护入度为 0 的节点
   - DFS 解法中，注意三种状态的转换逻辑
