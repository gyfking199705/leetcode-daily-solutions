LeetCode #337 - 打家劫舍 III

在上次打劫完一条街道和一圈房屋后，小偷发现了一个新的目标——二叉树结构的房屋群 🌳

Medium 树形DP 后序遍历 递归

📋 题目描述

小偷又发现了一个新的可行窃的地区。这个地区只有一个入口，我们称之为 root。

除了 root 之外，每栋房子有且只有一个「父」房子与之相连。一番侦察之后，聪明的小偷意识到「这个地方的所有房屋的排列类似于一棵二叉树」。如果两个直接相连的房子在同一天晚上被打劫，房屋将自动报警。

给定二叉树的 root。返回在不触动警报的情况下，小偷能够盗取的最高金额。

示例 1: root = [3,2,3,null,3,null,1]

3
/  \
2 3
  \     \
3 1

输出: 7 (偷 3 + 3 + 1)

💡 核心思路

🎯 关键洞察：每个节点只有两种状态

  • 状态0 (不偷当前节点)：子节点可以偷，也可以不偷，取最大值
  • 状态1 (偷当前节点)：子节点绝对不能偷

🌲 为什么用后序遍历？

  • 计算当前节点需要知道左右子树的结果
  • 自底向上：先从叶子节点开始计算，再向上传递
  • 符合「后序遍历」的顺序：左 → 右 → 根

核心技巧总结

  • 状态设计：为每个节点设计「偷/不偷」两种状态，返回二元组
  • 后序遍历：子树结果 → 当前节点结果，自底向上传递
  • 树形DP：在树上做DP时，通常需要遍历整棵树，每个节点计算后返回给父节点
  • 记忆化：如果直接递归会重复计算，可用 @lru_cache 优化（虽然本题不需要，因为每个节点只访问一次）
