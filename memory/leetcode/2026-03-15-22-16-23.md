## 📝 题目描述

给你一个链表数组，每个链表都已经按升序排列。请你将所有链表合并到一个升序链表中，返回合并后的链表。

**示例 1：**
```
输入：lists = [[1,4,5],[1,3,4],[2,6]]
输出：[1,1,2,3,4,4,5,6]
解释：链表数组如下：
[
  1->4->5,
  1->3->4,
  2->6
]
将它们合并到一个有序链表中得到：
1->1->2->3->4->4->5->6
```

**示例 2：**
```
输入：lists = []
输出：[]
```

**示例 3：**
```
输入：lists = [[]]
输出：[]
```

## 💡 解题思路

### 方法一：优先队列（最小堆）⭐ 推荐

**核心思想**：维护一个最小堆，堆中存储每个链表的当前头节点。每次取出最小的节点接到结果链表，然后将该节点的下一个节点（如果存在）加入堆。

**为什么用堆？**
- K个链表，每次要找最小值，暴力扫描需要 O(K)
- 使用最小堆可以将找最小值优化到 O(log K)

**算法步骤**：
1. 初始化最小堆，将所有非空链表的头节点加入堆
2. 创建哑节点（dummy）作为结果链表的头
3. 循环直到堆为空：
   - 取出堆顶最小节点
   - 接到结果链表末尾
   - 如果该节点有下一个节点，将 next 加入堆
4. 返回 dummy.next

### 方法二：分治法（归并思想）

**核心思想**：将K个链表两两配对合并，递归这个过程直到只剩一个链表。类似于归并排序的 merge 过程。

**算法步骤**：
1. 递归基例：如果链表数组为空返回 null；如果只剩一个链表直接返回
2. 将数组分为左右两半，分别递归合并
3. 合并两个有序链表（双指针法）
4. 返回合并结果

### 方法对比

| 解法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|-----------|-----------|------|------|
| **优先队列** | O(N log K) | O(K) | 逻辑清晰，适合流式数据 | 需要堆结构 |
| **分治法** | O(N log K) | O(log K) | 无需额外数据结构 | 递归栈开销 |
| 顺序合并 | O(K²N) | O(1) | 实现简单 | 效率太低 |

> N 为所有节点总数，K 为链表个数

## 🔧 代码实现

### Python - 优先队列法

```python
import heapq
from typing import List, Optional

# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
    
    # 用于堆比较
    def __lt__(self, other):
        return self.val < other.val

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        # 最小堆
        min_heap = []
        
        # 将所有非空链表的头节点加入堆
        for i, node in enumerate(lists):
            if node:
                heapq.heappush(min_heap, node)
        
        # 哑节点简化链表操作
        dummy = ListNode(0)
        curr = dummy
        
        # 循环处理堆
        while min_heap:
            # 取出最小节点
            node = heapq.heappop(min_heap)
            curr.next = node
            curr = curr.next
            
            # 将该节点的下一个节点加入堆
            if node.next:
                heapq.heappush(min_heap, node.next)
        
        return dummy.next
```

### Python - 分治法

```python
class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        if not lists:
            return None
        return self.merge(lists, 0, len(lists) - 1)
    
    def merge(self, lists: List[Optional[ListNode]], left: int, right: int) -> Optional[ListNode]:
        # 递归基例
        if left == right:
            return lists[left]
        
        # 分治
        mid = left + (right - left) // 2
        l1 = self.merge(lists, left, mid)
        l2 = self.merge(lists, mid + 1, right)
        
        return self.mergeTwoLists(l1, l2)
    
    def mergeTwoLists(self, l1: Optional[ListNode], l2: Optional[ListNode]) -> Optional[ListNode]:
        # 合并两个有序链表
        dummy = ListNode(0)
        curr = dummy
        
        while l1 and l2:
            if l1.val < l2.val:
                curr.next = l1
                l1 = l1.next
            else:
                curr.next = l2
                l2 = l2.next
            curr = curr.next
        
        # 拼接剩余节点
        curr.next = l1 if l1 else l2
        return dummy.next
```

### Java - 优先队列法

```java
class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        // 最小堆，按节点值排序
        PriorityQueue<ListNode> minHeap = new PriorityQueue<>((a, b) -> a.val - b.val);
        
        // 将所有非空链表头节点加入堆
        for (ListNode list : lists) {
            if (list != null) {
                minHeap.offer(list);
            }
        }
        
        // 哑节点
        ListNode dummy = new ListNode(-1);
        ListNode curr = dummy;
        
        // 循环处理
        while (!minHeap.isEmpty()) {
            ListNode node = minHeap.poll();
            curr.next = node;
            curr = curr.next;
            
            // 将下一个节点加入堆
            if (node.next != null) {
                minHeap.offer(node.next);
            }
        }
        
        return dummy.next;
    }
}
```

## 📊 复杂度分析

### 优先队列法
- **时间复杂度**：O(N log K)
  - 每个节点入堆、出堆各一次
  - 堆操作时间复杂度为 O(log K)
  - 总节点数为 N
- **空间复杂度**：O(K)
  - 堆中最多存储 K 个节点

### 分治法
- **时间复杂度**：O(N log K)
  - 每层合并所有节点需要 O(N)
  - 递归深度为 O(log K)
- **空间复杂度**：O(log K)
  - 递归调用栈的深度
  - 合并两个链表只需要 O(1) 额外空间

## 🎯 关键要点总结

### 1. 哑节点（Dummy Node）技巧
- 使用 dummy 节点可以简化链表操作，避免处理头节点为空的边界情况
- 最后返回 dummy.next 即可

### 2. 堆中元素比较（Python）
- Python 的 heapq 需要可比较的对象
- 为 ListNode 实现 `__lt__` 方法，或者存储元组 `(val, node)`

### 3. 边界情况处理
- 输入数组为空
- 数组中包含空链表
- 所有链表都为空

### 4. 选择哪种方法？
- **优先队列**：代码直观，适合面试时快速写出
- **分治法**：空间更优，展示对递归和分治的掌握
