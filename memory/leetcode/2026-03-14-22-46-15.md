## 🎯 LeetCode 精讲 #15 - 三数之和

### 📋 题目描述

**给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。**

**请你返回所有和为 0 且不重复的三元组。**

> **注意：** 答案中不可以包含重复的三元组。

---

**示例 1：**
```
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0
不同的三元组是 [-1,0,1] 和 [-1,-1,2]
注意，输出的顺序和三元组的顺序并不重要
```

**示例 2：**
```
输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0
```

**示例 3：**
```
输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0
```

---

**约束条件：**
- `3 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

---

### 💡 解题思路

#### 核心思想：排序 + 双指针

三数之和是「两数之和」的进阶版，但难度提升了不少，关键在于**去重**。

#### 为什么不能直接用哈希表？

两数之和可以用哈希表 `O(n)` 解决，但三数之和如果套用，需要 `O(n²)` 的枚举 + 哈希查找，而且**去重非常复杂**。

#### 排序的优势：

1. **去重变得简单**：排序后相同数字相邻，容易跳过重复
2. **双指针优化**：固定一个数，剩余两个数用双指针从两端向中间查找

#### 算法步骤：

```
1. 对数组进行排序
2. 固定第一个数 nums[i]，问题转化为「在 i 后面的区间找两数之和等于 -nums[i]」
3. 使用双指针 left 和 right 在剩余区间查找
4. 跳过所有重复元素，避免重复答案
```

#### 详细执行流程：

以 `nums = [-1, 0, 1, 2, -1, -4]` 为例：

**第 1 步：排序**
```
排序后：[-4, -1, -1, 0, 1, 2]
```

**第 2 步：固定第一个数，双指针查找**

| i | 固定数 | 目标值 | 双指针区间 | 找到的三元组 |
|---|-------|--------|-----------|-------------|
| 0 | -4 | 4 | [-1,-1,0,1,2] | 无 |
| 1 | -1 | 1 | [-1,0,1,2] | [-1,-1,2], [-1,0,1] |
| 2 | -1 | 1 | [0,1,2] | 跳过（重复） |
| 3 | 0 | 0 | [1,2] | 无 |
| 4 | 1 | -1 | [2] | 不足两个数，结束 |

#### 关键：如何去重？

**第一层去重（固定数）：**
```python
if i > 0 and nums[i] == nums[i-1]:
    continue  # 跳过重复的固定数
```

**第二层去重（左指针）：**
```python
while left < right and nums[left] == nums[left+1]:
    left += 1  # 跳过重复的左指针值
```

**第三层去重（右指针）：**
```python
while left < right and nums[right] == nums[right-1]:
    right -= 1  # 跳过重复的右指针值
```

#### 双指针移动规则：

```
当前三数之和 sum = nums[i] + nums[left] + nums[right]

- sum == 0: 找到一个答案，双指针同时收缩
- sum < 0:  说明 left 太小，left++（需要更大的数）
- sum > 0:  说明 right 太大，right--（需要更小的数）
```

### 📝 参考代码

```python
from typing import List

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        """
        三数之和 - 排序 + 双指针解法
        
        时间复杂度: O(n²) - 外层循环 O(n)，双指针 O(n)
        空间复杂度: O(1) 或 O(log n)（取决于排序算法）
        """
        n = len(nums)
        if n < 3:
            return []
        
        # 排序，方便去重和使用双指针
        nums.sort()
        result = []
        
        for i in range(n - 2):  # 只需要遍历到倒数第三个数
            # 剪枝 1: 如果当前数已经大于0，后面都大于0，不可能和为0
            if nums[i] > 0:
                break
            
            # 剪枝 2: 跳过重复的固定数（去重关键）
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            
            # 双指针初始化
            left, right = i + 1, n - 1
            target = -nums[i]  # 需要找的两数之和
            
            while left < right:
                current_sum = nums[left] + nums[right]
                
                if current_sum == target:
                    # 找到一组答案
                    result.append([nums[i], nums[left], nums[right]])
                    
                    # 去重：跳过左边重复的元素
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    
                    # 去重：跳过右边重复的元素
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    
                    # 双指针同时收缩，寻找下一组
                    left += 1
                    right -= 1
                
                elif current_sum < target:
                    # 和太小，需要增大，左指针右移
                    left += 1
                
                else:
                    # 和太大，需要减小，右指针左移
                    right -= 1
        
        return result


# ==================== 测试代码 ====================

def test_solution():
    """测试用例"""
    sol = Solution()
    
    # 示例 1
    nums1 = [-1, 0, 1, 2, -1, -4]
    result1 = sol.threeSum(nums1)
    print(f"示例 1: {result1}")
    assert sorted([sorted(triplet) for triplet in result1]) == [[-1, -1, 2], [-1, 0, 1]]
    
    # 示例 2
    nums2 = [0, 1, 1]
    result2 = sol.threeSum(nums2)
    print(f"示例 2: {result2}")
    assert result2 == []
    
    # 示例 3
    nums3 = [0, 0, 0]
    result3 = sol.threeSum(nums3)
    print(f"示例 3: {result3}")
    assert result3 == [[0, 0, 0]]
    
    # 额外测试：多个重复
    nums4 = [-2, 0, 0, 2, 2]
    result4 = sol.threeSum(nums4)
    print(f"额外测试: {result4}")
    assert result4 == [[-2, 0, 2]]
    
    print("✅ 所有测试通过!")


if __name__ == "__main__":
    test_solution()
```

#### 代码逐行解析：

```python
# 剪枝 1: nums[i] > 0 时直接结束
# 原因：数组已排序，如果最小的数都 > 0，后面不可能和为 0
if nums[i] > 0:
    break  # 不是 continue，是 break！

# 剪枝 2: 跳过重复固定数
# 如果 nums[i] 和 nums[i-1] 相同，
# 那么以 nums[i] 为固定数的所有组合都已经找过了
if i > 0 and nums[i] == nums[i - 1]:
    continue

# 双指针查找
left, right = i + 1, n - 1
target = -nums[i]  # 剩余两数需要满足的和

# 三数之和转化为：nums[left] + nums[right] == target
```

### ⏱️ 复杂度分析

| 指标 | 复杂度 | 说明 |
|-----|-------|-----|
| **时间复杂度** | O(n²) | 外层遍历 O(n)，内层双指针 O(n)，总共 O(n²) |
| **空间复杂度** | O(1) 或 O(log n) | 仅使用常数额外空间（排序可能使用 O(log n)） |

| 解法 | 时间复杂度 | 空间复杂度 | 难点 |

| 哈希表 | O(n²) | O(n) | 去重极复杂 |
| **排序+双指针** | **O(n²)** | **O(1)** | ✅ 最优解 |

### 🔑 关键要点

#### 1. 为什么排序后不会漏解？

排序只是改变了数字的顺序，没有改变数字本身。我们遍历的是索引组合 `(i, left, right)`，不是具体的值，所以不会漏掉任何可能的组合。

#### 2. 三处去重的逻辑：

```
┌─────────────────────────────────────────────────────────┐
│  去重位置        判断条件                    操作        │
├─────────────────────────────────────────────────────────┤
│  固定数 i       nums[i] == nums[i-1]       continue    │
│  左指针         nums[left] == nums[left+1] left++      │
│  右指针         nums[right] == nums[right-1] right--   │
└─────────────────────────────────────────────────────────┘
```

#### 3. 常见错误：

```python
# ❌ 错误：在和之前去重
if nums[left] == nums[left-1]:  # 这会漏掉有效解！
    left += 1

# ✅ 正确：在找到一个解之后去重
while left < right and nums[left] == nums[left + 1]:
    left += 1
```

#### 4. 为什么用 `break` 而不是 `continue`？

```python
if nums[i] > 0:
    break      # ✅ 后面所有数都 > 0，直接结束
    # continue # ❌ 效率低，虽然结果正确
```

### 🎯 总结

**三数之和**是双指针技巧的经典应用，核心在于：

1. **排序**是前提，让去重和双指针成为可能
2. **固定一个数**，将三数转化为两数问题
3. **三处去重**（固定数、左指针、右指针）避免重复答案
4. **双指针移动**根据当前和的大小决定收缩方向

这道题考察的是**在复杂约束条件下正确使用双指针的能力**，是面试中非常高频的题目。
