🎯 LeetCode #300

最长递增子序列

Longest Increasing Subsequence

Medium 动态规划 贪心 + 二分查找

📋 题目描述

给你一个整数数组 nums，找到其中最长严格递增子序列的长度。


子序列 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，[3,6,2,7] 是数组 [0,3,1,6,2,2,7] 的子序列。

示例 1：
输入：nums = [10,9,2,5,3,7,101,18]
输出：4
解释：最长递增子序列是 [2,3,7,101]，因此长度为 4。
示例 2：
输入：nums = [0,1,0,3,2,3]
输出：4
示例 3：
输入：nums = [7,7,7,7,7,7,7]
输出：1

💡 核心考点

本题是面试中的必考经典，考查对动态规划的深入理解，以及能否从 O(n²) 优化到 O(n log n) 的算法思维。两个解法都要掌握！

💡 解题思路

🔹 方法一：动态规划 (O(n²)) ▼

核心思想

定义 dp[i] 为：以第 i 个元素结尾的最长递增子序列的长度。

状态转移

对于每个位置 i，遍历所有之前的位置 j (0 ≤ j < i)：

  • 如果 nums[j] < nums[i]，则 nums[i] 可以接在 nums[j] 后面
  • dp[i] = max(dp[i], dp[j] + 1)

⚠️ 复杂度提醒

时间复杂度 O(n²)，当 n = 10⁵ 时会超时。需要更优解法！

🔹 方法二：贪心 + 二分查找 (O(n log n)) ⭐推荐 ▼

核心思想（非常巧妙！）

维护一个数组 tails，其中 tails[i] 表示：长度为 i+1 的递增子序列的最小结尾元素。

🔑 关键洞察

为了让后续能形成更长的序列，我们希望结尾元素越小越好！这就是贪心思想。

算法流程

  1. 遍历每个数 num
  2. 在 tails 中找到第一个 ≥ num 的位置 pos（二分查找）
  3. 如果 num 比所有元素都大，追加到末尾
  4. 否则，用 num 替换 tails[pos]（贪心：保持更小的结尾）

✨ 为什么 tails 是有序的？

用反证法：如果 tails[i] > tails[j] 且 i < j，那长度为 i+1 的序列可以接在长度为 j 的序列后面形成更长的序列，矛盾！所以 tails 一定递增。

📝 代码实现

Python - 动态规划解法 O(n²) 
1class Solution:
2    def lengthOfLIS(self, nums: List[int]) -> int:
3        if not nums:
4            return 0
5        
6        n = len(nums)
7        # dp[i] 表示以 nums[i] 结尾的最长递增子序列长度
8        dp = [1] * n
9        
10        for i in range(1, n):
11            for j in range(i):
12                if nums[j] < nums[i]:  # 可以接在后面
13                    dp[i] = max(dp[i], dp[j] + 1)
14        
15        return max(dp)  # 返回所有位置的最大值
Python - 贪心 + 二分查找 O(n log n) ⭐ 
1class Solution:
2    def lengthOfLIS(self, nums: List[int]) -> int:
3        if not nums:
4            return 0
5        
6        # tails[i] 表示长度为 i+1 的递增子序列的最小结尾元素
7        tails = []
8        
9        for num in nums:
10            # 二分查找：找到第一个 >= num 的位置
11            left, right = 0, len(tails)
12            while left < right:
13                mid = (left + right) // 2
14                if tails[mid] < num:
15                    left = mid + 1
16                else:
17                    right = mid
18            
19            if left == len(tails):
20                # num 比所有元素都大，可以延长序列
21                tails.append(num)
22            else:
23                # 贪心：用更小的 num 替换，保持后续扩展能力
24                tails[left] = num
25        
26        return len(tails)
Python - 使用 bisect 模块 (最简洁) 
1import bisect
2
3class Solution:
4    def lengthOfLIS(self, nums: List[int]) -> int:
5        tails = []
6        for num in nums:
7            idx = bisect.bisect_left(tails, num)
8            if idx == len(tails):
9                tails.append(num)
10            else:
11                tails[idx] = num
12        return len(tails)

📊 复杂度分析

解法 时间复杂度 空间复杂度 适用场景
动态规划 O(n²) O(n) n ≤ 10⁴，或需要输出具体序列
贪心 + 二分 O(n log n) O(n) n ≤ 10⁵，面试标准答案 ⭐

⚠️ 注意

贪心+二分方法虽然能求出 LIS 的长度，但 tails 数组不是真正的 LIS！例如 [1, 4, 5, 2, 3] 中，处理完后的 tails 是 [1, 2, 3]，但实际的 LIS 是 [1, 4, 5] 或 [1, 2, 3]。

💡 输出具体序列的方法

维护一个 prev[i] 数组记录每个元素的前驱，同时用 tails 记录每个长度对应的元素下标。当更新 tails 时，同步更新前驱关系，最后通过回溯得到完整序列。
