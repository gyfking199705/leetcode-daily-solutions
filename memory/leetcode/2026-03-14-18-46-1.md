## 🎯 LeetCode 精讲 #1 - 两数之和

---

### 📋 题目描述

**来源**: LeetCode 第1题  
**难度**: Easy  
**链接**: https://leetcode.cn/problems/two-sum/

#### 题目原文
> 给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值** *target* 的那 **两个** 整数，并返回它们的数组下标。
> 
> 你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。
> 
> 你可以按任意顺序返回答案。

#### 示例

**示例 1:**
```
输入: nums = [2, 7, 11, 15], target = 9
输出: [0, 1]
解释: 因为 nums[0] + nums[1] == 9，返回 [0, 1]。
```

**示例 2:**
```
输入: nums = [3, 2, 4], target = 6
输出: [1, 2]
```

**示例 3:**
```
输入: nums = [3, 3], target = 6
输出: [0, 1]
```

#### 约束条件
- `2 <= nums.length <= 10^4`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`
- 只会存在一个有效答案

---

### 💡 解题思路

#### 方法一：暴力枚举（Brute Force）

**思路描述:**
枚举数组中的每一个数 `x`，寻找数组中是否存在 `target - x`。

**详细分析:**
1. 使用两层循环，外层循环遍历每个元素 `nums[i]`
2. 内层循环遍历 `i` 之后的元素 `nums[j]`
3. 检查是否满足 `nums[i] + nums[j] == target`
4. 如果找到，返回 `[i, j]`

**时间复杂度**: O(n²) — 两层嵌套循环  
**空间复杂度**: O(1) — 只使用了常数额外空间

#### 方法二：哈希表（Hash Table）⭐ 推荐

**思路描述:**
利用哈希表快速查找的特性，将"寻找 target - x"的时间复杂度从 O(n) 降到 O(1)。

**详细分析:**
1. 创建一个哈希表 `num_to_index`，用于存储「数值 → 下标」的映射
2. 遍历数组，对于当前元素 `nums[i]`：
   - 计算 `complement = target - nums[i]`（即我们需要找的另一个数）
   - 在哈希表中查找 `complement` 是否存在
   - 如果存在，说明找到了答案，返回 `[哈希表中存的补数下标, i]`
   - 如果不存在，将 `nums[i]` 和其下标 `i` 存入哈希表
3. 继续遍历直到找到答案

**为什么这样是可行的？**
- 哈希表的查找操作平均时间复杂度是 O(1)
- 每遍历到一个新元素，我们先查「需要的数」是否存在，再把自己存进去
- 这样确保不会用到同一个元素两次（先查再存）

**时间复杂度**: O(n) — 只遍历一次数组，每次哈希操作 O(1)  
**空间复杂度**: O(n) — 哈希表最多存储 n 个元素

### 📝 参考代码

#### Python 实现（哈希表解法）

```python
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        """
        两数之和 - 哈希表解法
        
        思路：利用哈希表存储已遍历的元素，将查找时间从O(n)降到O(1)
        
        参数:
            nums: 整数数组
            target: 目标和
            
        返回:
            两个数的下标列表
        """
        # 哈希表：数值 -> 下标
        num_to_index = {}
        
        # 遍历数组
        for i, num in enumerate(nums):
            # 计算需要的补数
            complement = target - num
            
            # 检查补数是否已在哈希表中
            if complement in num_to_index:
                # 找到了！返回两个下标
                return [num_to_index[complement], i]
            
            # 将当前数存入哈希表，供后续元素查找
            num_to_index[num] = i
        
        # 根据题意，一定有解，这里不会执行到
        return []
```

#### 暴力枚举解法（对比）

```python
class SolutionBruteForce:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        """
        暴力枚举解法 - 时间O(n²)，空间O(1)
        """
        n = len(nums)
        for i in range(n):
            for j in range(i + 1, n):
                if nums[i] + nums[j] == target:
                    return [i, j]
        return []
```

#### 其他语言参考

**Java:**
```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            if (map.containsKey(complement)) {
                return new int[] { map.get(complement), i };
            }
            map.put(nums[i], i);
        }
        return new int[0];
    }
}
```

**C++:**
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> mp;
        for (int i = 0; i < nums.size(); i++) {
            int complement = target - nums[i];
            if (mp.count(complement)) {
                return {mp[complement], i};
            }
            mp[nums[i]] = i;
        }
        return {};
    }
};
```

### ⏱️ 复杂度分析

| 解法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| 暴力枚举 | O(n²) | O(1) | n 很小（<1000），空间受限 |
| 哈希表 | **O(n)** | **O(n)** | **通用推荐**，时间和空间平衡 |

**详细说明:**
- **时间 O(n)**: 只需遍历数组一次，哈希表的插入和查找都是 O(1) 平均时间
- **空间 O(n)**: 最坏情况下需要将数组中 n-1 个元素存入哈希表

---

### 🔑 关键要点

1. **哈希表是核心**: 这道题是「空间换时间」的经典案例，哈希表将查找复杂度从 O(n) 降到 O(1)

2. **先查后存**: 顺序很重要！
   - 先检查 `complement` 是否在表中（避免用到当前元素）
   - 再将当前元素存入表中（供后续元素使用）

3. **边界情况处理**:
   - ✅ 数组中有重复数字（如 `[3, 3]`）
   - ✅ 负数（如 `[-1, -2, -3]`）
   - ✅ 大数值（在 int 范围内均可）

4. **为什么不用排序+双指针？**
   - 排序后原数组下标会丢失
   - 虽然能做到 O(nlogn)，但需要额外存储原下标
   - 哈希表 O(n) 更优
