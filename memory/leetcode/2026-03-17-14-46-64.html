LeetCode #64 - 最小路径和

Minimum Path Sum | 动态规划经典入门题 | 难度：Medium

📋 题目描述

给定一个包含非负整数的 m x n 网格 grid，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。

移动规则：每次只能向下或者向右移动一步。

示例：
输入: grid = [[1,3,1],[1,5,1],[4,2,1]]
输出: 7
解释: 路径 1→3→1→1→1 的总和为 7（注意只能向下或向右移动）

🎯 解题思路

这是一个经典的动态规划问题。核心思想是：

  • 设 dp[i][j] 表示从起点到达位置 (i,j) 的最小路径和
  • 由于只能向右或向下移动，所以到达 (i,j) 只能从 (i-1,j) 或 (i,j-1) 来
  • 状态转移方程：dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]

📊 复杂度分析

解法 时间复杂度 空间复杂度 说明
基础DP（二维） O(m × n) O(m × n) 需要额外的dp数组存储所有状态
滚动数组优化 O(m × n) O(n) 只保留当前行，空间减半
原地修改（最优） O(m × n) O(1) 直接修改grid数组，不额外使用空间

💡 关键技巧与易错点

  • 边界处理：第一行和第一列只能单向移动，需要单独初始化
  • 空间优化：如果允许修改输入数组，可以直接在grid上原地计算，达到O(1)空间
  • 与「最大路径和」的区别：最大路径和会有负数导致贪心失效，但最小路径和全是非负数，DP是唯一正解
  • 路径还原：如果需要输出具体路径，需要额外记录每个位置是从哪个方向来的
