LeetCode #155 - 最小栈

Easy 设计一个支持 push, pop, top 操作，并能在常数时间内检索到最小元素的栈。

📋 题目描述

设计一个支持 push，pop，top 操作，并能在常数时间内检索到最小元素的栈。

需要实现的方法：
• MinStack() — 初始化堆栈对象
• push(val) — 将元素 val 推入堆栈
• pop() — 删除堆栈顶部的元素
• top() — 获取堆栈顶部的元素
• getMin() — 获取堆栈中的最小元素

约束条件：

  • -2³¹ ≤ val ≤ 2³¹ - 1
  • pop、top 和 getMin 操作总是在非空栈上调用
  • 最多调用 3 × 10⁴ 次 push, pop, top, getMin

示例：

MinStack minStack = new MinStack();
minStack.push(-2);
minStack.push(0);
minStack.push(-3);
minStack.getMin();   // 返回 -3
minStack.pop();
minStack.top();      // 返回 0
minStack.getMin();   // 返回 -2

💡 核心思路

问题的关键在于：如何在弹出元素后，仍然知道剩余元素中的最小值？

主栈
存储所有元素
→
辅助栈
同步存储最小值
→
O(1) 查询
getMin() 常数时间

辅助栈的维护规则：

  1. push 时：辅助栈压入当前元素和辅助栈顶较小值（或当前元素）
  2. pop 时：辅助栈同步弹出，栈顶始终对应当前最小值
空间优化技巧：辅助栈只在遇到更小或相等的元素时才入栈，相等时也要入栈（处理重复最小值的情况）

📝 解题技巧总结

  1. 双栈技巧：一个主栈存储数据，一个辅助栈维护状态（最小值、最大值等）
  2. 同步操作：辅助栈和主栈的 push/pop 必须保持同步
  3. 边界情况：处理重复元素时，辅助栈也要压入（保证 pop 后最小值正确）
  4. 空间优化：辅助栈只在必要时入栈（遇到更小元素时才压入）
