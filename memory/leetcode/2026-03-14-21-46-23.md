## 🎯 LeetCode 精讲 #23 - 合并K个升序链表

---

### 📋 题目描述

**来源**: LeetCode 第23题  
**难度**: Hard  
**链接**: https://leetcode.cn/problems/merge-k-sorted-lists/

#### 题目原文
> 给你一个链表数组，每个链表都已经按升序排列。
> 
> 请你将所有链表合并到一个升序链表中，返回合并后的链表。

#### 示例

**示例 1:**
```
输入: lists = [[1,4,5],[1,3,4],[2,6]]
输出: [1,1,2,3,4,4,5,6]
解释: 链表数组如下：
[
  1->4->5,
  1->3->4,
  2->6
]
将它们合并到一个有序链表中得到：
1->1->2->3->4->4->5->6
```

**示例 2:**
```
输入: lists = []
输出: []
```

**示例 3:**
```
输入: lists = [[]]
输出: []
```

#### 约束条件
- `k == lists.length`
- `0 <= k <= 10^4`
- `0 <= lists[i].length <= 500`
- `-10^4 <= lists[i][j] <= 10^4`
- `lists[i]` 按升序排列
- `lists[i].length` 的总和不超过 `10^4`

### 💡 解题思路

#### 方法一：顺序合并（Brute Force）

**思路描述**
- 用一个变量 `ans` 存储已合并的结果链表
- 依次遍历链表数组，每次将 `ans` 与当前链表合并
- 合并两个有序链表使用双指针法

**代码实现**
```python
# 伪代码
def mergeKLists(lists):
    ans = None
    for lst in lists:
        ans = mergeTwoLists(ans, lst)  # 合并两个链表
    return ans

def mergeTwoLists(l1, l2):
    # 双指针合并两个有序链表
    dummy = ListNode(0)
    curr = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            curr.next = l1
            l1 = l1.next
        else:
            curr.next = l2
            l2 = l2.next
        curr = curr.next
    curr.next = l1 if l1 else l2
    return dummy.next
```

**复杂度分析**
- 假设每个链表平均长度为 n，共有 k 个链表
- 第1次合并：0 + n = n，第2次：n + n = 2n，第3次：2n + n = 3n...
- 总操作次数：n + 2n + 3n + ... + kn = n(1+2+...+k) = O(nk²)
- **时间复杂度**: O(nk²)
- **空间复杂度**: O(1)，只使用常数额外空间（不考虑递归栈）

---

#### 方法二：分治合并 ⭐ 推荐

**思路描述**
- 采用分治的思想，将 k 个链表两两配对合并
- 第一轮后：k/2 个链表，每个长度 2n
- 第二轮后：k/4 个链表，每个长度 4n
- ...直到只剩一个链表

**图解**
```
初始: [list1] [list2] [list3] [list4] [list5] [list6] [list7] [list8]
      
第1轮:  [  合并后的1-2  ] [  合并后的3-4  ] [  合并后的5-6  ] [  合并后的7-8  ]

第2轮:        [      合并后的1-4       ] [      合并后的5-8       ]

第3轮:                  [           最终结果            ]

总共需要 log(k) 轮合并
```

**复杂度分析**
- 每轮合并操作总数：所有节点数 = nk
- 总共有 log(k) 轮
- **时间复杂度**: O(nk × log(k))
- **空间复杂度**: O(log(k))，递归栈深度

---

#### 方法三：优先队列/最小堆 ⭐⭐ 最优解法

**思路描述**
- 维护一个大小为 k 的最小堆，堆中存放每个链表的当前头节点
- 每次从堆中弹出最小值节点，接到结果链表
- 如果该节点还有后继，将后继节点加入堆中

**详细步骤**
```
初始状态:
    lists = [[1,4,5], [1,3,4], [2,6]]
    堆: [1(来自list1), 1(来自list2), 2(来自list3)]

第1步: 弹出1(来自list1), 结果: [1], 将4加入堆
       堆: [1(来自list2), 2(来自list3), 4(来自list1)]

第2步: 弹出1(来自list2), 结果: [1,1], 将3加入堆
       堆: [2(来自list3), 3(来自list2), 4(来自list1)]

第3步: 弹出2(来自list3), 结果: [1,1,2], 将6加入堆
       堆: [3(来自list2), 4(来自list1), 6(来自list3)]

...以此类推
```

**为什么用堆？**
- 每次要找 k 个链表头中的最小值
- 堆可以在 O(log k) 时间内完成插入和弹出最小值
- 总共有 nk 个节点，每个节点都要入堆出堆一次

### 📝 参考代码

#### Python 实现（优先队列法）

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

import heapq

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        """
        使用优先队列（最小堆）合并K个有序链表
        
        思路：
        1. 将k个链表的头节点放入最小堆
        2. 每次取出堆顶元素（当前最小值），接到结果链表
        3. 如果取出的节点还有next，将next加入堆
        4. 重复直到堆为空
        """
        
        # 自定义包装类，用于比较ListNode
        # Python3中需要包装，因为ListNode不能直接比较
        class Wrapper:
            def __init__(self, node):
                self.node = node
            def __lt__(self, other):
                # 定义比较规则：按节点值比较
                return self.node.val < other.node.val
        
        # 初始化最小堆
        min_heap = []
        
        # 将所有非空链表的头节点加入堆
        for l in lists:
            if l:
                heapq.heappush(min_heap, Wrapper(l))
        
        # 虚拟头节点，简化操作
        dummy = ListNode(0)
        curr = dummy
        
        # 循环直到堆为空
        while min_heap:
            # 弹出最小值节点
            wrapper = heapq.heappop(min_heap)
            node = wrapper.node
            
            # 接到结果链表
            curr.next = node
            curr = curr.next
            
            # 如果该节点还有后继，加入堆
            if node.next:
                heapq.heappush(min_heap, Wrapper(node.next))
        
        return dummy.next
```

#### Python 实现（分治法）

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        """
        分治法合并K个有序链表
        时间复杂度：O(nk log k)，空间复杂度：O(log k)
        """
        if not lists:
            return None
        
        return self.mergeRange(lists, 0, len(lists) - 1)
    
    def mergeRange(self, lists, left, right):
        """递归合并区间 [left, right] 内的链表"""
        if left == right:
            return lists[left]
        
        mid = (left + right) // 2
        # 分：合并左半部分
        left_merged = self.mergeRange(lists, left, mid)
        # 分：合并右半部分
        right_merged = self.mergeRange(lists, mid + 1, right)
        # 治：合并两个有序链表
        return self.mergeTwoLists(left_merged, right_merged)
    
    def mergeTwoLists(self, l1, l2):
        """合并两个有序链表（标准双指针法）"""
        dummy = ListNode(0)
        curr = dummy
        
        while l1 and l2:
            if l1.val <= l2.val:
                curr.next = l1
                l1 = l1.next
            else:
                curr.next = l2
                l2 = l2.next
            curr = curr.next
        
        # 接上剩余部分
        curr.next = l1 if l1 else l2
        
        return dummy.next
```

#### Python 实现（顺序合并 - 简单但非最优）

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        """
        顺序合并：每次合并两个链表
        时间复杂度：O(nk²)，空间复杂度：O(1)
        """
        ans = None
        for lst in lists:
            ans = self.mergeTwoLists(ans, lst)
        return ans
    
    def mergeTwoLists(self, l1, l2):
        """合并两个有序链表"""
        if not l1:
            return l2
        if not l2:
            return l1
        
        dummy = ListNode(0)
        curr = dummy
        
        while l1 and l2:
            if l1.val <= l2.val:
                curr.next = l1
                l1 = l1.next
            else:
                curr.next = l2
                l2 = l2.next
            curr = curr.next
        
        curr.next = l1 if l1 else l2
        return dummy.next
```

### ⏱️ 复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|----------|
| 顺序合并 | O(nk²) | O(1) | k很小，代码最简单 |
| 分治合并 | O(nk log k) | O(log k) | k较大，递归栈可接受 |
| 优先队列 | O(nk log k) | O(k) | k较大，需要稳定性能 |

**说明：**
- n = 每个链表的平均长度
- k = 链表数量
- 优先队列法和分治法的时间复杂度相同，但优先队列的常数因子更大一些
- 分治法的空间复杂度来自递归栈，优先队列来自堆存储

---

### 🔑 关键要点

1. **合并两个有序链表是基础**
   - 双指针法是标准解法，必须熟练掌握
   - 使用虚拟头节点（dummy）可以简化边界处理

2. **分治的核心思想**
   - 将大问题分解为相似的子问题
   - 两两配对减少问题规模
   - 时间复杂度从 O(nk²) 优化到 O(nk log k)

3. **优先队列的妙用**
   - 当需要"每次取最小/最大"时考虑堆
   - Python中使用 `heapq` 模块
   - 注意：Python3中需要包装类来定义比较规则

4. **边界条件处理**
   - 输入为空列表 `[]`
   - 输入包含空链表 `[[]]`
   - 链表节点值相等的情况
