## 🎯 LeetCode 精讲 #11 - 盛最多水的容器

---

### 📋 题目描述

**来源**: LeetCode 第11题  
**难度**: Medium  
**链接**: https://leetcode.cn/problems/container-with-most-water/

#### 题目原文
> 给定一个长度为 `n` 的整数数组 `height`。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])`。
> 
> 找出其中的两条线，使得它们与 `x` 轴共同构成的容器可以容纳最多的水。
> 
> 返回容器可以储存的最大水量。
> 
> **说明**：你不能倾斜容器。

#### 示例

**示例 1:**
```
输入: height = [1,8,6,2,5,4,8,3,7]
输出: 49
解释: 图中垂直线代表输入数组 [1,8,6,2,5,4,8,3,7]。在此情况下，容器能够容纳水（表示为蓝色部分）的最大值为 49。
```

**示例 2:**
```
输入: height = [1,1]
输出: 1
```

#### 约束条件
- `n == height.length`
- `2 <= n <= 10^5`
- `0 <= height[i] <= 10^4`

```
      ↑                           ↑
      └───────────────────────────┘
              width = 8 - 1 = 7
              height = min(8, 7) = 7
              area = 7 * 7 = 49 ✓
```

**容器的面积计算:**
```
面积 = 底边宽度 × 有效高度
     = (right - left) × min(height[left], height[right])
```

---

#### 二、算法选择：双指针 (Two Pointers)

**核心洞察：**

对于位置 `i` 和 `j`（假设 `i < j`）：
- 底边宽度固定为 `j - i`
- 有效高度取决于**较矮的那根线**：`min(height[i], height[j])`

**关键观察：**

如果我们有两个指针分别指向数组的两端，要移动哪一个？

假设 `height[left] < height[right]`：
- 当前面积受限于左边的较短高度
- 如果我们移动右指针 `right--`：
  - 宽度变小了
  - 高度最多还是 `height[left]`（甚至更矮）
  - **面积必然变小或不变** ❌
  
- 如果我们移动左指针 `left++`：
  - 宽度变小了
  - 但高度**可能变大**！
  - **有可能得到更大的面积** ✓

**结论：每次移动较矮的那个指针！**

---

#### 三、算法步骤详解

```
初始化: left = 0, right = n - 1, maxArea = 0

while left < right:
    1. 计算当前面积:
       width = right - left
       h = min(height[left], height[right])
       area = width * h
       maxArea = max(maxArea, area)
    
    2. 移动指针:
       if height[left] < height[right]:
           left++   // 左边更矮，移动左边
       else:
           right--  // 右边更矮或相等，移动右边

返回 maxArea
```

---

#### 四、正确性证明

**为什么移动较高的指针不可能得到更优解？**

假设当前状态 `left`, `right`，且 `height[left] < height[right]`。

当前面积：`area = (right - left) × height[left]`

考虑所有包含 `right` 但不包含 `left` 的组合：
- 这些组合的右端点都是 `right`
- 左端点在 `(left, right)` 范围内
- 底边宽度一定小于 `right - left`
- 高度最多是 `height[left]`（甚至可能更矮）

**因此，任何包含 `right` 但排除 `left` 的组合，面积一定 ≤ 当前面积。**

这意味着我们可以**安全地排除 `right` 与 `left` 右侧任何位置的组合**，直接移动 `left`。

### 📝 完整代码实现

#### Python 解法

```python
class Solution:
    def maxArea(self, height: list[int]) -> int:
        """
        双指针法求解最大水容器问题
        
        思路：
        - 使用两个指针从两端向中间移动
        - 每次计算当前面积并更新最大值
        - 移动较矮的指针，因为较高的指针保留有希望遇到更高的线
        
        时间复杂度: O(n) - 只需遍历一次
        空间复杂度: O(1) - 只使用常数额外空间
        """
        left, right = 0, len(height) - 1
        max_area = 0
        
        while left < right:
            # 计算当前容器的面积
            width = right - left
            h = min(height[left], height[right])
            current_area = width * h
            
            # 更新最大面积
            max_area = max(max_area, current_area)
            
            # 移动较矮的指针
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        
        return max_area
```

#### Java 解法

```java
class Solution {
    public int maxArea(int[] height) {
        int left = 0;
        int right = height.length - 1;
        int maxArea = 0;
        
        while (left < right) {
            // 计算当前面积
            int width = right - left;
            int h = Math.min(height[left], height[right]);
            int currentArea = width * h;
            
            // 更新最大面积
            maxArea = Math.max(maxArea, currentArea);
            
            // 移动较矮的指针
            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }
        
        return maxArea;
    }
}
```

### ⏱️ 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| **时间复杂度** | O(n) | 双指针从两端向中间移动，每个元素最多访问一次 |
| **空间复杂度** | O(1) | 只使用常数额外空间（几个变量） |

**对比其他方法：**
- 暴力枚举：O(n²) 时间，O(1) 空间 ❌ 超时
- 动态规划：不适合此问题（无最优子结构）
- 双指针：O(n) 时间，O(1) 空间 ✓ 最优

### 🔍 关键点总结

#### 核心要点
1. **面积公式**: `(right - left) × min(height[left], height[right])`
2. **指针移动规则**: 始终移动**较矮**的那一侧
3. **贪心正确性**: 移动较高的指针不可能得到更优解

#### 易错点
- ❌ 移动较高的指针（会错过潜在最优解）
- ❌ 忘记取 `min` 作为高度（容器高度受限于短板）
- ❌ 数组越界（注意边界条件 `left < right`）

#### 面试高频追问
1. **为什么移动较矮的指针是正确的？**
   - 因为当前面积受限于较矮的高度
   - 移动较高的指针，宽度减小，高度不可能增加（仍受限于较矮的边）
   - 只有移动较矮的指针，才有可能遇到更高的线，从而增加面积

2. **如果数组中有0怎么办？**
   - 0作为高度时，与任何其他元素构成的面积都是0
   - 算法会自动处理，不影响结果

3. **能否用单调栈？**
   - 可以，但复杂度过高
   - 单调栈通常用于"下一个更大/更小元素"问题
   - 此问题双指针是最优解

### 🎯 面试高频考点

1. **时间复杂度为什么是 O(n) 而不是 O(n²)？**
   - 虽然有两层循环的表象，但两个指针是**相向移动**
   - 每次循环只移动一个指针，总共移动 n-1 次
   - 每个元素最多被访问一次

2. **能否优化常数因子？**
   - 基本无法优化，已经是 O(n) 的线性扫描
   - 可以添加提前终止条件（如当前最大可能面积）

3. **变体问题**：如果要求返回具体的两个下标呢？
   - 在更新 maxArea 时同时记录 left 和 right
   - 最后返回这两个下标即可

### 💻 变体实现：返回具体位置

```python
class Solution:
    def maxAreaWithIndices(self, height: list[int]) -> tuple[int, int, int]:
        """
        返回: (最大面积, 左下标, 右下标)
        """
        left, right = 0, len(height) - 1
        max_area = 0
        best_left = best_right = 0
        
        while left < right:
            width = right - left
            h = min(height[left], height[right])
            current_area = width * h
            
            if current_area > max_area:
                max_area = current_area
                best_left, best_right = left, right
            
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        
        return max_area, best_left, best_right


# 测试
sol = Solution()
height = [1, 8, 6, 2, 5, 4, 8, 3, 7]
area, i, j = sol.maxAreaWithIndices(height)
print(f"最大面积: {area}")
print(f"位置: [{i}] = {height[i]}, [{j}] = {height[j]}")
print(f"验证: ({j} - {i}) * min({height[i]}, {height[j]}) = {area}")
```

### 🧠 算法思想提炼

**双指针法**的核心思想：
1. **缩小搜索空间**：通过指针移动排除不可能产生更优解的组合
2. **贪心决策**：每次基于当前信息做出局部最优选择
3. **相向移动**：两个指针从两端向中间靠拢，保证每个元素被考虑

这种思想广泛应用于：
- 有序数组的查找（两数之和II）
- 回文判断
- 数组去重
- 滑动窗口问题
