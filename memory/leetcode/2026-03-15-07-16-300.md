# LeetCode 300 - 最长递增子序列

**难度:** 中等  
**标签:** 数组、动态规划、二分查找  
**通过率:** 约 55%

## 📋 题目描述

给定一个整数数组 `nums`，找到其中**最长严格递增子序列的长度**。

**子序列**的定义：通过删除原数组中的某些元素（也可以不删除）而得到的新数组，且不改变剩下元素的相对顺序。

### 示例

```
输入: nums = [10, 9, 2, 5, 3, 7, 101, 18]
输出: 4
解释: 最长递增子序列是 [2, 3, 7, 101]，长度为 4

输入: nums = [0, 1, 0, 3, 2, 3]
输出: 4
解释: 最长递增子序列可以是 [0, 1, 2, 3] 或 [0, 1, 3] 等

输入: nums = [7, 7, 7, 7, 7, 7, 7]
输出: 1
解释: 严格递增，所以只有一个元素
```

---

## 🧠 解题思路

### 方法一：动态规划（O(n²)）

**核心思想：** 对于每个位置 `i`，计算以 `nums[i]` 结尾的最长递增子序列长度。

**状态定义：**
- `dp[i]` = 以 `nums[i]` 结尾的最长递增子序列的长度

**状态转移：**
```
对于每个 i，遍历所有 j < i：
如果 nums[j] < nums[i]：
    dp[i] = max(dp[i], dp[j] + 1)
```

**初始状态：**
- `dp[i] = 1`（每个元素本身构成长度为1的子序列）

**答案：**
- `max(dp)`，即所有位置结尾的最长子序列长度的最大值

**为什么这样设计？**
- 我们需要"以某个位置结尾"，这样才能通过前面的结果递推
- 如果只用"最长长度"而不记录结尾，无法判断能否接在当前元素后面

### 方法二：贪心 + 二分查找（O(n log n)）

**核心思想：** 维护一个数组 `tails`，其中 `tails[i]` 表示长度为 `i+1` 的递增子序列的最小结尾元素。

**关键观察：**
- 对于长度相同的递增子序列，结尾元素越小，后续越有可能扩展
- 因此我们只关心每个长度的"最小结尾"

**算法流程：**
```
遍历每个数 x：
    如果 x 比 tails 中所有元素都大 → 追加到末尾，长度+1
    否则 → 找到第一个 >= x 的位置，用 x 替换它
```

**二分查找的作用：**
- tails 数组是单调递增的（可以用反证法证明）
- 所以在 tails 中查找可以用二分，复杂度 O(log n)

## 💻 代码实现

### 方法一：动态规划

```python
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 0:
            return 0
        
        # dp[i] 表示以 nums[i] 结尾的最长递增子序列长度
        dp = [1] * n
        
        for i in range(1, n):
            for j in range(i):
                if nums[j] < nums[i]:
                    dp[i] = max(dp[i], dp[j] + 1)
        
        return max(dp)
```

**执行过程演示：**
```
nums = [10, 9, 2, 5, 3, 7, 101, 18]

i=0, nums[0]=10: dp = [1, 1, 1, 1, 1, 1, 1, 1]
i=1, nums[1]=9:  9<10, 不更新 → dp = [1, 1, 1, 1, 1, 1, 1, 1]
i=2, nums[2]=2:  2<10,2<9, 不更新 → dp = [1, 1, 1, 1, 1, 1, 1, 1]
i=3, nums[3]=5:  5>2, dp[3]=dp[2]+1=2 → dp = [1, 1, 1, 2, 1, 1, 1, 1]
i=4, nums[4]=3:  3>2, dp[4]=2 → dp = [1, 1, 1, 2, 2, 1, 1, 1]
i=5, nums[5]=7:  7>2,7>5,7>3, dp[5]=3 → dp = [1, 1, 1, 2, 2, 3, 1, 1]
i=6, nums[6]=101: 101>所有, dp[6]=4 → dp = [1, 1, 1, 2, 2, 3, 4, 1]
i=7, nums[7]=18:  18>2,5,3,7, dp[7]=4 → dp = [1, 1, 1, 2, 2, 3, 4, 4]

答案: max(dp) = 4
```

### 方法二：贪心 + 二分查找

```python
import bisect

class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        tails = []  # tails[i] = 长度为 i+1 的递增子序列的最小结尾
        
        for x in nums:
            # 找到第一个 >= x 的位置
            pos = bisect.bisect_left(tails, x)
            
            if pos == len(tails):
                # x 比所有结尾都大，可以扩展更长的子序列
                tails.append(x)
            else:
                # 替换，保持该长度下的最小结尾
                tails[pos] = x
        
        return len(tails)
```

**执行过程演示：**
```
nums = [10, 9, 2, 5, 3, 7, 101, 18]

x=10:  tails=[], pos=0, tails=[10]
x=9:   tails=[10], pos=0 (9<10), tails=[9]  ← 替换，因为9更小更有潜力
x=2:   tails=[9], pos=0 (2<9), tails=[2]
x=5:   tails=[2], pos=1 (5>2), tails=[2, 5]
x=3:   tails=[2,5], pos=1 (3<5), tails=[2, 3]  ← 长度为2的子序列，3比5更有潜力
x=7:   tails=[2,3], pos=2 (7>3), tails=[2, 3, 7]
x=101: tails=[2,3,7], pos=3, tails=[2, 3, 7, 101]
x=18:  tails=[2,3,7,101], pos=3 (18<101), tails=[2, 3, 7, 18]

答案: len(tails) = 4
```

**注意：** tails 数组本身不一定是合法的子序列，但它记录的长度是正确的。

## 📊 复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| 动态规划 | O(n²) | O(n) | n ≤ 2500，思路直观 |
| 贪心+二分 | O(n log n) | O(n) | n 很大，追求效率 |

**空间复杂度说明：**
- DP 方法只需要 dp 数组
- 贪心方法只需要 tails 数组，其长度最多为答案长度

## 🔍 关键要点总结

### 1. 为什么 tails 数组是递增的？
用反证法：假设 `tails[i] > tails[i+1]`，那么长度为 `i+2` 的子序列去掉最后一个元素，得到一个长度为 `i+1` 且结尾更小的子序列，这与 `tails[i]` 的定义矛盾。

### 2. 替换操作的正确性
当用 `x` 替换 `tails[pos]` 时，我们保证了：
- 长度为 `pos+1` 的子序列仍然存在（只是换了结尾）
- 新的结尾 `x` 更小，给后续扩展留下更大空间

### 3. 严格递增 vs 非严格递增
- 本题要求**严格递增**：使用 `bisect_left` 找第一个 `>= x` 的位置
- 如果允许**非严格递增**（即可以相等）：使用 `bisect_right` 找第一个 `> x` 的位置

# DP 方法可以记录路径
parent = [-1] * n  # 记录前驱
for j in range(i):
    if nums[j] < nums[i] and dp[j] + 1 > dp[i]:
        dp[i] = dp[j] + 1
        parent[i] = j
