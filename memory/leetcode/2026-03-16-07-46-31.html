LeetCode #31

下一个排列

Medium 数组 双指针

📋 题目描述

整数数组的一个 排列 就是将其所有成员按序列或者线性顺序排列。

例如，arr = [1,2,3] ，以下这些都可以视作 arr 的排列：[1,2,3]、[1,3,2]、[3,1,2]、[2,3,1] 。

整数数组的 下一个排列 是指其整数的下一个字典序更大的排列。更正式地，如果数组的所有排列根据其字典顺序从小到大排列在一个容器中，那么数组的 下一个排列 就是在这个有序容器中排在它后面的那个排列。如果不存在下一个更大的排列，那么这个数组必须重排为 字典序最小的排列（即，其元素按升序排列）。

例如，arr = [1,2,3] 的下一个排列是 [1,3,2] 。

类似地，arr = [2,3,1] 的下一个排列是 [3,1,2] 。

而 arr = [3,2,1] 的下一个排列是 [1,2,3] ，因为 [3,2,1] 不存在一个字典序更大的排列。

示例 1：
输入：nums = [1,2,3]
输出：[1,3,2]
示例 2：
输入：nums = [3,2,1]
输出：[1,2,3]
示例 3：
输入：nums = [1,1,5]
输出：[1,5,1]

💡 解题思路

💡 核心洞察

下一个排列的本质是：找到最小的增量，使得新排列刚好比当前排列大。这要求我们从右往左找到第一个「可以增大」的位置，然后做最小的调整。

算法流程（三步法）

1
从右向左找"拐点"
找到第一个满足 nums[i] < nums[i+1] 的位置 i
↓
2
在右侧找"交换点"
从右向左找到第一个满足 nums[j] > nums[i] 的位置 j
↓
3
交换并反转
交换 nums[i] 和 nums[j]，然后反转 i 之后的所有元素

31class Solution:
    def nextPermutation(self, nums: List[int]) -> None:
        """
        原地修改数组为下一个排列
        三步法：找拐点 → 找交换点 → 交换+反转
        """
        n = len(nums)
        
        # Step 1: 从右向左找第一个升序对 nums[i] < nums[i+1]
        # i 就是"拐点"，需要从这里开始增大
        i = n - 2
        while i >= 0 and nums[i] >= nums[i + 1]:
            i -= 1
        
        # 如果找到了拐点（不是完全降序序列）
        if i >= 0:
            # Step 2: 在右侧从右向左找第一个大于 nums[i] 的数
            # 由于右侧是降序的，所以从右往左第一个大于 nums[i] 的就是最小的大于 nums[i] 的数
            j = n - 1
            while j >= 0 and nums[j] <= nums[i]:
                j -= 1
            
            # Step 3: 交换 nums[i] 和 nums[j]
            nums[i], nums[j] = nums[j], nums[i]
        
        # Step 4: 反转 i 之后的所有元素
        # 如果 i < 0（完全降序），则反转整个数组
        # 否则只反转 i 之后的部分，使其变为升序（最小排列）
        left, right = i + 1, n - 1
        while left < right:
            nums[left], nums[right] = nums[right], nums[left]
            left += 1
            right -= 1

关键代码解析

  • 找拐点（第9-11行）：从右向左遍历，找到第一个 nums[i] < nums[i+1] 的位置。这意味着从 i 开始可以「增大」排列。
  • 找交换点（第16-18行）：在 i 的右侧，从右向左找到第一个大于 nums[i] 的数。由于右侧是降序的，这个数就是「最小的大于 nums[i] 的数」，保证增量最小。
  • 反转（第24-28行）：交换后，i 右侧仍是降序的，反转后变为升序，保证这部分是「以新nums[i]开头的最小排列」。

📊 复杂度分析

指标 复杂度 说明
时间复杂度 O(n) 最坏情况下需要遍历整个数组两次（找拐点和反转），n 为数组长度
空间复杂度 O(1) 只使用常数个额外变量，原地修改数组
🎯 面试要点
  • 为什么不能排序？ 排序时间复杂度 O(n log n)，且有额外空间消耗。本题要求原地修改，且必须是「下一个」排列，不是任意更大排列。
  • 为什么从右往左找？ 因为我们要找「最小的增量」，从右边开始保证只改动尽可能少的低位。
  • 为什么要反转而不是排序？ 因为反转是 O(n) 的，而 i 右侧原本是降序，反转后就是升序，正好是字典序最小的排列。
