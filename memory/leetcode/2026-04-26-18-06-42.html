42. 接雨水

Hard 双指针 动态规划 单调栈 数组
2026-04-26 第238期 | 预计阅读 8 分钟

题目描述

给定 n 个非负整数，表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

输入: height = [0,1,0,2,1,0,1,3,2,1,2,1]
输出: 6
解释: 上面是由数组 [0,1,0,2,1,0,1,3,2,1,2,1] 表示的高度图，
      在这种情况下，可以接 6 个单位的雨水。

约束条件：

  • n == height.length
  • 1 <= n <= 2 × 10⁴
  • 0 <= height[i] <= 10⁵

核心思路

关键观察

每个位置能接的雨水，取决于「左边最高柱」和「右边最高柱」的较小值。当前柱若低于这个较小值，就能接到差值这么多的水。

暴力思路

对每个位置，分别向左、向右扫描找最大高度。时间 O(n²)，空间 O(1)。直观但面试不可接受。

动态规划

预处理每个位置的左边最大值和右边最大值，存到两个数组。然后一次遍历计算雨水量。时间 O(n)，空间 O(n)。

双指针（最优）

左右指针向中间移动，维护左右两侧的最大高度。哪边矮，哪边先计算（因为矮的那边决定了当前位置的水位）。时间 O(n)，空间 O(1)。

单调栈

维护一个递减栈，遇到更高的柱子时弹出计算凹槽内的雨水。按层计算，时间 O(n)，空间 O(n)。适合理解「凹槽」结构。

解题步骤（双指针法）

1

初始化双指针与最大值

left = 0, right = n-1，left_max = 0, right_max = 0。left_max 记录 [0..left] 的最大值，right_max 记录 [right..n-1] 的最大值。

2

移动较矮的一侧

比较 height[left] 和 height[right]：若左边矮，处理左指针；否则处理右指针。核心逻辑：矮的那一侧决定了当前位置的水位上限。

3

处理左指针

如果 height[left] >= left_max，更新 left_max = height[left]（当前柱子成为新的左边界）；否则，当前位置能接 left_max - height[left] 的水，加入答案。然后 left++。

4

处理右指针（对称）

如果 height[right] >= right_max，更新 right_max = height[right]；否则，当前位置能接 right_max - height[right] 的水，加入答案。然后 right--。

5

循环终止

当 left > right 时结束，返回累计雨水量。

class Solution:
    def trap(self, height: List[int]) -> int:
        if not height:
            return 0
        
        left, right = 0, len(height) - 1
        left_max, right_max = 0, 0
        ans = 0
        
        while left <= right:
            if height[left] < height[right]:
                # 左边矮，左边决定水位
                if height[left] >= left_max:
                    # 当前柱高于左边界，成为新的左边界
                    left_max = height[left]
                else:
                    # 可以接水
                    ans += left_max - height[left]
                left += 1
            else:
                # 右边矮（或相等），右边决定水位
                if height[right] >= right_max:
                    # 当前柱高于右边界，成为新的右边界
                    right_max = height[right]
                else:
                    # 可以接水
                    ans += right_max - height[right]
                right -= 1
        
        return ans
class Solution:
    def trap(self, height: List[int]) -> int:
        if not height:
            return 0
        
        n = len(height)
        left_max = [0] * n
        right_max = [0] * n
        
        # 预处理左边最大
        left_max[0] = height[0]
        for i in range(1, n):
            left_max[i] = max(left_max[i-1], height[i])
        
        # 预处理右边最大
        right_max[n-1] = height[n-1]
        for i in range(n-2, -1, -1):
            right_max[i] = max(right_max[i+1], height[i])
        
        # 计算雨水
        ans = 0
        for i in range(n):
            ans += min(left_max[i], right_max[i]) - height[i]
        
        return ans
class Solution:
    def trap(self, height: List[int]) -> int:
        stack = []  # 维护递减栈，存下标
        ans = 0
        
        for i, h in enumerate(height):
            while stack and h > height[stack[-1]]:
                # 当前柱子高于栈顶，可以形成凹槽接水
                top = stack.pop()
                if not stack:
                    break  # 只有一侧有边界，无法接水
                
                # 凹槽宽度
                distance = i - stack[-1] - 1
                # 凹槽高度 = 两侧较矮边界 - 底部高度
                bounded_height = min(h, height[stack[-1]]) - height[top]
                ans += distance * bounded_height
            
            stack.append(i)
        
        return ans
class Solution {
    public int trap(int[] height) {
        int left = 0, right = height.length - 1;
        int leftMax = 0, rightMax = 0;
        int ans = 0;
        
        while (left <= right) {
            if (height[left] < height[right]) {
                if (height[left] >= leftMax) {
                    leftMax = height[left];
                } else {
                    ans += leftMax - height[left];
                }
                left++;
            } else {
                if (height[right] >= rightMax) {
                    rightMax = height[right];
                } else {
                    ans += rightMax - height[right];
                }
                right--;
            }
        }
        return ans;
    }
}

复杂度分析

解法 时间 空间 适用场景

动态规划 O(n) O(n) 思路清晰，容易写对

关键洞察与易错点

常见错误

  • 双指针法中，height[left] < height[right] 用 <= 也可以，但逻辑要对称。关键是「矮的一侧决定水位」。
  • 单调栈法容易忘记 if not stack: break，弹出后栈空说明只有单侧边界，无法形成凹槽。
  • DP 法中忘记取 min(left_max, right_max)，误用 max 导致水溢出。
  • 边界情况：数组长度小于 3 时无法接水，应直接返回 0。

面试追问

  • 为什么双指针法正确？ → 当 height[left] < height[right] 时，left 位置的水位由 left_max 决定（因为 right 侧存在比 left 更高的柱子，right_max 必然 ≥ height[right] > height[left]，所以 min(left_max, right_max) = left_max）。
  • 能否接彩色雨水？ → 按层/按柱都可以，本质一样。
  • 扩展：接雨水 II（407题） → 三维版本，需要用优先队列（最小堆）从边界向内部推进，时间 O(mn log(mn))。
  • 扩展：柱状图中最大的矩形（84题） → 本题是找凹槽，84题是找凸峰，都用单调栈，但处理逻辑相反（84用递增栈找左右第一个更矮的）。

第7步: h[3]=2 == h[8]=2，处理右边。2>=2 更新 right_max=2，right=7
第8步: h[3]=2 < h[7]=3，处理左边。2>=1 更新 left_max=2，left=4
