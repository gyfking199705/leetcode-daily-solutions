🎯 LeetCode #148 - 排序链表

Sort List | 归并排序 + 快慢指针找中点 | Hard

Hard 归并排序 链表 分治

📋 题目描述

给你链表的头结点 head ，请将其按 升序 排列并返回 排序后的链表 。

进阶要求：

  • 你可以在 O(n log n) 时间复杂度和常数级空间复杂度下，对链表进行排序吗？

示例 1：

输入：head = [4,2,1,3]
输出：[1,2,3,4]

示例 2：

输入：head = [-1,5,3,4,0]
输出：[-1,0,3,4,5]

核心考点：

  • 归并排序：链表排序最优选择，天然适合 O(n log n) 时间
  • 快慢指针：找到链表中点，将链表一分为二
  • 链表合并：双指针技巧合并两个有序链表
  • 递归分治：自顶向下的分治策略

💡 解题思路详解

1. 为什么选择归并排序？

链表无法像数组那样快速随机访问，所以快速排序不适合（需要partition时频繁访问）。而归并排序：

  • 只需要顺序访问，完美适配链表
  • 稳定排序，时间复杂度稳定为 O(n log n)
  • 可以自底向上实现常数空间

2. 快慢指针找中点

快指针走2步，慢指针走1步。当快指针到达末尾，慢指针正好在中点。关键点：找到中点后要断开链表。

3. 合并两个有序链表

使用虚拟头节点 dummy 简化操作，双指针逐个比较，小的先接入结果链表。

4. 递归终止条件

当链表为空或只有一个节点时，直接返回（已经有序）。
