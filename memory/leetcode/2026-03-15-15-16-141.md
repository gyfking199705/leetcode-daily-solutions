# LeetCode 141. 环形链表 - 详细精讲

## 题目信息

**题号**: 141. Linked List Cycle  
**难度**: Easy  
**标签**: 哈希表、链表、双指针  
**核心算法**: 快慢指针（Floyd's Cycle Detection Algorithm）

---

## 题目描述

给定一个链表的头节点 `head`，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达该节点，则链表中存在环。为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。**注意：`pos` 不作为参数进行传递**。仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 `true` ；否则，返回 `false` 。

**示例 1：**
```
输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点。
```

**示例 2：**
```
输入：head = [1,2], pos = 0
输出：true
解释：链表中有一个环，其尾部连接到第一个节点。
```

**示例 3：**
```
输入：head = [1], pos = -1
输出：false
解释：链表中没有环。
```

**约束条件：**
- 链表中节点的数目范围是 `[0, 10^4]`
- `-10^5 <= Node.val <= 10^5`
- `pos` 为 `-1` 或者链表中的一个有效索引

## 解题思路

### 方法一：哈希表法（直观但非最优）

**思路**：遍历链表，用哈希表记录访问过的节点。如果遇到一个已经访问过的节点，说明有环。

```python
def hasCycle(self, head: Optional[ListNode]) -> bool:
    visited = set()
    current = head
    while current:
        if current in visited:
            return True
        visited.add(current)
        current = current.next
    return False
```

**时间复杂度**: O(n)  
**空间复杂度**: O(n) — 需要存储访问过的节点

---

### 方法二：快慢指针（Floyd判圈算法）⭐ 推荐

**核心思想**：想象两个人在环形跑道上跑步，速度快的人一定会追上速度慢的人。

**算法步骤**：
1. 初始化两个指针：`slow`（慢指针，每次走1步）和 `fast`（快指针，每次走2步）
2. 同时从起点出发
3. 如果链表无环，`fast` 会先到达末尾（`fast` 或 `fast.next` 为 `None`）
4. 如果链表有环，`fast` 会在环内追上 `slow`（两者相遇）

**为什么快指针一定能追上慢指针？**

假设环的长度为 `L`，当慢指针进入环时：
- 慢指针在环内位置为 `0`
- 快指针可能在环内某个位置 `k`

每次迭代：
- 慢指针前进一步：`+1`
- 快指针前进两步：`+2`

相对速度为 `2 - 1 = 1`（快指针每步靠近慢指针1个单位）。由于环是有限的，快指针必定能追上慢指针。

**为什么需要检查 `fast` 和 `fast.next`？**

```
fast = fast.next.next  # 快指针走两步
```

如果 `fast` 为 `None`，说明链表已结束，无环。  
如果 `fast.next` 为 `None`，`fast.next.next` 会报错，所以需要检查两者。

## 完整代码实现

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        """
        使用快慢指针判断链表是否有环
        
        Args:
            head: 链表头节点
            
        Returns:
            bool: True表示有环，False表示无环
        """
        # 边界情况：空链表或只有一个节点，不可能有环
        if not head or not head.next:
            return False
        
        # 初始化快慢指针
        slow = head  # 慢指针，每次走一步
        fast = head.next  # 快指针，每次走两步（从第二个节点开始避免初始相遇）
        
        # 循环直到快指针到达末尾
        while fast and fast.next:
            # 检查是否相遇
            if slow == fast:
                return True
            
            # 慢指针前进一步
            slow = slow.next
            # 快指针前进两步
            fast = fast.next.next
        
        # 快指针到达末尾，说明无环
        return False
```

**另一种更常见的写法**（快指针也从 head 开始）：

```python
class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        if not head or not head.next:
            return False
        
        slow = head
        fast = head
        
        while fast and fast.next:
            slow = slow.next      # 慢指针走一步
            fast = fast.next.next # 快指针走两步
            
            if slow == fast:      # 相遇则有环
                return True
        
        return False
```

## 复杂度分析

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| **时间复杂度** | O(n) | 无环时遍历整个链表；有环时，设环外长度为 `a`，环长为 `b`，则最多遍历 `a + b` 次 |
| **空间复杂度** | O(1) | 只使用了两个指针，常数额外空间 |

**为什么是 O(n) 而不是 O(n²)？**

有环情况下：
- 设链表头到环入口距离为 `a`
- 环的长度为 `b`
- 慢指针进入环后，快指针最多再追 `b` 步就能相遇
- 总步数 ≈ `a + b`，即 O(n)

---

## 关键要点总结

1. **快慢指针的核心原理**：相对速度为1，保证环内必相遇
2. **边界条件处理**：空链表、单节点链表需要特殊判断
3. **循环终止条件**：`fast` 和 `fast.next` 都要检查（快指针走两步）
4. **空间优化**：相比哈希表的 O(n) 空间，双指针只需 O(1) 空间

## 扩展：找到环的入口

如果题目要求返回环的入口节点（LeetCode 142），可以在相遇后：

```python
def detectCycle(head):
    # 第一阶段：判断是否有环，找到相遇点
    slow = fast = head
    has_cycle = False
    
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            has_cycle = True
            break
    
    if not has_cycle:
        return None
    
    # 第二阶段：找入口
    # 数学证明：头节点到入口距离 = 相遇点到入口距离
    ptr = head
    while ptr != slow:
        ptr = ptr.next
        slow = slow.next
    
    return ptr  # 环的入口
```

**数学证明简述**：设头到入口为 `a`，入口到相遇点为 `b`，相遇点回入口为 `c`。
- 慢指针走的距离：`a + b`
- 快指针走的距离：`a + b + n(b + c)`（n圈）
- 由于快是慢的2倍：`2(a + b) = a + b + n(b + c)`
- 化简得：`a = (n-1)(b+c) + c`
- 即：从头走 `a` 步 = 从相遇点走 `n-1` 圈 + `c` 步，两者在入口相遇

## 面试高频考点

1. **为什么快指针走2步，可以走3步吗？**  
   可以，但2步是最优的。走更多步可能跳过慢指针导致多绕圈，且代码更复杂。

2. **为什么一定能相遇？**  
   相对速度为1，每步距离减少1，环长有限，必相遇。

3. **空间复杂度为什么重要？**  
   大规模数据时，O(n)的哈希表可能内存溢出，O(1)的双指针更安全。
