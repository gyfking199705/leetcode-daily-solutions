# 🎯 LeetCode 精讲 #23 - 合并K个升序链表

---

### 📋 题目描述

**来源**: LeetCode 第23题  
**难度**: Hard  
**链接**: https://leetcode.cn/problems/merge-k-sorted-lists/

#### 题目原文
> 给你一个链表数组，每个链表都已经按升序排列。
> 
> 请你将所有链表合并到一个升序链表中，返回合并后的链表。

#### 示例

**示例 1:**
```
输入: lists = [[1,4,5],[1,3,4],[2,6]]
输出: [1,1,2,3,4,4,5,6]
解释: 链表数组如下:
[
  1->4->5,
  1->3->4,
  2->6
]
将它们合并到一个有序链表中得到:
1->1->2->3->4->4->5->6
```

**示例 2:**
```
输入: lists = []
输出: []
```

**示例 3:**
```
输入: lists = [[]]
输出: []
```

#### 约束条件
- `k == lists.length`
- `0 <= k <= 10^4`
- `0 <= lists[i].length <= 500`
- `-10^4 <= lists[i][j] <= 10^4`
- `lists[i]` 按升序排列
- `lists[i].length` 的总和不超过 `10^4`

### 💡 解题思路

这道题是「合并两个有序链表」的进阶版，核心在于**如何高效地从k个链表中找到最小节点**。

#### 方法一：顺序合并（暴力法）

**核心思想**
逐个合并链表，先合并lists[0]和lists[1]，结果再与lists[2]合并，以此类推。

**复杂度分析**
- 时间：O(k² × n)，其中n是平均链表长度。第i次合并需要遍历O(i×n)个节点。
- 空间：O(1)，仅使用常数额外空间

---

#### 方法二：分治合并 ⭐推荐

**核心思想**
采用归并排序的思想，两两配对合并，每次合并后链表数量减半。

**图解过程**
```
初始: [L1, L2, L3, L4, L5, L6, L7, L8]

第1轮: 两两合并
  L1+L2 → M12
  L3+L4 → M34
  L5+L6 → M56
  L7+L8 → M78
结果: [M12, M34, M56, M78]

第2轮: 继续合并
  M12+M34 → M1234
  M56+M78 → M5678
结果: [M1234, M5678]

第3轮: 最终合并
  M1234+M5678 → 最终结果
```

**复杂度分析**
- 时间：O(kn × log k)，共log k轮合并，每轮处理所有kn个节点
- 空间：O(log k)，递归栈深度

---

#### 方法三：优先队列（最小堆）⭐面试最佳

**核心思想**
维护一个大小为k的小顶堆，堆顶始终是当前k个链表头中的最小值。每次取出最小值后，将该节点所在链表的下一个节点入堆。

**图解过程**
```
lists = [[1,4,5],[1,3,4],[2,6]]

初始状态（堆中存放各链表头节点）:
堆: [1(L0), 1(L1), 2(L2)]  ← 两个1，按链表索引区分

步骤1: 弹出最小值1(L0)，结果链表: 1
       将L0的下一个节点4入堆
堆: [1(L1), 2(L2), 4(L0)]

步骤2: 弹出最小值1(L1)，结果链表: 1→1
       将L1的下一个节点3入堆
堆: [2(L2), 3(L1), 4(L0)]

步骤3: 弹出最小值2(L2)，结果链表: 1→1→2
       将L2的下一个节点6入堆
堆: [3(L1), 4(L0), 6(L2)]

... 继续直到堆为空
```

**复杂度分析**
- 时间：O(kn × log k)，每个节点入堆出堆一次，堆操作O(log k)
- 空间：O(k)，堆的大小最多为k

### 🔧 代码实现

#### 分治法代码（Python）

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        if not lists:
            return None
        
        # 分治：两两合并
        def mergeTwo(l1: ListNode, l2: ListNode) -> ListNode:
            dummy = ListNode(0)
            tail = dummy
            
            while l1 and l2:
                if l1.val <= l2.val:
                    tail.next = l1
                    l1 = l1.next
                else:
                    tail.next = l2
                    l2 = l2.next
                tail = tail.next
            
            # 连接剩余部分
            tail.next = l1 if l1 else l2
            return dummy.next
        
        # 迭代式分治，避免递归深度问题
        n = len(lists)
        interval = 1
        
        while interval < n:
            for i in range(0, n - interval, interval * 2):
                lists[i] = mergeTwo(lists[i], lists[i + interval])
            interval *= 2
        
        return lists[0] if lists else None
```

#### 优先队列代码（Python）

```python
import heapq

# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        # 自定义包装类，使ListNode可比较
        class Wrapper:
            def __init__(self, node):
                self.node = node
            def __lt__(self, other):
                return self.node.val < other.node.val
        
        # 初始化小顶堆
        heap = []
        for l in lists:
            if l:
                heapq.heappush(heap, Wrapper(l))
        
        dummy = ListNode(0)
        tail = dummy
        
        # 不断弹出最小值，并入堆下一个节点
        while heap:
            wrapper = heapq.heappop(heap)
            node = wrapper.node
            tail.next = node
            tail = tail.next
            
            if node.next:
                heapq.heappush(heap, Wrapper(node.next))
        
        return dummy.next
```

#### 优先队列代码（Java）

```java
import java.util.PriorityQueue;

class ListNode {
    int val;
    ListNode next;
    ListNode(int x) { val = x; }
}

class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        if (lists == null || lists.length == 0) {
            return null;
        }
        
        // 小顶堆，按节点值排序
        PriorityQueue<ListNode> heap = new PriorityQueue<>((a, b) -> a.val - b.val);
        
        // 初始化：将k个链表的头节点入堆
        for (ListNode head : lists) {
            if (head != null) {
                heap.offer(head);
            }
        }
        
        ListNode dummy = new ListNode(0);
        ListNode tail = dummy;
        
        // 每次取出最小值，并入堆下一个节点
        while (!heap.isEmpty()) {
            ListNode minNode = heap.poll();
            tail.next = minNode;
            tail = tail.next;
            
            if (minNode.next != null) {
                heap.offer(minNode.next);
            }
        }
        
        return dummy.next;
    }
}
```

#### 优先队列代码（C++）

```cpp
struct ListNode {
    int val;
    ListNode *next;
    ListNode() : val(0), next(nullptr) {}
    ListNode(int x) : val(x), next(nullptr) {}
    ListNode(int x, ListNode *next) : val(x), next(next) {}
};

class Solution {
public:
    // 自定义比较函数
    struct Compare {
        bool operator()(ListNode* a, ListNode* b) {
            return a->val > b->val;  // 小顶堆用大于号
        }
    };
    
    ListNode* mergeKLists(vector<ListNode*>& lists) {
        // 小顶堆
        priority_queue<ListNode*, vector<ListNode*>, Compare> heap;
        
        // 初始化
        for (ListNode* head : lists) {
            if (head != nullptr) {
                heap.push(head);
            }
        }
        
        ListNode* dummy = new ListNode(0);
        ListNode* tail = dummy;
        
        while (!heap.empty()) {
            ListNode* minNode = heap.top();
            heap.pop();
            
            tail->next = minNode;
            tail = tail->next;
            
            if (minNode->next != nullptr) {
                heap.push(minNode->next);
            }
        }
        
        return dummy->next;
    }
};
```

### 📊 复杂度对比

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| 顺序合并 | O(k²n) | O(1) | k很小，或内存极受限 |
| 分治合并 | O(kn·log k) | O(log k) | k较大，递归深度可控 |
| 优先队列 | O(kn·log k) | O(k) | **面试首选**，代码简洁 |

### 🎯 关键要点总结

1. **核心问题**：从k个有序序列中找全局最小值 → 用堆优化到O(log k)

2. **为什么不用排序所有元素？**
   - 链表不支持随机访问，无法直接排序
   - 各链表已经有序，利用这个特性可以做到O(kn·log k)
   - 完全排序需要O(kn·log(kn))，且需要额外空间

3. **边界情况处理**：
   - `lists`为空 → 返回None
   - `lists`中有空链表 → 入堆时跳过
   - 所有链表都为空 → 堆为空，返回None

4. **Python的堆注意事项**：
   - 默认ListNode没有比较方法，需要包装成可比较的对象
   - 或者存入`(val, idx, node)`元组，用idx打破相等值的比较歧义
