LeetCode #142 - 环形链表 II

Medium Floyd判圈算法 · 双指针

📋 题目描述

给定一个链表的头节点 head，返回链表开始入环的第一个节点。如果链表无环，则返回 null。

核心问题：不仅判断是否有环（141题），还要找到环的入口位置！

示例：

输入：head = [3,2,0,-4], pos = 1
输出：索引为 1 的节点（值为 2 的节点）
解释：链表中有一个环，其尾部连接到第二个节点。

🔍 算法原理：Floyd判圈算法

🧮 数学证明（为什么相遇后能找到入口）

设：从头节点到环入口距离为 a，环入口到相遇点为 b，相遇点回到环入口为 c

  • 慢指针走的距离：a + b
  • 快指针走的距离：a + b + n(b + c) = 2(a + b) （快是慢的2倍）
  • 推导得：a = c + (n-1)(b+c)
  • 结论：从头节点和相遇点同时出发，最终会在环入口相遇！

📊 复杂度分析

指标 复杂度 说明
时间复杂度 O(n) 最多遍历链表两次（快慢指针相遇 + 找入口）
空间复杂度 O(1) 只使用两个指针，常数额外空间

💡 关键技巧总结

  • 第一阶段：快慢指针找到相遇点（证明有环）
  • 第二阶段：一个指针回到头，两指针同速前进，相遇点即入口
  • 数学本质：利用速度差2倍建立方程，消去环的周长项
  • 边界：快指针/快指针.next为空则无环
