## 🎯 LeetCode 精讲 #3 - 无重复字符的最长子串

#### 题目原文

> 给定一个字符串 `s`，请你找出其中**不含有重复字符的 最长子串**的长度。

---

#### 示例

**示例 1：**
```
输入: s = "abcabcbb"
输出: 3 
解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。
```

**示例 2：**
```
输入: s = "bbbbb"
输出: 1
解释: 因为无重复字符的最长子串是 "b"，所以其长度为 1。
```

**示例 3：**
```
输入: s = "pwwkew"
输出: 3
解释: 因为无重复字符的最长子串是 "wke"，所以其长度为 3。
     请注意，你的答案必须是 子串 的长度，"pwke" 是一个子序列，不是子串。
```

#### 约束条件

- `0 <= s.length <= 5 * 10^4`
- `s` 由英文字母、数字、符号和空格组成

### 💡 解题思路

#### 🔍 核心洞察：滑动窗口

这道题的关键在于：**我们需要找到一个连续的子串，其中没有任何重复字符**。

如果暴力枚举所有子串并检查是否有重复字符，时间复杂度是 O(n³)，显然不够高效。

**关键观察**：
1. 当我们从左向右遍历字符串时，如果当前窗口 `[left, right]` 内没有重复字符，我们可以安全地将 `right` 向右移动，扩大窗口
2. 一旦发现 `s[right]` 在窗口内已经存在，我们需要收缩左边界 `left`，直到重复字符被移出窗口
3. 在这个过程中，我们始终维护一个"无重复字符"的窗口，并记录其最大长度

这就是**滑动窗口**算法的精髓：**动态调整窗口边界，始终保持窗口内满足特定条件**。

---

#### 🧠 算法设计

**数据结构选择**：
- 使用 **HashSet** 来存储当前窗口内的字符，实现 O(1) 的查找
- 或使用 **哈希表** 记录字符的最新位置，进一步优化

**双指针 + 哈希表优化解法**：

```
初始化:
    left = 0                    // 窗口左边界
    max_len = 0                 // 记录最大长度
    char_index = {}             // 哈希表：字符 → 最新位置

遍历 right 从 0 到 n-1:
    if s[right] 在 char_index 中且位置 >= left:
        // 发现重复，需要移动左边界
        left = char_index[s[right]] + 1
    
    // 更新字符最新位置
    char_index[s[right]] = right
    
    // 更新最大长度
    max_len = max(max_len, right - left + 1)

返回 max_len
```

### 📝 完整代码

#### Python 实现

```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        """
        滑动窗口 + 哈希表
        时间复杂度: O(n)
        空间复杂度: O(min(m, n))，m为字符集大小
        """
        char_index = {}  # 记录字符最新出现的位置
        left = 0         # 窗口左边界
        max_len = 0      # 最大长度
        
        for right, char in enumerate(s):
            # 如果字符已在窗口内，移动左边界
            if char in char_index and char_index[char] >= left:
                left = char_index[char] + 1
            
            # 更新字符位置
            char_index[char] = right
            
            # 更新最大长度
            current_len = right - left + 1
            max_len = max(max_len, current_len)
        
        return max_len
```

#### C++ 实现

```cpp
#include <string>
#include <unordered_map>
#include <algorithm>

class Solution {
public:
    int lengthOfLongestSubstring(std::string s) {
        std::unordered_map<char, int> charIndex;  // 字符 -> 最新位置
        int left = 0;
        int maxLen = 0;
        
        for (int right = 0; right < s.length(); ++right) {
            char c = s[right];
            
            // 如果字符在窗口内，移动左边界
            if (charIndex.find(c) != charIndex.end() && charIndex[c] >= left) {
                left = charIndex[c] + 1;
            }
            
            // 更新字符位置
            charIndex[c] = right;
            
            // 更新最大长度
            maxLen = std::max(maxLen, right - left + 1);
        }
        
        return maxLen;
    }
};
```

#### Java 实现

```java
import java.util.HashMap;
import java.util.Map;

class Solution {
    public int lengthOfLongestSubstring(String s) {
        Map<Character, Integer> charIndex = new HashMap<>();
        int left = 0;
        int maxLen = 0;
        
        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            
            // 如果字符在窗口内，移动左边界
            if (charIndex.containsKey(c) && charIndex.get(c) >= left) {
                left = charIndex.get(c) + 1;
            }
            
            // 更新字符位置
            charIndex.put(c, right);
            
            // 更新最大长度
            maxLen = Math.max(maxLen, right - left + 1);
        }
        
        return maxLen;
    }
}
```

### ⏱️ 复杂度分析

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| **时间复杂度** | O(n) | 每个字符最多被访问两次（right指针和left指针各一次） |
| **空间复杂度** | O(min(m, n)) | m是字符集大小，n是字符串长度。哈希表最多存储当前窗口的字符 |

**为什么时间复杂度是 O(n)？**
- `right` 指针从左到右遍历一次，共 n 次
- `left` 指针最多也从左到右移动 n 次
- 总操作次数是 2n，即 O(n)

### 🎯 面试高频考点

1. **子串 vs 子序列的区别？**
   - 子串：连续的一段字符
   - 子序列：不需要连续，保持相对顺序即可

2. **为什么不用 HashSet 而用 HashMap？**
   - HashSet 也能做，但需要额外的循环来找到重复字符的位置
   - HashMap 直接记录位置，可以将 `left` 一步到位，更高效

3. **`char_index[char] >= left` 这个条件能去掉吗？**
   - **不能！** 这是关键优化。例如 `s = "abba"`：
     - 当 right=3 遇到第二个 'a'，第一个 'a' 在位置 0
     - 但此时 left 已经移动到 2（因为中间遇到了 'b' 的重复）
     - 如果不检查 `>= left`，会把 left 错误地移回 1

4. **空间复杂度为什么是 O(min(m, n))？**
   - 当字符集很小时（如只有小写字母，m=26），最多存 26 个键
   - 当字符串很短时，最多存 n 个字符
   - 所以是 min(m, n)

### 💡 核心要点总结

1. **滑动窗口的本质**：动态维护一个满足条件的区间，通过双指针调整边界
2. **双指针移动策略**：右指针扩展窗口，左指针收缩窗口（当条件不满足时）
3. **哈希表优化**：记录字符位置实现 O(1) 跳转，避免不必要的遍历
4. **边界条件**：`char_index[c] >= left` 是防止重复字符不在当前窗口时的误判
