# LeetCode 15 - 三数之和 精讲报告

**日期**: 2026-03-16  
**题号**: 15  
**标题**: 三数之和 (3Sum)  
**难度**: Medium  
**核心方法**: 排序 + 双指针

---

## 一、题目描述

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足：
- `i != j`、`i != k` 且 `j != k`
- `nums[i] + nums[j] + nums[k] == 0`

**返回所有和为 0 且不重复的三元组**。

**注意**：答案中不可以包含重复的三元组。

### 示例

```
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
- nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0
- nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0
- nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0
不同的三元组是 [-1,0,1] 和 [-1,-1,2]
注意：输出的顺序和三元组的顺序并不重要
```

### 约束条件
- `3 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

---

## 二、解题思路

### 2.1 暴力解法（不可行）

三重循环枚举所有三元组，时间复杂度 O(n³)，对于 n=3000 会超时。

### 2.2 最优解法：排序 + 双指针

这是解决 K-Sum 类问题的**标准范式**。

#### 核心思想
将问题转化为 **"固定一个数，在剩余数组中找两数之和等于目标值"**。

#### 为什么要排序？

1. **方便去重**：排序后相同数字相邻，可以通过比较相邻元素来跳过重复值
2. **使用双指针**：有序数组中，根据当前和的大小可以决定指针移动方向

#### 算法步骤

1. **排序**：将数组从小到大排序
2. **固定第一个数**：遍历数组，`nums[i]` 作为三元组的第一个数
   - **剪枝优化**：如果 `nums[i] > 0`，因为数组已排序，后面的数都大于0，三数之和不可能为0，直接结束循环
   - **去重**：如果 `i > 0` 且 `nums[i] == nums[i-1]`，说明这个数字作为第一个数已经处理过了，跳过
3. **双指针寻找后两个数**：
   - 左指针 `left = i + 1`，右指针 `right = n - 1`
   - 当 `left < right` 时循环：
     - 计算总和 `s = nums[i] + nums[left] + nums[right]`
     - **如果 `s == 0`**：找到一组解，加入结果集，移动左右指针并跳过重复值
     - **如果 `s < 0`**：和太小，需要变大，`left++`
     - **如果 `s > 0`**：和太大，需要变小，`right--`

## 三、代码实现

### Python 实现

```python
class Solution:
    def threeSum(self, nums: list[int]) -> list[list[int]]:
        n = len(nums)
        if n < 3:
            return []
        
        nums.sort()  # 排序
        res = []
        
        for i in range(n - 2):
            # 剪枝：当前数已经大于0，后面不可能有三数之和为0
            if nums[i] > 0:
                break
            
            # 去重：跳过相同的第一个数
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            
            left, right = i + 1, n - 1
            
            while left < right:
                total = nums[i] + nums[left] + nums[right]
                
                if total == 0:
                    res.append([nums[i], nums[left], nums[right]])
                    
                    # 去重：跳过相同的左指针值
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    # 去重：跳过相同的右指针值
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    
                    left += 1
                    right -= 1
                    
                elif total < 0:
                    left += 1  # 和太小，左指针右移
                else:
                    right -= 1  # 和太大，右指针左移
        
        return res
```

### Java 实现

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> res = new ArrayList<>();
        int n = nums.length;
        if (n < 3) return res;
        
        Arrays.sort(nums);
        
        for (int i = 0; i < n - 2; i++) {
            if (nums[i] > 0) break;  // 剪枝
            if (i > 0 && nums[i] == nums[i - 1]) continue;  // 去重
            
            int left = i + 1, right = n - 1;
            
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                
                if (sum == 0) {
                    res.add(Arrays.asList(nums[i], nums[left], nums[right]));
                    
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;
                    
                    left++;
                    right--;
                } else if (sum < 0) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        
        return res;
    }
}
```

                    right--;
                } else if (sum < 0) {

                } else {
                    right--;
                }
            }
        }

};

## 五、关键要点总结

### 5.1 去重逻辑（面试常考！）

本题最大的难点在于**如何优雅地去重**，需要三层去重：

1. **外层循环去重**：`if (i > 0 && nums[i] == nums[i-1]) continue`
   - 必须在 `i > 0` 时判断，否则 `i-1` 越界

2. **左指针去重**：找到解后，`while (left < right && nums[left] == nums[left+1]) left++`

3. **右指针去重**：找到解后，`while (left < right && nums[right] == nums[right-1]) right--`

### 5.2 剪枝优化

`if (nums[i] > 0) break` 非常关键：
- 数组已排序，如果当前数 > 0，后续所有数都 ≥ 它
- 三数之和不可能等于 0，直接退出循环
- 这在数组有大量正数时能大幅提升效率

### 5.3 双指针的精髓

- 将三重循环优化为双重循环
- 利用有序性，根据当前和的大小决定指针移动方向
- 这是从 O(n³) 优化到 O(n²) 的核心

## 八、易错点提醒

1. **去重时越界**：判断 `i > 0` 再去比较 `nums[i]` 和 `nums[i-1]`
2. **指针移动顺序**：找到解后，先跳过重复值，再移动指针
3. **左指针起点**：必须从 `i + 1` 开始，避免重复使用元素
4. **break vs continue**：`nums[i] > 0` 用 break（直接退出），重复元素用 continue（跳过本次）
