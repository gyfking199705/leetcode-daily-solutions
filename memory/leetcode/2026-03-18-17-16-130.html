LeetCode #130 - 被围绕的区域

Medium DFS / BFS 逆向思维 + 边界搜索的经典例题

📋 题目描述

给你一个 m x n 的矩阵 board ，由若干字符 'X' 和 'O' 组成，找到所有被 'X' 围绕的区域，并将这些区域里所有的 'O' 用 'X' 填充。

🎯 核心考点

这道题的本质是「连通区域」问题。直接处理比较困难，关键技巧是逆向思维：

  • 被围绕的区域 = 不与边界相连的 'O' 区域
  • 从四条边界的 'O' 出发，标记所有与边界相连的区域
  • 最后遍历整个矩阵，未标记的 'O' 就是被围绕的区域

📝 示例

输入：
board = [
  ["X","X","X","X"],
  ["X","O","O","X"],
  ["X","X","O","X"],
  ["X","O","X","X"]
]
输出：
[
  ["X","X","X","X"],
  ["X","X","X","X"],
  ["X","X","X","X"],
  ["X","O","X","X"]
]

解释：中间的三个 'O' 被 'X' 围绕，最下面的 'O' 与边界相连，保留。

💡 关键洞察

这道题和「岛屿数量」的区别在于：岛屿问题找连通块，本题则是找与边界不连通的区域。从边界开始搜索（而非内部）是解题的关键。

📊 复杂度分析

指标 复杂度 说明
时间复杂度 O(m × n) 每个单元格最多被访问两次（标记阶段 + 翻转阶段）
空间复杂度 O(m × n) 递归栈深度，最坏情况下（全是'O'）为矩阵大小
🚀 优化方向

BFS版本： 使用队列替代递归栈，空间复杂度相同但避免了递归深度限制。

并查集版本： 将边界'O'连接到一个虚拟节点，最后检查每个'O'是否与虚拟节点连通。

🎯 解题思路总结

  1. 逆向思维： 正难则反，不从内部找被围绕区域，而是从边界找「不被围绕」的区域
  2. 边界扫描： 遍历四条边，对每条边上的'O'启动DFS/BFS
  3. 标记连通： 将所有与边界相连的'O'标记为特殊字符（如'E'）
  4. 最终处理： 遍历矩阵，'O'→'X'（被围绕），'E'→'O'（恢复）
🧠 举一反三

这种「从边界反向搜索」的技巧还可以应用于：

  • 太平洋大西洋水流问题： 从海洋边界反向找能到达该海洋的单元格
  • 逃离迷宫： 从出口反向BFS找最短路径
  • 连通性判断： 判断内部点是否与边界连通
