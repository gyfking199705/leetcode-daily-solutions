## 🎯 LeetCode 精讲 #4 - 寻找两个正序数组的中位数

---

### 📋 题目描述

**难度**: 困难  
**链接**: https://leetcode.cn/problems/median-of-two-sorted-arrays/

给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。请你找出并返回这两个正序数组的 **中位数** 。

算法的时间复杂度应该为 `O(log (m+n))` 。

---

**示例 1：**
```
输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并数组 = [1,2,3] ，中位数 2
```

**示例 2：**
```
输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并数组 = [1,2,3,4] ，中位数 (2 + 3) / 2 = 2.5
```

**提示：**
- `nums1.length == m`
- `nums2.length == n`
- `0 <= m <= 1000`
- `0 <= n <= 1000`
- `1 <= m + n <= 2000`
- `-10^6 <= nums1[i], nums2[i] <= 10^6`

---

### 💡 解题思路

这道题是 LeetCode 上著名的"困难"题，核心在于 `O(log(m+n))` 的时间复杂度要求。

#### 方法一：合并数组（暴力法）
- **思路**：直接合并两个有序数组，然后取中位数
- **时间复杂度**：O(m+n)
- **不符合要求**，仅用于理解

#### 方法二：二分查找（正解）⭐

##### 核心思想

**问题转化**：找中位数 = 找第 k 小的数

- 如果 `m+n` 是奇数，中位数是第 `(m+n+1)/2` 小的数
- 如果 `m+n` 是偶数，中位数是第 `(m+n)/2` 小和第 `(m+n)/2+1` 小的数的平均值

**二分策略**：
每次从两个数组中各排除 `k/2` 个元素，直到找到第 k 小的数。

##### 具体步骤

1. **确保 nums1 是较短的数组**：减少边界情况处理
2. **递归/迭代找第 k 小**：
   - 比较 `nums1[k/2-1]` 和 `nums2[k/2-1]`
   - 如果 `nums1[k/2-1] < nums2[k/2-1]`，说明 `nums1` 的前 `k/2` 个元素都不可能是第 k 小的数，排除它们
   - 反之亦然
3. **边界处理**：当一个数组被排除完时，直接在另一个数组中取

### 📝 参考代码

```python
class Solution:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        """
        寻找两个有序数组的中位数
        核心思想：二分查找第 k 小的数
        """
        # 确保 nums1 是较短的数组，简化边界处理
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1
        
        m, n = len(nums1), len(nums2)
        total = m + n
        
        # 中位数位置
        # 奇数：第 (total+1)//2 个
        # 偶数：第 total//2 和 total//2 + 1 个的平均值
        left, right = (total + 1) // 2, (total + 2) // 2
        
        # 只需要找两次第 k 小的数
        return (self.findKth(nums1, 0, nums2, 0, left) + 
                self.findKth(nums1, 0, nums2, 0, right)) / 2
    
    def findKth(self, nums1: list[int], i: int, nums2: list[int], j: int, k: int) -> int:
        """
        寻找两个有序数组中第 k 小的数
        nums1[i:] 和 nums2[j:] 表示当前考虑的子数组
        """
        # 边界情况1：nums1 已经用完
        if i >= len(nums1):
            return nums2[j + k - 1]
        
        # 边界情况2：nums2 已经用完
        if j >= len(nums2):
            return nums1[i + k - 1]
        
        # 边界情况3：只需要找第1小的数
        if k == 1:
            return min(nums1[i], nums2[j])
        
        # 正常二分情况
        # 从 nums1 和 nums2 各取 k//2 个元素进行比较
        mid1 = float('inf')  # 如果 nums1 剩余元素不足 k//2，设为无穷大
        mid2 = float('inf')  # 如果 nums2 剩余元素不足 k//2，设为无穷大
        
        if i + k // 2 - 1 < len(nums1):
            mid1 = nums1[i + k // 2 - 1]
        if j + k // 2 - 1 < len(nums2):
            mid2 = nums2[j + k // 2 - 1]
        
        # 比较并排除较小的那一部分
        if mid1 < mid2:
            # nums1 的前 k//2 个元素不可能是第 k 小的，排除
            return self.findKth(nums1, i + k // 2, nums2, j, k - k // 2)
        else:
            # nums2 的前 k//2 个元素不可能是第 k 小的，排除
            return self.findKth(nums1, i, nums2, j + k // 2, k - k // 2)


# 更简洁的迭代版本（推荐面试使用）
class SolutionIterative:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        """
        迭代版本：使用指针移动来模拟二分
        """
        m, n = len(nums1), len(nums2)
        p1 = p2 = 0  # 两个指针
        
        # 需要找到的位置
        mid_index = (m + n) // 2
        prev = curr = 0  # 记录上一个和当前值
        
        for i in range(mid_index + 1):
            prev = curr
            # 选择较小的那个移动指针
            if p1 < m and (p2 >= n or nums1[p1] <= nums2[p2]):
                curr = nums1[p1]
                p1 += 1
            else:
                curr = nums2[p2]
                p2 += 1
        
        # 根据总长度的奇偶性返回结果
        if (m + n) % 2 == 1:
            return float(curr)  # 奇数，返回当前值
        else:
            return (prev + curr) / 2.0  # 偶数，返回平均值


# 最简洁的二分查找版本（最优解）
class SolutionOptimal:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        """
        最优二分：在较短的数组上进行二分
        时间复杂度：O(log(min(m,n)))
        """
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1
        
        m, n = len(nums1), len(nums2)
        left, right = 0, m
        
        # 目标：左半部分有 (m+n+1)//2 个元素
        half_len = (m + n + 1) // 2
        
        while left <= right:
            # i 表示 nums1 中取多少个元素到左半部分
            i = (left + right) // 2
            # j 表示 nums2 中取多少个元素到左半部分
            j = half_len - i
            
            # 边界值处理
            nums1_left = float('-inf') if i == 0 else nums1[i - 1]
            nums1_right = float('inf') if i == m else nums1[i]
            nums2_left = float('-inf') if j == 0 else nums2[j - 1]
            nums2_right = float('inf') if j == n else nums2[j]
            
            # 检查是否满足条件：左半部分的所有元素 <= 右半部分的所有元素
            if nums1_left <= nums2_right and nums2_left <= nums1_right:
                # 找到了正确的分割点
                if (m + n) % 2 == 1:
                    # 奇数，返回左半部分的最大值
                    return max(nums1_left, nums2_left)
                else:
                    # 偶数，返回左半部分最大值和右半部分最小值的平均
                    left_max = max(nums1_left, nums2_left)
                    right_min = min(nums1_right, nums2_right)
                    return (left_max + right_min) / 2.0
            
            elif nums1_left > nums2_right:
                # nums1 的左半部分太大，需要向左移动
                right = i - 1
            else:
                # nums1 的左半部分太小，需要向右移动
                left = i + 1
        
        raise ValueError("Input arrays are not sorted!")
```

### 🔑 关键要点

#### 1. 问题转化技巧
```
找中位数 → 找第 k 小的数
奇数长度：第 (n+1)/2 个
偶数长度：第 n/2 和第 n/2+1 个的平均值
```

#### 2. 二分查找的核心逻辑
```
每次排除 k/2 个元素，问题规模减半
比较 A[k/2-1] 和 B[k/2-1]：
  - 较小的一方前 k/2 个元素不可能包含第 k 小
  - 排除它们，问题变成找第 k-k/2 小
```

#### 3. 最优解的巧妙之处
在较短的数组上进行二分，将复杂度降到 `O(log(min(m,n)))`：
- 不用真的找第 k 小，而是找到一个"分割点"
- 使得左半部分元素个数 = 右半部分元素个数（或差1）
- 且左半部分的所有元素 <= 右半部分的所有元素

#### 4. 边界处理技巧
```python
# 用无穷大/无穷小处理边界
nums1_left = float('-inf') if i == 0 else nums1[i - 1]
nums1_right = float('inf') if i == m else nums1[i]
```
