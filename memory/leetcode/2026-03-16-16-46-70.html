LeetCode #70 - 爬楼梯 Easy

动态规划入门经典 | 斐波那契数列模型 | 空间优化技巧

📋 题目描述

假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

示例 1：
输入：n = 2
输出：2
解释：有两种方法可以爬到楼顶
1. 1 阶 + 1 阶
2. 2 阶
示例 2：
输入：n = 3
输出：3
解释：有三种方法可以爬到楼顶
1. 1 阶 + 1 阶 + 1 阶
2. 1 阶 + 2 阶
3. 2 阶 + 1 阶
💡 核心思路

到达第 n 阶楼梯的方法数 = 到达第 n-1 阶的方法数 + 到达第 n-2 阶的方法数

因为最后一步要么是跨1阶，要么是跨2阶。

🔄 状态转移可视化

1f(1)
→
2f(2)
→
3f(3)
→
5f(4)
→
8f(5)

递推公式：f(n) = f(n-1) + f(n-2)

🎬 算法执行过程（点击步骤或代码行查看对应关系）

Step 1 初始化边界条件
f(1)=1, f(2)=2
dp[1]=1, dp[2]=2
Step 2 进入递推循环
从第3阶开始计算
i = 3
Step 3 状态转移
当前方法数 = 前一阶 + 前两阶
dp[3] = dp[2] + dp[1] = 2 + 1 = 3
Step 4 返回结果
返回第n阶的方法数
return dp[n]

Python 实现（动态规划）



1class Solution:


2    def climbStairs(self, n: int) -> int:


3        # 边界情况处理


4        if n <= 2:


5            return n


6        


7        # dp[i] 表示到达第i阶的方法数


8        dp = [0] * (n + 1)


9        dp[1], dp[2] = 1, 2


10        


11        for i in range(3, n + 1):


12            dp[i] = dp[i - 1] + dp[i - 2]


13        


14        return dp[n]

⚡ 空间优化版本

观察发现，计算 dp[i] 只需要前两个状态，因此可以将空间复杂度从 O(n) 优化到 O(1)。



1class Solution:


2    def climbStairs(self, n: int) -> int:


3        if n <= 2:


4            return n


5        


6        # 只保留前两个状态


7        prev2, prev1 = 1, 2  # f(n-2), f(n-1)


8        


9        for _ in range(3, n + 1):


10            curr = prev1 + prev2


11            prev2 = prev1


12            prev1 = curr


13        


14        return prev1
🎯 空间优化原理

滚动变量技巧：用 prev2、prev1、curr 三个变量滚动前进，无需数组存储所有中间状态。

📊 复杂度分析

解法 时间复杂度 空间复杂度 说明
动态规划（数组） O(n) O(n) 直观易懂，适合理解状态转移
动态规划（滚动变量） O(n) O(1) 最优空间，面试推荐写法
矩阵快速幂 O(log n) O(1) 纯炫技，实际不常用
通项公式 O(1) O(1) 斐波那契通项，有精度问题
