K个一组翻转链表

Hard 链表 递归 2026-04-27

题目描述

给你链表的头节点 head ，每 k 个节点一组进行翻转，请你返回修改后的链表。

k 是一个正整数，它的值小于或等于链表的长度。

如果节点总数不是 k 的整数倍，那么请将最后剩余的节点保持原有顺序。

你不能只是单纯的改变节点内部的值，而是需要实际进行节点交换。

🎯 交互式演示 (k = 3)

💡 解题思路

核心思想：分段翻转 + 连接

关键观察： 这道题是 206 题「反转链表」的进阶版。206 题翻转整条链表，本题要求每 k 个节点翻转一次。

解题框架：
  1. 先判断剩余节点是否够 k 个，不够则直接返回
  2. 够 k 个：翻转这 k 个节点（类似 206 题的翻转逻辑）
  3. 翻转后，原来的头变成尾，需要递归处理后面的链表
  4. 将翻转后的尾节点指向递归返回的新头
递归思维： 假设后面的链表已经处理好了（递归调用），我们只需要处理好当前这 k 个节点的翻转，并正确连接即可。

连接逻辑：翻转后的尾节点（原来的头节点）→ 后面处理好的新头
Python3 简洁递归实现 
class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        # Step 1: 检查剩余节点是否够k个
        count = 0
        cur = head
        while cur and count < k:
            cur = cur.next
            count += 1
        
        # 不够k个，不翻转，直接返回
        if count < k:
            return head
        
        # Step 2: 翻转这k个节点 (类似206题)
        prev, curr = None, head
        for _ in range(k):
            nxt = curr.next
            curr.next = prev
            prev = curr
            curr = nxt
        
        # Step 3: head现在是翻转后的尾节点
        # 递归处理剩余部分，并连接
        head.next = self.reverseKGroup(curr, k)
        
        # Step 4: prev是翻转后的新头
        return prev

📊 复杂度分析

O(n)
时间复杂度
O(n/k)
递归栈空间

每个节点恰好被访问两次（一次计数，一次翻转），时间复杂度线性。
递归深度为链表长度除以 k，即 n/k 层。
