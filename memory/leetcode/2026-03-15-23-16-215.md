# LeetCode 215 - 数组中的第K个最大元素

**日期**: 2026-03-15

**难度**: Medium  
**标签**: Top K, 快速选择, 堆

## 题目描述

给定整数数组 `nums` 和整数 `k`，请返回数组中第 `k` 个最大的元素。

注意：你需要找的是数组排序后的第 `k` 个**最大**的元素，而不是第 `k` 个**不同**的元素（重复元素算多个）。

**示例 1：**
```
输入: nums = [3,2,1,5,6,4], k = 2
输出: 5
解释: 排序后 [1,2,3,4,5,6]，第2大是5
```

**示例 2：**
```
输入: nums = [3,2,3,1,2,4,5,5,6], k = 4
输出: 4
解释: 排序后 [1,2,2,3,3,4,5,5,6]，第4大是4
```

---

## 核心考点

这道题是 **Top K 问题** 的基础模板，考察：
1. **快速选择算法 (Quick Select)** — 平均 O(n) 时间复杂度
2. **堆/优先队列** — O(n log k) 空间换时间
3. **分治思想** — 快排分区(partition)的变体

---

## 解法一：快速选择算法（推荐）

### 算法思想

快速选择是快速排序的变种。我们不需要对整个数组完全排序，只需要找到第 k 大的元素。

**关键洞察**：
- 第 k 大 = 升序排序后索引为 `n-k` 的元素
- 每次分区(partition)后，pivot 的位置就是它在最终排序中的位置
- 如果 pivot 位置 > 目标位置 → 在左半边继续找
- 如果 pivot 位置 < 目标位置 → 在右半边继续找

### 完整代码

```python
import random
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        """
        寻找数组中第 k 个最大的元素
        时间复杂度: 期望 O(n)
        空间复杂度: O(log n) - 递归栈
        """
        # 第 k 大 = 升序索引 n-k
        target_index = len(nums) - k
        return self.quickSelect(nums, 0, len(nums) - 1, target_index)
    
    def quickSelect(self, nums: List[int], left: int, right: int, target: int) -> int:
        """快速选择核心逻辑"""
        if left == right:
            return nums[left]
        
        # 分区操作
        pivot_index = self.partition(nums, left, right)
        
        if pivot_index == target:
            return nums[pivot_index]
        elif pivot_index < target:
            # 目标在右半部分
            return self.quickSelect(nums, pivot_index + 1, right, target)
        else:
            # 目标在左半部分
            return self.quickSelect(nums, left, pivot_index - 1, target)
    
    def partition(self, nums: List[int], left: int, right: int) -> int:
        """
        分区函数（Lomuto 分区方案）
        返回 pivot 最终所在的索引
        """
        # 【关键优化】随机选择 pivot，避免最坏 O(n²)
        rand_idx = random.randint(left, right)
        nums[rand_idx], nums[right] = nums[right], nums[rand_idx]
        
        pivot = nums[right]
        i = left  # i 指向"<= pivot 区域"的下一个位置
        
        for j in range(left, right):
            if nums[j] <= pivot:
                nums[i], nums[j] = nums[j], nums[i]
                i += 1
        
        # 将 pivot 放到正确位置
        nums[i], nums[right] = nums[right], nums[i]
        return i
```

### 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 平均 | **O(n)** | 每次期望处理一半数据: n + n/2 + n/4 + ... = 2n |
| 最坏 | O(n²) | 每次分区极不平衡（如已排序数组）|
| 空间 | O(log n) | 递归调用栈深度 |

**为什么平均是 O(n)？**
```
总工作量 = n + n/2 + n/4 + n/8 + ...
         = n × (1 + 1/2 + 1/4 + ...)
         = n × 2 = 2n = O(n)
```

## 解法二：最小堆（工程常用）

```python
import heapq
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        """
        维护大小为 k 的最小堆
        时间复杂度: O(n log k)
        空间复杂度: O(k)
        """
        min_heap = []
        
        for num in nums:
            if len(min_heap) < k:
                heapq.heappush(min_heap, num)
            elif num > min_heap[0]:
                # 当前数比堆顶大，替换堆顶
                heapq.heapreplace(min_heap, num)
        
        # 堆顶就是第 k 大
        return min_heap[0]
```

**适用场景**：
- 数据流场景（无法一次性加载全部数据）
- k << n 时效率更高
- 代码简单，不易出错

## 解法三：Pythonic 写法（面试保底）

```python
class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        return heapq.nlargest(k, nums)[-1]
```

或者一行排序（不符合 O(n) 要求，但面试可提）：
```python
return sorted(nums, reverse=True)[k-1]  # O(n log n)
```

## 面试追问要点

### Q1: 为什么随机化 pivot 很重要？
如果不随机，在已排序数组上每次选第一个/最后一个作为 pivot，会导致分区极不平衡，退化为 O(n²)。随机化后期望达到 O(n)。

### Q2: 如果要求第 k 小呢？
- 第 k 小 = 升序索引 k-1
- 或者直接用 max 堆（大小为 k）

### Q3: 海量数据（无法放入内存）怎么办？
使用 **最小堆**：
1. 维护大小为 k 的堆
2. 流式处理数据
3. 空间只需 O(k)

## 核心要点总结

1. **Top K 问题的通解**：快速选择 O(n) / 最小堆 O(n log k)
2. **索引转换**：第 k 大 ↔ 升序索引 `n-k`
3. **随机化**：快速选择必须随机 pivot，否则最坏 O(n²)
4. **分区函数**：是快排/快速选择的核心，务必掌握
5. **工程实践**：堆解法代码简洁，实际开发更常用

> 💡 **面试技巧**：先提排序保底，再讲快速选择展示深度，最后提堆展示工程思维。
