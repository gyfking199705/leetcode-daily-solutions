LeetCode #322 - 零钱兑换

完全背包问题 | 动态规划 | 经典面试题

📋 题目描述

给你一个整数数组 coins，表示不同面额的硬币；以及一个整数 amount，表示总金额。

计算并返回可以凑成总金额所需的最少硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。

你可以认为每种硬币的数量是无限的。

示例 1：

输入: coins = [1,2,5], amount = 11

输出: 3

解释: 11 = 5 + 5 + 1

示例 2：

输入: coins = [2], amount = 3

输出: -1

📊 复杂度分析

维度 值 说明
时间复杂度 O(n × amount) n是硬币种类数，需要遍历每个硬币和每个金额
空间复杂度 O(amount) 一维DP数组，只需存储每个金额的最少硬币数

💡 核心思路解析

1. 状态定义

dp[i] = 凑成金额 i 所需的最少硬币个数

2. 状态转移方程

dp[i] = min(dp[i], dp[i - coin] + 1)

对于每个硬币coin，如果coin ≤ i，则考虑使用这个硬币

3. 初始化

dp[0] = 0（凑成0元需要0个硬币）

dp[i] = amount + 1 或 float('inf')（表示初始不可达）
