5. 最长回文子串

Longest Palindromic Substring

📅 2026-04-27 18:06 🏷️ 字符串 / 双指针 / 动态规划 ⭐ 高频指数: ★★★★★

📋题目描述

给你一个字符串 s，找到 s 中最长的回文子串。

回文串：正读和反读都相同的字符串。例如 "aba"、"racecar"。

子串：连续的字符序列，与"子序列"不同。

示例 1：
输入: s = "babad"
输出: "bab"（或 "aba"，两者长度相同）

示例 2：
输入: s = "cbbd"
输出: "bb"

💡核心观察

回文串的对称性

回文串的核心特征是中心对称。对于一个回文串，存在一个"中心"（单个字符或两个字符之间的间隙），从中心向两侧扩展时，对应的字符始终相等。

关键洞察：如果已知一个子串是回文的，那么在其两侧添加相同的字符，新子串仍然是回文的。

例如："aba" 是回文 → "cabac" 也是回文（两侧都加 'c'）

这个观察直接导向中心扩展法：遍历每个可能的中心，向两边扩展，记录最长的回文串。

中心扩展法（面试推荐）

思路：回文串的中心可能是单个字符（奇数长度如 "aba"）或两个字符之间（偶数长度如 "abba"）。遍历每个位置作为中心，向两侧扩展直到不再回文。

def longestPalindrome(self, s: str) -> str: if not s or len(s) < 1: return "" start, end = 0, 0 for i in range(len(s)): # 奇数长度回文（中心是单个字符） len1 = self.expandAroundCenter(s, i, i) # 偶数长度回文（中心是两个字符之间） len2 = self.expandAroundCenter(s, i, i + 1) max_len = max(len1, len2) if max_len > end - start: # 计算起始和结束位置 start = i - (max_len - 1) // 2 end = i + max_len // 2 return s[start:end + 1] def expandAroundCenter(self, s, left, right): while left >= 0 and right < len(s) and s[left] == s[right]: left -= 1 right += 1 return right - left - 1 # 实际长度
复杂度值说明
时间O(n²)共 2n-1 个中心，每个最多扩展 n 次
空间O(1)仅使用常数额外空间
面试要点：这是面试中最推荐的解法。思路直观，代码简洁，空间复杂度最优。注意奇偶中心的处理技巧。

动态规划

思路：定义 dp[i][j] 表示子串 s[i:j+1] 是否为回文串。利用"如果两端相等且内部是回文，则整体是回文"的状态转移。

状态转移：
dp[i][j] = (s[i] == s[j]) and (j - i < 3 or dp[i+1][j-1])

# j-i<3 表示长度≤3，此时只要两端相等必是回文
def longestPalindrome(self, s: str) -> str: n = len(s) if n < 2: return s dp = [[False] * n for _ in range(n)] start, max_len = 0, 1 # 单个字符都是回文 for i in range(n): dp[i][i] = True # 按长度从小到大填表 for length in range(2, n + 1): for i in range(n - length + 1): j = i + length - 1 if s[i] == s[j]: if length <= 3: dp[i][j] = True else: dp[i][j] = dp[i + 1][j - 1] if dp[i][j] and length > max_len: start = i max_len = length return s[start:start + max_len]
复杂度值说明
时间O(n²)双重循环填表
空间O(n²)二维DP数组
面试要点：DP解法展示了对子问题分解的理解。注意填表顺序（按长度从小到大），确保计算 dp[i][j] 时 dp[i+1][j-1] 已经计算过。

Manacher算法（线性时间最优解）

思路：通过在字符间插入特殊字符（如 #），统一处理奇偶长度，然后利用回文串的对称性质，避免重复计算。

核心优化：维护当前最右回文串的右边界 right 和中心 center。对于位置 i，如果 i 在 right 左侧，则其关于 center 对称的位置 j 的信息可以直接利用（取边界限制）。
def longestPalindrome(self, s: str) -> str: # 预处理：插入 # 统一奇偶 t = '#' + '#'.join(s) + '#' n = len(t) p = [0] * n # p[i] = 以i为中心的回文半径 center = right = 0 for i in range(n): # 利用对称位置信息 if i < right: mirror = 2 * center - i p[i] = min(right - i, p[mirror]) # 尝试扩展 while i + p[i] + 1 < n and i - p[i] - 1 >= 0 and \ t[i + p[i] + 1] == t[i - p[i] - 1]: p[i] += 1 # 更新最右边界 if i + p[i] > right: center, right = i, i + p[i] # 找到最大半径 max_len, center_idx = max((v, i) for i, v in enumerate(p)) start = (center_idx - max_len) // 2 return s[start:start + max_len]
复杂度值说明
时间O(n)线性时间，每个字符最多被访问常数次
空间O(n)预处理后的字符串和半径数组
面试要点：Manacher是"加分项"。面试中先写中心扩展，提到Manacher是线性最优解即可。除非明确要求最优复杂度，否则不需要现场写出完整Manacher。

🎯面试连环追问

1
如果要求输出所有最长回文子串？
中心扩展时，若发现与当前max_len相等的回文，加入结果集。注意去重。
2
回文子串的个数？
DP解法中统计所有 dp[i][j]=True 的个数，或中心扩展时每个中心扩展次数之和。
3
最长回文子序列（不连续）？
变成经典DP：dp[i][j] = dp[i+1][j-1]+2（若s[i]==s[j]）否则 max(dp[i+1][j], dp[i][j-1])。
4
在流式输入中实时找最长回文？
EERTREE（回文树）数据结构，每个字符O(1)均摊时间维护所有回文后缀。
5
空间优化到O(1)怎么做？
中心扩展天然O(1)。DP能否优化？不行，因为需要子问题信息。Manacher需要O(n)。
