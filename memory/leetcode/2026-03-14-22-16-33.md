## 🎯 LeetCode 精讲 #33 - 搜索旋转排序数组

---

### 📋 题目描述

**原题链接**: https://leetcode.cn/problems/search-in-rotated-sorted-array/

整数数组 `nums` 按升序排列，数组中的值 **互不相同**。

在传递给函数之前，`nums` 在预先未知的某个下标 `k`（`0 <= k < nums.length`）上进行了 **旋转**，使数组变为 `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（下标 **从 0 开始** 计数）。

给你 **旋转后** 的数组 `nums` 和一个整数 `target` ，如果 `nums` 中存在这个目标值 `target` ，则返回它的下标，否则返回 `-1` 。

**示例 1：**
```
输入：nums = [4,5,6,7,0,1,2], target = 0
输出：4
```

**示例 2：**
```
输入：nums = [4,5,6,7,0,1,2], target = 3
输出：-1
```

**示例 3：**
```
输入：nums = [1], target = 0
输出：-1
```

**提示：**
- `1 <= nums.length <= 5000`
- `-10^4 <= nums[i] <= 10^4`
- `nums` 中的每个值都 **独一无二**
- 题目数据保证 `nums` 在预先未知的某个下标上进行了旋转
- `-10^4 <= target <= 10^4`

**进阶：** 你可以设计一个时间复杂度为 `O(log n)` 的解决方案吗？

---

### 💡 解题思路

#### 核心观察 🔍

这道题是二分查找的经典变种。旋转后的数组虽然整体无序，但有一个关键性质：

> **将数组从中间分开，一定有一半是有序的，另一半可能有序也可能无序**

例如：`[4,5,6,7,0,1,2]`
- 从中间分开：`[4,5,6,7]` 和 `[0,1,2]`
- 左边 `[4,5,6,7]` 是有序的（升序）
- 右边 `[0,1,2]` 也是有序的

再看：`[6,7,0,1,2,3,4]`
- 从中间分开：`[6,7,0]` 和 `[1,2,3,4]`
- 左边无序，右边 `[1,2,3,4]` 有序

#### 算法步骤 📝

1. **初始化**：左右指针 `left = 0`, `right = len(nums) - 1`

2. **二分循环**：当 `left <= right` 时循环
   
   a. 计算中点：`mid = left + (right - left) // 2`
   
   b. **找到目标**：如果 `nums[mid] == target`，返回 `mid`
   
   c. **判断哪半边有序**：
      - 如果 `nums[left] <= nums[mid]`：左半边有序
      - 否则：右半边有序
   
   d. **在有序半边判断目标位置**：
      
      **情况1：左半边有序** (`nums[left] <= nums[mid]`)
      ```
      判断 target 是否在 [nums[left], nums[mid]) 范围内：
      - 如果是：right = mid - 1（去左边找）
      - 否则：left = mid + 1（去右边找）
      ```
      
      **情况2：右半边有序** (`nums[mid] < nums[left]`)
      ```
      判断 target 是否在 (nums[mid], nums[right]] 范围内：
      - 如果是：left = mid + 1（去右边找）
      - 否则：right = mid - 1（去左边找）
      ```

3. **未找到**：返回 `-1`

#### 为什么这样判断？🤔

关键在于：**有序的一半可以帮助我们确定目标值是否在这一半**

- 如果左半边有序且 `target` 在左半边的范围内 → 去左边找
- 如果左半边有序但 `target` 不在范围内 → 目标只能在右边
- 右半边同理

#### 图示理解 📊

```
数组：[4, 5, 6, 7, 0, 1, 2]，target = 0

初始：left=0(4), right=6(2)
      mid=3(7)
      
      左半边 [4,5,6,7] 有序
      target=0 不在 [4,7] 范围内
      → 去右边，left = 4

第2轮：left=4(0), right=6(2)
       mid=5(1)
       
       左半边 [0,1] 相对于原数组有序
       target=0 在 [0,1] 范围内
       → 去左边，right = 4

第3轮：left=4(0), right=4(0)
       mid=4(0) == target
       → 返回 4 ✓
```

### 📝 参考代码

```python
class Solution:
    def search(self, nums: list[int], target: int) -> int:
        """
        在旋转排序数组中搜索目标值
        时间复杂度: O(log n)
        空间复杂度: O(1)
        """
        if not nums:
            return -1
        
        left, right = 0, len(nums) - 1
        
        while left <= right:
            # 计算中点（防止整数溢出）
            mid = left + (right - left) // 2
            
            # 情况1：直接找到目标
            if nums[mid] == target:
                return mid
            
            # 情况2：左半边有序 [left, mid] 是升序
            if nums[left] <= nums[mid]:
                # 判断 target 是否在左半边范围内
                # 注意：因为 nums[mid] != target，所以范围是 [left, mid)
                if nums[left] <= target < nums[mid]:
                    # target 在左半边，去左边找
                    right = mid - 1
                else:
                    # target 不在左半边，去右边找
                    left = mid + 1
            
            # 情况3：右半边有序 (mid, right] 是升序
            else:
                # 判断 target 是否在右半边范围内
                # 注意：因为 nums[mid] != target，所以范围是 (mid, right]
                if nums[mid] < target <= nums[right]:
                    # target 在右半边，去右边找
                    left = mid + 1
                else:
                    # target 不在右半边，去左边找
                    right = mid - 1
        
        # 未找到目标
        return -1


# ==================== 测试代码 ====================

def test_solution():
    """测试各种情况"""
    sol = Solution()
    
    # 测试用例1：基本旋转数组
    nums1 = [4, 5, 6, 7, 0, 1, 2]
    assert sol.search(nums1, 0) == 4, f"Test1 failed: expected 4, got {sol.search(nums1, 0)}"
    assert sol.search(nums1, 3) == -1, f"Test1 failed: expected -1"
    print("✓ 测试用例1通过：基本旋转数组")
    
    # 测试用例2：单元素数组
    nums2 = [1]
    assert sol.search(nums2, 0) == -1
    assert sol.search(nums2, 1) == 0
    print("✓ 测试用例2通过：单元素数组")
    
    # 测试用例3：未旋转的数组
    nums3 = [1, 2, 3, 4, 5]
    assert sol.search(nums3, 3) == 2
    assert sol.search(nums3, 6) == -1
    print("✓ 测试用例3通过：未旋转数组")
    
    # 测试用例4：边界值
    nums4 = [1, 3]
    assert sol.search(nums4, 3) == 1
    assert sol.search(nums4, 1) == 0
    print("✓ 测试用例4通过：两元素数组")
    
    # 测试用例5：大范围旋转
    nums5 = [5, 6, 7, 8, 9, 1, 2, 3, 4]
    assert sol.search(nums5, 9) == 4
    assert sol.search(nums5, 1) == 5
    print("✓ 测试用例5通过：大范围旋转")
    
    print("\n🎉 所有测试用例通过！")


if __name__ == "__main__":
    test_solution()
```

#### 代码关键点解读 🔑

1. **`mid = left + (right - left) // 2`**
   - 防止 `(left + right)` 溢出的写法
   - 在 Python 中整数不会溢出，但这是良好习惯

2. **`nums[left] <= nums[mid]` 的判断**
   - 注意是 **<=** 不是 **<**
   - 当 `left == mid` 时（两元素情况），需要 `=` 来判断左半边有序

3. **边界条件的包含/不包含**
   - `nums[left] <= target < nums[mid]`：左边包含 `left`，不包含 `mid`（因为已经判断过 `nums[mid] != target`）
   - `nums[mid] < target <= nums[right]`：右边不包含 `mid`，包含 `right`

### ⏱️ 复杂度分析

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| **时间复杂度** | O(log n) | 每次循环将搜索范围减半，二分查找特性 |
| **空间复杂度** | O(1) | 只使用了常数额外空间（left, right, mid）|

**最优情况**：O(1) - 中点直接命中目标

**最坏情况**：O(log n) - 需要一直二分直到范围为空

---

### 🔑 关键要点总结

1. **核心洞察**：旋转数组中，中点分割后必有一半有序

2. **判断策略**：
   - 先找有序的一半
   - 再看目标是否在有序半边的范围内
   - 在 → 去有序半边
   - 不在 → 去无序半边

3. **易错点**：
   - 边界条件的开闭（是否包含 `mid`）
   - `nums[left] <= nums[mid]` 要用 `<=`
   - 单元素、双元素等边界情况的测试

4. **变体延伸**：
   - 有重复元素的情况（需要额外处理 `nums[left] == nums[mid]`）
   - 找旋转点（最小值）
   - 找峰值元素
