## 📝 题目描述

给你一个链表数组，每个链表都已经按升序排列。

请你将所有链表合并到一个升序链表中，返回合并后的链表。

**示例 1：**
```
输入：lists = [[1,4,5],[1,3,4],[2,6]]
输出：[1,1,2,3,4,4,5,6]
解释：链表数组如下：
[
  1->4->5,
  1->3->4,
  2->6
]
将它们合并到一个有序链表中得到：
1->1->2->3->4->4->5->6
```

**示例 2：**
```
输入：lists = []
输出：[]
```

**示例 3：**
```
输入：lists = [[]]
输出：[]
```

**约束条件：**
- `k == lists.length`
- `0 <= k <= 10^4`
- `0 <= lists[i].length <= 500`
- `-10^4 <= lists[i][j] <= 10^4`
- `lists[i]` 按升序排列
- `lists[i].length` 的总和不超过 `10^4`

## 💡 解题思路

### 方法一：顺序合并（暴力法）

**思路：**
逐个合并链表。先合并第1个和第2个，然后将结果与第3个合并，依此类推。

**复杂度分析：**
- 时间复杂度：O(k² × n)，其中k是链表数量，n是平均链表长度。每次合并两个链表需要O(n)，总共需要合并k-1次，每次合并的链表长度递增。
- 空间复杂度：O(1)，只使用常数额外空间

**缺点：** 效率较低，当k很大时性能差。

---

### 方法二：分治合并（推荐）⭐

**思路：**
采用归并排序的思想，两两配对合并。将k个链表分成两组，每组分别合并，然后将两个结果再合并。

```
原始：[L1, L2, L3, L4, L5, L6, L7, L8]
第一轮：(L1,L2), (L3,L4), (L5,L6), (L7,L8) → 4个结果
第二轮：(L12,L34), (L56,L78) → 2个结果  
第三轮：(L1234,L5678) → 1个结果
```

**为什么快？**
- 每次都将问题规模减半，总共只需要 log(k) 轮合并
- 每轮合并的总工作量是 O(k × n)
- 总时间复杂度：O(k × n × log k)

**复杂度分析：**
- 时间复杂度：O(N × log k)，其中N是所有节点的总数，k是链表数量
- 空间复杂度：O(log k)，递归调用栈的深度

---

### 方法三：优先队列（最小堆）⭐

**思路：**
维护一个大小为k的最小堆，堆中存放每个链表的当前头节点。每次从堆中取出最小值加入结果链表，然后将该节点所在链表的下一个节点加入堆。

**步骤：**
1. 初始化：将所有非空链表的头节点加入最小堆
2. 循环直到堆为空：
   - 取出堆顶元素（当前最小值），加入结果链表
   - 如果该节点有下一个节点，将下一个节点加入堆
3. 返回结果链表

**复杂度分析：**
- 时间复杂度：O(N × log k)，每个节点进出堆一次，堆操作O(log k)
- 空间复杂度：O(k)，堆的大小最多为k

## 🔧 完整代码实现

### Python 实现（分治法）

```python
# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        if not lists:
            return None
        
        # 分治合并
        return self.mergeRange(lists, 0, len(lists) - 1)
    
    def mergeRange(self, lists: List[Optional[ListNode]], left: int, right: int) -> Optional[ListNode]:
        # 基本情况：只有一个链表
        if left == right:
            return lists[left]
        
        # 分成两半，分别合并
        mid = left + (right - left) // 2
        l1 = self.mergeRange(lists, left, mid)
        l2 = self.mergeRange(lists, mid + 1, right)
        
        # 合并两个有序链表
        return self.mergeTwoLists(l1, l2)
    
    def mergeTwoLists(self, l1: Optional[ListNode], l2: Optional[ListNode]) -> Optional[ListNode]:
        # 哑节点简化边界处理
        dummy = ListNode(0)
        current = dummy
        
        # 双指针合并
        while l1 and l2:
            if l1.val <= l2.val:
                current.next = l1
                l1 = l1.next
            else:
                current.next = l2
                l2 = l2.next
            current = current.next
        
        # 连接剩余部分
        current.next = l1 if l1 else l2
        
        return dummy.next
```

### Python 实现（优先队列/堆）

```python
import heapq

# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
    
    # 定义比较方法，使ListNode可比较（Python 3需要）
    def __lt__(self, other):
        return self.val < other.val

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        # 过滤空链表并初始化堆
        # heapq默认使用 < 比较，我们需要ListNode支持比较
        heap = []
        
        for i, lst in enumerate(lists):
            if lst:
                # (值, 链表索引, 节点)
                # 使用索引作为第二关键字，避免值相同时比较ListNode
                heapq.heappush(heap, (lst.val, i, lst))
        
        # 哑节点
        dummy = ListNode(0)
        current = dummy
        
        # 不断从堆中取出最小值
        while heap:
            val, idx, node = heapq.heappop(heap)
            
            # 将最小节点加入结果链表
            current.next = node
            current = current.next
            
            # 如果该链表还有后续节点，加入堆
            if node.next:
                heapq.heappush(heap, (node.next.val, idx, node.next))
        
        return dummy.next
```

### Java 实现（优先队列）

```java
import java.util.PriorityQueue;

class ListNode {
    int val;
    ListNode next;
    ListNode(int x) { val = x; }
}

class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        if (lists == null || lists.length == 0) {
            return null;
        }
        
        // 最小堆，按节点值排序
        PriorityQueue<ListNode> minHeap = new PriorityQueue<>(
            (a, b) -> a.val - b.val
        );
        
        // 将所有链表的头节点加入堆
        for (ListNode list : lists) {
            if (list != null) {
                minHeap.offer(list);
            }
        }
        
        // 哑节点
        ListNode dummy = new ListNode(0);
        ListNode current = dummy;
        
        // 不断取出最小值
        while (!minHeap.isEmpty()) {
            ListNode node = minHeap.poll();
            current.next = node;
            current = current.next;
            
            // 将下一个节点加入堆
            if (node.next != null) {
                minHeap.offer(node.next);
            }
        }
        
        return dummy.next;
    }
}
```

### C++ 实现（分治法）

```cpp
struct ListNode {
    int val;
    ListNode *next;
    ListNode() : val(0), next(nullptr) {}
    ListNode(int x) : val(x), next(nullptr) {}
    ListNode(int x, ListNode *next) : val(x), next(next) {}
};

class Solution {
public:
    ListNode* mergeKLists(vector<ListNode*>& lists) {
        if (lists.empty()) return nullptr;
        return mergeRange(lists, 0, lists.size() - 1);
    }
    
private:
    ListNode* mergeRange(vector<ListNode*>& lists, int left, int right) {
        if (left == right) return lists[left];
        
        int mid = left + (right - left) / 2;
        ListNode* l1 = mergeRange(lists, left, mid);
        ListNode* l2 = mergeRange(lists, mid + 1, right);
        
        return mergeTwoLists(l1, l2);
    }
    
    ListNode* mergeTwoLists(ListNode* l1, ListNode* l2) {
        ListNode dummy(0);
        ListNode* current = &dummy;
        
        while (l1 && l2) {
            if (l1->val <= l2->val) {
                current->next = l1;
                l1 = l1->next;
            } else {
                current->next = l2;
                l2 = l2->next;
            }
            current = current->next;
        }
        
        current->next = l1 ? l1 : l2;
        return dummy.next;
    }
};
```

## 📊 复杂度对比

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| 顺序合并 | O(k² × n) | O(1) | k很小时 |
| **分治合并** | **O(N × log k)** | **O(log k)** | **通用，推荐** |
| **优先队列** | **O(N × log k)** | **O(k)** | k适中，代码简洁 |

- N = 所有链表节点总数
- k = 链表数量
- n = 平均每个链表的长度

## 🎯 关键要点总结

### 1. 分治法的核心思想
```
问题：合并k个链表
分解：合并前k/2个 + 合并后k/2个
解决：合并两个有序链表（基础操作）
合并：递归合并结果
```

### 2. 合并两个有序链表的技巧
- 使用**哑节点(dummy)**简化边界处理
- 双指针比较，小的先接入
- 最后连接剩余部分

### 3. 优先队列的注意事项
- Python中需要处理相同值的情况（用索引作为第二关键字）
- Java中需要自定义Comparator
- 只将非空节点加入堆

### 4. 边界情况
- 输入为空数组 `[]`
- 数组中有空链表 `[[]]`
- 所有节点值相同
- 只有一个链表

## 💬 面试常问

**Q1: 为什么分治法比顺序合并快？**
> 顺序合并需要k-1轮，每轮都要处理越来越长的链表；分治法只需要log(k)轮，每轮的总工作量相同。

**Q2: 优先队列和分治法如何选择？**
> - k较小时两者差不多
> - k很大时，分治法空间更优（O(log k) vs O(k)）
> - 优先队列代码更简洁，适合面试快速实现

**Q3: 如果要求原地合并，不能用额外空间怎么办？**
> 使用分治法或顺序合并，空间复杂度可以做到O(1)。
