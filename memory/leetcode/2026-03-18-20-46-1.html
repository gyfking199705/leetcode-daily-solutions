🎯 LeetCode #1 - 两数之和

📋 题目描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出和为目标值 target 的那 两个 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且同一个元素在答案里不能重复出现。你可以按任意顺序返回答案。

示例 1

输入：nums = [2,7,11,15], target = 9

输出：[0,1]

解释：因为 nums[0] + nums[1] == 9，返回 [0, 1]

示例 2

输入：nums = [3,2,4], target = 6

输出：[1,2]

示例 3

输入：nums = [3,3], target = 6

输出：[0,1]

🎬 算法执行过程（点击步骤或代码行查看对应关系）

🚀 初始化

创建空哈希表，用于存储已遍历的数值及其下标

hashmap = {}

🔄 进入循环

用 enumerate 同时获取下标 i 和数值 num

i = 0, num = 2

💡 计算补数

目标值减去当前值，得到需要的「补数」

complement = target - num = 9 - 2 = 7

🔍 检查哈希表

补数 7 是否在哈希表中？不在，继续

7 in hashmap? → False

📝 存入哈希表

将当前值和下标存入哈希表，供后续查找

hashmap[2] = 0

🔄 第二轮循环

继续遍历，i = 1，num = 7

i = 1, num = 7

💡 再次计算补数

complement = 9 - 7 = 2

complement = 2

🎉 找到答案！

补数 2 在哈希表中！返回两个下标

2 in hashmap? → True, hashmap[2] = 0
return [0, 1]

Python 实现



1class Solution:


2    def twoSum(self, nums: List[int], target: int) -> List[int]:


3        hashmap = {}


4        for i, num in enumerate(nums):


5            complement = target - num


6            if complement in hashmap:


7                return [hashmap[complement], i]


8            hashmap[num] = i


9        return []  # 无解情况（按题意不会发生）

📊 复杂度分析

指标 暴力解法 哈希表解法（推荐）
时间复杂度 O(n²) - 双重循环 O(n) - 单次遍历
空间复杂度 O(1) - 无需额外空间 O(n) - 哈希表存储

💡 核心思路解析

关键洞察：对于每个数字，我们只需要知道「目标值减去当前值」的这个数，是否在前面出现过。

  • 暴力解法是固定一个数，然后遍历后面的所有数来找配对
  • 哈希表优化：边遍历边存，把「查找配对」从 O(n) 降到 O(1)
  • 这种「空间换时间」的策略是算法优化中最常见的技巧之一

为什么先检查再存入？避免同一个元素被使用两次（比如 [3,3] 这种情况，必须找到两个不同的下标）
