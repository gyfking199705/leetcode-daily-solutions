## 🎯 LeetCode 精讲 #4 - 寻找两个正序数组的中位数

---

### 📋 题目描述

**原题链接**: https://leetcode.cn/problems/median-of-two-sorted-arrays/

**难度**: 困难 ⭐⭐⭐  
**标签**: 数组、二分查找、分治

---

#### 题目原文

> 给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。
> 
> 请你找出并返回这两个正序数组的 **中位数**。
> 
> 算法的时间复杂度应该为 `O(log (m+n))`。

---

#### 示例

**示例 1：**
```
输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并数组 = [1,2,3]，中位数 2
```

**示例 2：**
```
输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并数组 = [1,2,3,4]，中位数 (2 + 3) / 2 = 2.5
```

---

#### 约束条件

- `nums1.length == m`
- `nums2.length == n`
- `0 <= m <= 1000`
- `0 <= n <= 1000`
- `1 <= m + n <= 2000`
- `-10^6 <= nums1[i], nums2[i] <= 10^6`

### 💡 解题思路

#### 🔍 核心洞察：在"两个有序数组"中找第k小

**中位数的本质**：
- 如果总元素个数为奇数 `2k+1`：中位数是第 `k+1` 小的元素
- 如果总元素个数为偶数 `2k`：中位数是第 `k` 小和第 `k+1` 小的平均值

**关键问题**：
> 如何在两个有序数组中找到第 `k` 小的元素？

**突破点**：
数组是有序的！如果我们比较 `nums1[k/2-1]` 和 `nums2[k/2-1]`，较小的那一部分元素一定不可能是第 `k` 小的，可以安全地排除。

**示例说明**：
```
nums1 = [1, 3, 5, 7, 9]    (长度5)
nums2 = [2, 4, 6, 8]       (长度4)

找第 4 小的元素：
- nums1[4/2-1] = nums1[1] = 3
- nums2[4/2-1] = nums2[1] = 4

比较：3 < 4
→ nums1的前2个元素 [1,3] 都不可能是第4小的
→ 排除 [1,3]，问题变成：
  nums1' = [5,7,9], nums2 = [2,4,6,8] 中找第 2 小的

继续递归...
```

---

#### 🎯 二分查找的具体实现

```
总长度 total = m + n

如果 total 是奇数：找第 (total//2 + 1) 小的元素
如果 total 是偶数：找第 (total//2) 小 和 第 (total//2 + 1) 小的平均值
```

**递归函数设计**：`findKth(nums1, start1, nums2, start2, k)`

```
1. 确保 nums1 从 start1 开始的可用长度 >= nums2 从 start2 开始的可用长度
   （保证 nums1 不会先被耗尽）

2. 如果 nums1 已耗尽，直接返回 nums2[start2 + k - 1]

3. 如果 k == 1，返回 min(nums1[start1], nums2[start2])

4. 取比较位置：
   i = min(start1 + k//2 - 1, len(nums1) - 1)  // nums1的比较点
   j = min(start2 + k//2 - 1, len(nums2) - 1)  // nums2的比较点

5. 比较 nums1[i] 和 nums2[j]：
   - 如果 nums1[i] <= nums2[j]：
     排除 nums1[start1...i]，递归 findKth(nums1, i+1, nums2, start2, k-(i-start1+1))
   - 否则：
     排除 nums2[start2...j]，递归 findKth(nums1, start1, nums2, j+1, k-(j-start2+1))
```

**为什么每次排除 k/2 个元素？**

因为 `nums1[k/2-1]` 和 `nums2[k/2-1]` 前面各至少有 `k/2-1` 个元素。
如果 `nums1[k/2-1] <= nums2[k/2-1]`，那么 `nums1[k/2-1]` 最多排在第 `k-1` 位，
所以 `nums1[0...k/2-1]` 这 `k/2` 个元素都不可能是第 `k` 小的。

---

#### 🎨 图解执行过程

```
nums1 = [1, 3, 5], nums2 = [2, 4]
找第 3 小的元素

初始状态：
nums1: [1, 3, 5]  从索引0开始
nums2: [2, 4]     从索引0开始
k = 3

第1轮：
i = min(0 + 3//2 - 1, 2) = min(0, 2) = 0  → nums1[0] = 1
j = min(0 + 3//2 - 1, 1) = min(0, 1) = 0  → nums2[0] = 2
比较：1 <= 2
→ 排除 nums1[0] = 1
→ 新问题：nums1=[3,5], nums2=[2,4] 中找第 2 小的

第2轮：
k = 2
i = min(0 + 2//2 - 1, 1) = min(0, 1) = 0  → nums1[0] = 3
j = min(0 + 2//2 - 1, 1) = min(0, 1) = 0  → nums2[0] = 2
比较：3 > 2
→ 排除 nums2[0] = 2
→ 新问题：nums1=[3,5], nums2=[4] 中找第 1 小的

第3轮：
k = 1
直接返回 min(3, 4) = 3

答案：第3小的元素是 3
```

### 📝 参考代码

```python
class Solution:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        """
        在两个有序数组中寻找中位数
        时间复杂度: O(log(m+n))
        空间复杂度: O(log(m+n)) - 递归栈深度
        """
        
        def findKth(arr1: list[int], start1: int, 
                    arr2: list[int], start2: int, k: int) -> int:
            """
            在 arr1[start1:] 和 arr2[start2:] 的并集中找第 k 小的元素
            k 是 1-based 的（即 k=1 表示找最小的）
            """
            # 确保 arr1 剩余长度 >= arr2 剩余长度
            # 这样如果有一个数组先耗尽，一定是 arr2
            if len(arr1) - start1 < len(arr2) - start2:
                return findKth(arr2, start2, arr1, start1, k)
            
            # 边界情况1: arr2 已经耗尽
            if start2 >= len(arr2):
                return arr1[start1 + k - 1]
            
            # 边界情况2: 找第1小的
            if k == 1:
                return min(arr1[start1], arr2[start2])
            
            # 取比较位置（不能越界）
            # 我们要比较的是两个数组中第 k//2 个候选元素
            i = min(start1 + k // 2 - 1, len(arr1) - 1)
            j = min(start2 + k // 2 - 1, len(arr2) - 1)
            
            if arr1[i] > arr2[j]:
                # arr2[j] 较小，排除 arr2[start2:j+1]
                # 这些元素个数 = j - start2 + 1
                # 新的问题：在剩余数组中找第 k - (j - start2 + 1) 小的
                return findKth(arr1, start1, arr2, j + 1, k - (j - start2 + 1))
            else:
                # arr1[i] 较小，排除 arr1[start1:i+1]
                return findKth(arr1, i + 1, arr2, start2, k - (i - start1 + 1))
        
        total = len(nums1) + len(nums2)
        
        if total % 2 == 1:
            # 奇数个：找第 (total//2 + 1) 个
            return float(findKth(nums1, 0, nums2, 0, total // 2 + 1))
        else:
            # 偶数个：找第 (total//2) 和第 (total//2 + 1) 个的平均值
            left = findKth(nums1, 0, nums2, 0, total // 2)
            right = findKth(nums1, 0, nums2, 0, total // 2 + 1)
            return (left + right) / 2.0


# ============== 测试代码 ==============

    print(f"输入: nums1={nums1}, nums2={nums2}")
    print(f"输出: {result:.5f}")  # 2.00000
    print()
    
    # 测试用例2

    print(f"输入: nums1={nums1}, nums2={nums2}")
    print(f"输出: {result:.5f}")  # 2.50000
    print()
    
    # 测试用例3：一个数组为空
    nums1, nums2 = [], [1]
    result = sol.findMedianSortedArrays(nums1, nums2)
    print(f"输入: nums1={nums1}, nums2={nums2}")
    print(f"输出: {result:.5f}")  # 1.00000
    print()
    
    # 测试用例4：包含负数
```

### ⏱️ 复杂度分析

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| **时间复杂度** | O(log(m+n)) | 每次递归排除 k/2 个元素，k 最大为 (m+n)/2 |
| **空间复杂度** | O(log(m+n)) | 递归栈深度，每次问题规模减半 |

**为什么是 O(log(m+n))？**

每次递归调用 `k` 至少减半（排除 k/2 个元素），所以递归深度为 O(log k) = O(log(m+n))。

### 🔑 关键要点

#### 1. 核心思想总结
```
要找第 k 小的元素？
  ↓
比较两个数组的第 k/2 个候选元素
  ↓
较小的那一侧可以安全排除 k/2 个元素
  ↓
问题规模减半，递归解决
```

#### 2. 边界条件处理
- ✅ 一个数组耗尽时，直接从另一个数组取
- ✅ k=1 时直接取最小值
- ✅ 数组长度不足 k/2 时，取到最后一个元素
- ✅ 确保较短的数组作为 arr2（避免先耗尽）

#### 3. 易错点
- ❌ 忘记处理数组长度不足 k/2 的情况
- ❌ k 的计算错误（注意是 1-based 还是 0-based）
- ❌ 排除元素后忘记更新 k 的值
- ❌ 偶数个元素时中位数计算错误

#### 4. 面试技巧
当被问到这道题时，可以这样展开：
1. **先讲暴力解法**：合并数组后排序，O((m+n)log(m+n))
2. **优化到 O(m+n)**：双指针逐个比较，类似归并排序的合并过程
3. **最终解法 O(log(m+n))**：二分查找第 k 小元素

### 🎓 算法变种：迭代写法

如果递归深度可能超出限制（虽然本题 m+n <= 2000 不会有问题），可以改写为迭代：

```python
class Solution:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        def findKth(arr1, arr2, k):
            # 始终保持 arr1 为较长的数组
            if len(arr1) < len(arr2):
                arr1, arr2 = arr2, arr1
            
            left, right = 0, len(arr2)
            
            # 二分查找分割线位置
            while left <= right:
                # partition2: arr2 的分割线位置
                partition2 = (left + right) // 2
                # partition1: arr1 的分割线位置
                partition1 = k - partition2
                
                # 处理边界
                nums2_left = float('-inf') if partition2 == 0 else arr2[partition2 - 1]
                nums2_right = float('inf') if partition2 == len(arr2) else arr2[partition2]
                nums1_left = float('-inf') if partition1 == 0 else arr1[partition1 - 1]
                nums1_right = float('inf') if partition1 == len(arr1) else arr1[partition1]
                
                # 检查分割是否正确
                if nums1_left <= nums2_right and nums2_left <= nums1_right:
                    # 找到正确分割
                    return max(nums1_left, nums2_left)
                elif nums1_left > nums2_right:
                    # arr1 的左边太大，需要左移 partition1，即右移 partition2
                    left = partition2 + 1
                else:
                    # arr2 的左边太大，需要左移 partition2
                    right = partition2 - 1
        
        total = len(nums1) + len(nums2)
        if total % 2 == 1:
            return float(findKth(nums1, nums2, total // 2 + 1))
        else:
            return (findKth(nums1, nums2, total // 2) + 
                    findKth(nums1, nums2, total // 2 + 1)) / 2.0
```

**迭代写法的核心思想**：
- 在两个数组中找一个"分割线"，使得左边元素个数 = k
- 分割线左边的所有元素 <= 分割线右边的所有元素
- 第 k 小的元素就是分割线左边最大的那个
