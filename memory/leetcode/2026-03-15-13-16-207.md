# 🎯 LeetCode 精讲 #207 - 课程表

---

### 📋 题目描述

**来源**: LeetCode 第207题  
**难度**: Medium  
**链接**: https://leetcode.cn/problems/course-schedule/

#### 题目原文
> 你这个学期必须选修 `numCourses` 门课程，记为 `0` 到 `numCourses - 1`。在选修某些课程之前需要一些先修课程。 先修课程按数组 `prerequisites` 给出，其中 `prerequisites[i] = [ai, bi]` ，表示如果要学习课程 `ai` 则 **必须** 先学习课程 `bi` 。
>
> 例如，先修课程对 `[0, 1]` 表示：想要学习课程 `0` ，你需要先完成课程 `1` 。
>
> 请你判断是否可能完成所有课程的学习？如果可以，返回 `true` ；否则，返回 `false` 。

#### 示例

**示例 1:**
```
输入: numCourses = 2, prerequisites = [[1,0]]
输出: true
解释: 总共有 2 门课程。学习课程 1 之前，你需要完成课程 0。这是可能的。
```

**示例 2:**
```
输入: numCourses = 2, prerequisites = [[1,0],[0,1]]
输出: false
解释: 总共有 2 门课程。学习课程 1 之前，你需要先完成课程 0；并且学习课程 0 之前，你还应先完成课程 1。这是不可能的（形成环）。
```

**示例 3:**
```
输入: numCourses = 4, prerequisites = [[1,0],[2,0],[3,1],[3,2]]
输出: true
解释: 一种可行的学习顺序是 [0,1,2,3] 或 [0,2,1,3]
```

#### 约束条件
- `1 <= numCourses <= 2000`
- `0 <= prerequisites.length <= 5000`
- `prerequisites[i].length == 2`
- `0 <= ai, bi < numCourses`
- `prerequisites[i]` 中的所有课程对 **互不相同**

---

### 💡 解题思路

这道题是**图论**中的经典问题，核心是判断**有向图是否存在环**。如果存在环，说明课程依赖关系互相矛盾，无法完成所有课程；如果不存在环，则存在合法的选课顺序。

这是一个典型的**拓扑排序**问题。

---

#### 方法一：拓扑排序 - BFS (Kahn算法) ⭐推荐

**核心思想**
> 不断移除入度为0的节点（即没有先修课程的课程），移除后将其后继课程的入度减1。如果最终能移除所有节点，则无环；否则有环。

**算法流程**

```
1. 构建邻接表：记录每个课程的后继课程
2. 计算入度数组：记录每个课程的先修课程数量
3. 将所有入度为0的课程加入队列
4. 循环处理队列：
   - 取出队首课程
   - 遍历其所有后继课程，入度减1
   - 如果后继课程入度变为0，加入队列
5. 统计处理的课程数，等于总课程数则无环
```

**图解示例**

以 `numCourses = 4, prerequisites = [[1,0],[2,0],[3,1],[3,2]]` 为例：

```
构建图：
0 ──→ 1 ──→ 3
└──→ 2 ──→┘

邻接表:      入度数组:
0: [1, 2]    0: 0 (无先修)
1: [3]       1: 1 (需先修0)
2: [3]       2: 1 (需先修0)  
3: []        3: 2 (需先修1和2)

处理过程:
Step 1: 课程0入度为0，开始学习 → 1,2入度变为0
Step 2: 课程1入度为0，开始学习 → 3入度变为1
Step 3: 课程2入度为0，开始学习 → 3入度变为0
Step 4: 课程3入度为0，开始学习 → 完成!

完成4门课程 ✓ 返回 true
```

**Python 代码实现**

```python
from collections import deque, defaultdict

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        # 1. 构建邻接表和入度数组
        graph = defaultdict(list)  # 邻接表：课程 -> 后继课程列表
        indegree = [0] * numCourses  # 入度数组
        
        for course, pre in prerequisites:
            graph[pre].append(course)  # pre -> course 的边
            indegree[course] += 1      # course 的入度+1
        
        # 2. 将所有入度为0的课程入队
        queue = deque()
        for i in range(numCourses):
            if indegree[i] == 0:
                queue.append(i)
        
        # 3. 拓扑排序
        studied = 0  # 已学习的课程数
        
        while queue:
            curr = queue.popleft()
            studied += 1
            
            # 遍历curr的所有后继课程
            for next_course in graph[curr]:
                indegree[next_course] -= 1  # 入度减1
                if indegree[next_course] == 0:
                    queue.append(next_course)  # 入度为0，可以学习
        
        # 4. 判断是否学完所有课程
        return studied == numCourses
```

**复杂度分析**
- **时间复杂度**: O(V + E)，其中 V 是课程数，E 是先修关系数。每个节点和边都被访问一次。
- **空间复杂度**: O(V + E)，邻接表和入度数组的空间。

#### 方法二：拓扑排序 - DFS (深度优先搜索)

**核心思想**
> 使用DFS遍历图，用三种状态标记节点：未访问、访问中、已访问。如果遇到"访问中"的节点，说明存在环。

**状态定义**
- `0` (未访问)：白色，节点尚未被访问
- `1` (访问中)：灰色，节点正在被DFS处理（在递归栈中）
- `2` (已访问)：黑色，节点及其所有后继都已处理完毕

**算法流程**

```
1. 构建邻接表
2. 初始化所有节点状态为"未访问"
3. 对每个未访问的节点执行DFS：
   - 标记当前节点为"访问中"
   - 递归访问所有后继节点
   - 如果后继节点是"访问中"状态，发现环，返回false
   - 标记当前节点为"已访问"
4. 所有DFS完成后无环，返回true
```

**图解示例**

```
形成环的情况: 0 → 1 → 0

DFS(0):
  标记0为"访问中"
  DFS(1):
    标记1为"访问中"
    DFS(0): 
      发现0已经是"访问中"状态！
      → 存在环，返回false
```

**Python 代码实现**

```python
from collections import defaultdict

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        # 构建邻接表
        graph = defaultdict(list)
        for course, pre in prerequisites:
            graph[pre].append(course)
        
        # 状态数组: 0=未访问, 1=访问中, 2=已访问
        state = [0] * numCourses
        
        def has_cycle(node: int) -> bool:
            """返回从node出发是否存在环"""
            if state[node] == 1:  # 访问中，发现环
                return True
            if state[node] == 2:  # 已访问，无需再处理
                return False
            
            # 标记为访问中
            state[node] = 1
            
            # 遍历所有后继节点
            for next_course in graph[node]:
                if has_cycle(next_course):
                    return True
            
            # 标记为已访问
            state[node] = 2
            return False
        
        # 对每个未访问的节点进行DFS
        for i in range(numCourses):
            if state[i] == 0:  # 只处理未访问的节点
                if has_cycle(i):
                    return False
        
        return True
```

**复杂度分析**
- **时间复杂度**: O(V + E)，每个节点和边都被访问一次。
- **空间复杂度**: O(V + E)，邻接表、状态数组和递归栈的空间。

---

### 📊 两种方法对比

| 特性 | BFS (Kahn) | DFS |
|------|-----------|-----|
| **核心** | 从入度为0的节点开始剥离 | 检测递归栈中的重复访问 |
| **空间** | O(V + E)，需要队列 | O(V + E)，需要递归栈 |
| **代码长度** | 稍长 | 稍短 |
| **直观性** | 更直观（模拟选课过程） | 需要理解三色标记 |
| **扩展性** | 容易输出拓扑序（学习顺序） | 也可以输出拓扑序（逆序） |
| **适用场景** | 推荐，尤其适合求具体顺序 | 代码简洁，适合快速实现 |

### 🎯 关键要点总结

1. **问题转化**：课程依赖 → 有向图，判断能否完成 → 判断有向图是否有环

2. **拓扑排序本质**：将有向无环图(DAG)的节点排成线性序列，保证所有边的起点在终点之前

3. **入度为0的节点**：没有前置依赖，可以立即执行（选课）

4. **环的检测**：
   - BFS：如果最终还有节点未处理，说明有环
   - DFS：如果在DFS过程中遇到"访问中"的节点，说明有环

5. **面试高频考点**：
   - 能讲清楚两种方法的原理和区别
   - 能快速写出BFS版本（Kahn算法）
   - 理解如何输出具体的拓扑排序序列（下一题：课程表 II）

### 💬 常见问题

**Q1: 为什么用队列而不用栈？**
> 队列保证先入先出，确保入度为0的节点按顺序处理。实际上用栈也可以，只是结果顺序不同。

**Q2: 如何输出具体的选课顺序？**
> 每次从队列取出课程时，将其加入结果列表。最终列表就是拓扑排序序列（可能有多种合法顺序）。

**Q3: 如果有多个入度为0的课程，选哪个有影响吗？**
> 没有影响，任选其一即可。这解释了为什么拓扑排序的结果不唯一。

**Q4: 空间复杂度中的 E 具体指什么？**
> E 是 prerequisites 的长度，即图中的边数。邻接表需要存储这些边。
