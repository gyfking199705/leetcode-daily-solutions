33. 搜索旋转排序数组

Medium 二分查找 数组 旋转数组
2026-04-26 第237期 | 预计阅读 6 分钟

题目描述

整数数组 nums 按升序排列，数组中的值 互不相同。

在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。

例如，[0,1,2,4,5,6,7] 在下标 3 处经旋转后可能变为 [4,5,6,7,0,1,2]。

给你 旋转后 的数组 nums 和一个整数 target，如果 nums 中存在这个目标值 target，则返回它的下标，否则返回 -1。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

输入: nums = [4,5,6,7,0,1,2], target = 0
输出: 4

输入: nums = [4,5,6,7,0,1,2], target = 3
输出: -1

输入: nums = [1], target = 0
输出: -1

核心思路

旋转数组性质

旋转后数组被分成两段升序区间。以旋转点为界，左半段和右半段各自保持升序。

二分变形

常规二分要求全局有序，这里局部有序。关键：判断哪一半是有序的，再判断 target 是否在其中。

有序段判定

通过比较 nums[mid] 和 nums[right]（或 nums[left]）即可确定哪一半是升序的。

O(log n) 强制要求

题目硬性要求对数时间，排除线性扫描。二分是唯一选择，必须处理旋转带来的无序。

解题步骤

1

初始化左右指针

left = 0, right = nums.length - 1，标准二分框架。

2

循环条件与中间值

while (left <= right)，取 mid = left + (right - left) / 2。如果 nums[mid] == target，直接返回 mid。

3

判断哪一半有序

比较 nums[mid] 和 nums[right]：
• 若 nums[mid] < nums[right]，则右半段 [mid, right] 是升序的
• 若 nums[mid] > nums[right]，则左半段 [left, mid] 是升序的
（由于元素互不相同，不存在相等的情况）

4

在有序段中判断 target 位置

若右半段有序：判断 target 是否在 (nums[mid], nums[right]] 范围内，是则 left = mid + 1，否则 right = mid - 1。
若左半段有序：判断 target 是否在 [nums[left], nums[mid]) 范围内，是则 right = mid - 1，否则 left = mid + 1。

5

循环结束未找到

返回 -1。

代码实现

class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        
        while left <= right:
            mid = left + (right - left) // 2
            
            # 直接命中
            if nums[mid] == target:
                return mid
            
            # 判断哪一半是有序的
            if nums[mid] < nums[right]:
                # 右半段 [mid, right] 有序
                if nums[mid] < target <= nums[right]:
                    # target 在右半段，收缩左边界
                    left = mid + 1
                else:
                    # target 不在右半段，去左半段找
                    right = mid - 1
            else:
                # 左半段 [left, mid] 有序（题目保证元素不同，不会出现等于）
                if nums[left] <= target < nums[mid]:
                    # target 在左半段，收缩右边界
                    right = mid - 1
                else:
                    # target 不在左半段，去右半段找
                    left = mid + 1
        
        return -1
class Solution {
    public int search(int[] nums, int target) {
        int left = 0, right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) return mid;
            
            if (nums[mid] < nums[right]) {
                // 右半段有序
                if (nums[mid] < target && target <= nums[right]) {
                    left = mid + 1;
                } else {
                    right = mid - 1;
                }
            } else {
                // 左半段有序
                if (nums[left] <= target && target < nums[mid]) {
                    right = mid - 1;
                } else {
                    left = mid + 1;
                }
            }
        }
        return -1;
    }
}

复杂度分析

时间复杂度

O(log n)

标准的二分查找，每次排除一半元素

空间复杂度

O(1)

仅使用常数个指针变量

关键洞察与易错点

常见错误

  • 使用 nums[mid] 和 nums[left] 比较判断有序段也可以，但边界处理容易出错。推荐和 nums[right] 比较，直觉更顺。
  • 边界条件写成 nums[mid] <= target <= nums[right] 时漏掉等号，导致遗漏 target 恰好等于边界值的情况。
  • 旋转数组可能存在完全未旋转的情况（k=0），即原数组本身完全有序，算法必须兼容。
  • 数组只有一个元素时，要正确处理。

面试追问

  • 如果数组中存在重复元素？ → 退化到 O(n)，最坏情况无法判断哪半有序（如 [1,0,1,1,1]）。
  • 如何找到旋转点（最小元素）？ → 同样是二分变形，比较 nums[mid] 和 nums[right]，找最小值下标。
  • 旋转数组的其他变种？ → 81题（含重复元素）、153题（找最小值）、154题（含重复找最小值）。
