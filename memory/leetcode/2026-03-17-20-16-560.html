🎯 LeetCode #560 - 和为 K 的子数组

前缀和 + 哈希表的经典组合｜子数组计数问题

📋 题目描述

给你一个整数数组 nums 和一个整数 k，请你统计并返回 该数组中和为 k 的子数组的个数。

子数组是数组中元素的连续非空序列。

示例 1： 
输入: nums = [1,1,1], k = 2
输出: 2
解释: 有两个和为 2 的子数组: [1,1]（下标 0-1）和 [1,1]（下标 1-2）
示例 2： 
输入: nums = [1,2,3], k = 3
输出: 2
解释: 有两个和为 3 的子数组: [1,2] 和 [3]

Medium 前缀和 哈希表 数组

💡 核心思路

这道题的暴力解法是枚举所有子数组，时间复杂度 O(n²)，会超时。关键是发现以下数学性质：

前缀和定义： prefix[i] 表示 nums[0..i-1] 的和（即前 i 个元素的和）
子数组和公式： sum(i,j) = prefix[j+1] - prefix[i]

我们要找满足 sum(i,j) = k 的子数组，即：

prefix[j+1] - prefix[i] = k → prefix[i] = prefix[j+1] - k

这意味着：当我们遍历到位置 j 时，如果前面出现过 prefix[j+1]-k 这个前缀和，那么从那个位置到 j 就形成了一个符合条件的子数组！

因此，我们可以用一个哈希表记录每个前缀和出现的次数，遍历数组时：

  1. 计算当前前缀和 prefix
  2. 查询哈希表中 prefix - k 出现的次数，累加到答案
  3. 将当前 prefix 加入哈希表

🎓 关键点总结

    
    
      
  • 前缀和思想：将子数组和问题转化为两个前缀和的差
    • 
      
  • 哈希表优化：用空间换时间，将查找 O(n) 降为 O(1)
    • 
      
  • 初始化技巧：prefix[0] = 0 出现 1 次，处理从数组开头开始的子数组
    • 
      
  • 计数而非记录：哈希表存的是出现次数，不是下标
    • 
    

    
    

      易错点：一定要先查询 prefix-k 的计数，再更新当前 prefix 的计数！顺序颠倒会导致重复计数。
