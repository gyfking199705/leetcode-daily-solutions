# 🎯 LeetCode 精讲 #56 - 合并区间

---

## 📋 题目描述

**来源**: LeetCode 第56题  
**难度**: Medium  
**链接**: https://leetcode.cn/problems/merge-intervals/

### 题目原文
> 以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [starti, endi]` 。请你合并所有重叠的区间，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。

### 示例

**示例 1:**
```
输入: intervals = [[1,3],[2,6],[8,10],[15,18]]
输出: [[1,6],[8,10],[15,18]]
解释: 区间 [1,3] 和 [2,6] 重叠, 将它们合并为 [1,6]。
```

**示例 2:**
```
输入: intervals = [[1,4],[4,5]]
输出: [[1,5]]
解释: 区间 [1,4] 和 [4,5] 可被视为重叠区间。
```

### 约束条件
- `1 <= intervals.length <= 10^4`
- `intervals[i].length == 2`
- `0 <= starti <= endi <= 10^4`

---

## 💡 解题思路

### 核心思想：排序 + 贪心

合并区间的关键是：**让可能重叠的区间排在一起**，这样就能顺序处理、逐个合并。

#### 为什么先排序？

如果不排序，区间是乱序的，比如：`[[1,3], [8,10], [2,6], [15,18]]`
- 处理 `[1,3]` 时，不知道后面还有 `[2,6]` 和它重叠
- 必须保存所有信息回头再处理，复杂度高

**按区间起点排序后**：`[[1,3], [2,6], [8,10], [15,18]]`
- 起点小的在前，如果两个区间有重叠，它们一定相邻
- 一次遍历即可完成合并！

#### 贪心策略

维护一个「当前合并区间」`[cur_start, cur_end]`：
1. 如果新区间的起点 `<= cur_end`，说明重叠，扩展 `cur_end = max(cur_end, 新区间.end)`
2. 如果新区间的起点 `> cur_end`，说明不重叠，保存当前区间，开启新区间

### 图解过程

```
原始区间: [[1,3], [2,6], [8,10], [15,18]]
          ↓ 按起点排序
排序后:   [[1,3], [2,6], [8,10], [15,18]]

遍历过程:
  初始: cur = [1,3]
  
  i=1: [2,6] 起点2 <= 3，重叠！
       cur = [1, max(3,6)] = [1,6]
  
  i=2: [8,10] 起点8 > 6，不重叠
       保存 [1,6]，cur = [8,10]
  
  i=3: [15,18] 起点15 > 10，不重叠
       保存 [8,10]，cur = [15,18]
  
  结束: 保存 [15,18]

结果: [[1,6], [8,10], [15,18]]
```

## 🔧 完整代码

```python
from typing import List

class Solution:
    """
    LeetCode 56: 合并区间
    思路: 按起点排序 + 贪心合并
    """
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        if not intervals or len(intervals) <= 1:
            return intervals
        
        # Step 1: 按区间起点升序排序
        intervals.sort(key=lambda x: x[0])
        
        # Step 2: 贪心合并
        merged = []
        cur_start, cur_end = intervals[0]
        
        for i in range(1, len(intervals)):
            start, end = intervals[i]
            
            if start <= cur_end:
                # 重叠，扩展当前区间
                cur_end = max(cur_end, end)
            else:
                # 不重叠，保存当前区间，开启新区间
                merged.append([cur_start, cur_end])
                cur_start, cur_end = start, end
        
        # 别忘了最后一个区间
        merged.append([cur_start, cur_end])
        
        return merged


# ============ 测试代码 ============
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例1: 基本重叠
    test1 = [[1,3],[2,6],[8,10],[15,18]]
    print(f"输入: {test1}")
    print(f"输出: {sol.merge(test1)}")  # [[1,6],[8,10],[15,18]]
    
    # 测试用例2: 边界重叠
    test2 = [[1,4],[4,5]]
    print(f"\n输入: {test2}")
    print(f"输出: {sol.merge(test2)}")  # [[1,5]]
    
    # 测试用例3: 完全包含
    test3 = [[1,4],[2,3]]
    print(f"\n输入: {test3}")
    print(f"输出: {sol.merge(test3)}")  # [[1,4]]
    
    # 测试用例4: 多个连续重叠
    test4 = [[1,4],[0,2],[3,5]]
    print(f"\n输入: {test4}")
    print(f"输出: {sol.merge(test4)}")  # [[0,5]]
    
    # 测试用例5: 无重叠
    test5 = [[1,2],[3,4],[5,6]]
    print(f"\n输入: {test5}")
    print(f"输出: {sol.merge(test5)}")  # [[1,2],[3,4],[5,6]]
    
    # 测试用例6: 单区间
    test6 = [[1,1]]
    print(f"\n输入: {test6}")
    print(f"输出: {sol.merge(test6)}")  # [[1,1]]
```

## 🎯 关键要点总结

### 1. 为什么排序后不会漏掉重叠区间？

**证明**：假设排序后两个重叠区间 `A` 和 `B` 不相邻
- 设 `A=[a1,a2], B=[b1,b2]`，重叠意味着 `a1 <= b1 <= a2` 或 `b1 <= a1 <= b2`
- 排序后 `a1 <= b1`，所以只可能是 `a1 <= b1 <= a2`
- 如果中间有区间 `C=[c1,c2]`，则 `a1 <= c1 <= b1 <= a2`
- 这意味着 `C` 也与 `A` 重叠！合并 `A` 和 `C` 后得到的区间包含 `A∪C∪B`
- 因此相邻检查不会漏掉任何重叠

### 2. 区间重叠的条件

两个区间 `[a,b]` 和 `[c,d]`（假设 `a <= c`，即已排序）：
- **重叠**：`c <= b`（起点在上一区间内）
- **不重叠**：`c > b`（起点在上一区间外）

### 3. 贪心正确性

每一步都尽可能扩展当前区间，这种局部最优导致全局最优：
- 对于重叠区间，合并成一个最大的区间是唯一的合理选择
- 对于不重叠区间，必须分开保存

## 💡 代码技巧

**Python 排序技巧**：
```python
# 按起点排序

from operator import itemgetter
intervals.sort(key=itemgetter(0))
```

- 空数组、单元素数组
