## 一、题目描述

### 题目原文
> 给定一个非负整数数组 `nums`，你最初位于数组的第一个下标。
> 
> 数组中的每个元素代表你在该位置可以跳跃的最大长度。
> 
> 判断你是否能够到达最后一个下标。

### 示例

**示例 1:**
```
输入: nums = [2,3,1,1,4]
输出: true
解释: 可以先跳 1 步，从下标 0 到达下标 1，然后再从下标 1 跳 3 步到达最后一个下标。
```

**示例 2:**
```
输入: nums = [3,2,1,0,4]
输出: false
解释: 无论从哪开始，总会到达下标为 3 的位置，但该位置的跳跃长度是 0，无法到达最后一个下标。
```

### 约束条件
- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 10^5`

## 二、解题思路

### 2.1 问题分析

这道题的关键在于：我们**不需要**知道具体怎么跳，只需要判断**是否能到达终点**。

核心观察点：
- 每个位置的值表示从该位置**最远能跳到哪里**
- 如果能到达位置 `i`，那么从位置 `i` 能到达的所有位置都能到达
- 问题的本质是：判断是否存在一条"可达性"链从起点延伸到终点

### 2.2 贪心思想

与其考虑"每一步跳多远"，不如换个角度：

> 维护一个变量 `maxReach`，表示**当前能到达的最远位置**。

遍历数组时：
1. 如果当前位置 `i` 超过了 `maxReach`，说明这个位置无法到达，直接返回 `false`
2. 更新 `maxReach = max(maxReach, i + nums[i])`，表示从当前位置能跳得更远
3. 如果 `maxReach >= n-1`，说明可以到达终点，提前返回 `true`

### 2.3 为什么贪心有效？

这是典型的"区间覆盖"问题：
- 从位置 `i` 可以到达的区间是 `[i, i + nums[i]]`
- 如果当前能到达的区间能够覆盖到下一个位置，就可以"延伸"这个可达区间
- 贪心选择：每次选择能到达的最远位置，因为越远越好，给后续留下更多选择空间

## 三、代码实现

### 3.1 Python 实现

```python
from typing import List

class Solution:
    def canJump(self, nums: List[int]) -> bool:
        """
        贪心算法：维护最远可到达位置
        
        思路：
        1. 遍历数组，维护 maxReach 表示当前能到达的最远下标
        2. 如果当前下标 i > maxReach，说明无法到达 i，返回 False
        3. 更新 maxReach = max(maxReach, i + nums[i])
        4. 如果 maxReach >= n-1，说明可以到达终点
        
        时间复杂度: O(n)
        空间复杂度: O(1)
        """
        n = len(nums)
        maxReach = 0  # 当前能到达的最远位置
        
        for i in range(n):
            # 如果当前位置已经无法到达，返回 False
            if i > maxReach:
                return False
            
            # 更新最远可达位置
            maxReach = max(maxReach, i + nums[i])
            
            # 如果已经可以到达终点，提前返回
            if maxReach >= n - 1:
                return True
        
        return True


# 测试代码
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例 1
    nums1 = [2, 3, 1, 1, 4]
    print(f"输入: {nums1}")
    print(f"输出: {sol.canJump(nums1)}")  # true
    print()
    
    # 测试用例 2
    nums2 = [3, 2, 1, 0, 4]
    print(f"输入: {nums2}")
    print(f"输出: {sol.canJump(nums2)}")  # false
    print()
    
    # 测试用例 3: 边界情况
    nums3 = [0]
    print(f"输入: {nums3}")
    print(f"输出: {sol.canJump(nums3)}")  # true（已经在终点）
    print()
    
    # 测试用例 4: 大数情况
    nums4 = [5, 0, 0, 0, 0, 0]
    print(f"输入: {nums4}")
    print(f"输出: {sol.canJump(nums4)}")  # true（一步直达）
```

### 3.2 执行过程可视化

以 `nums = [2,3,1,1,4]` 为例：

```
初始状态: maxReach = 0

i=0: nums[0]=2
     - i=0 <= maxReach=0 ✓（可达）
     - maxReach = max(0, 0+2) = 2
     - 当前可到达区间: [0, 2]

i=1: nums[1]=3
     - i=1 <= maxReach=2 ✓（可达）
     - maxReach = max(2, 1+3) = 4
     - maxReach=4 >= n-1=4 ✓，返回 true！
```

以 `nums = [3,2,1,0,4]` 为例：

```
初始状态: maxReach = 0

i=0: nums[0]=3
     - maxReach = max(0, 0+3) = 3
     - 可到达区间: [0, 3]

i=1: nums[1]=2
     - i=1 <= 3 ✓
     - maxReach = max(3, 1+2) = 3

i=2: nums[2]=1
     - i=2 <= 3 ✓
     - maxReach = max(3, 2+1) = 3

i=3: nums[3]=0
     - i=3 <= 3 ✓
     - maxReach = max(3, 3+0) = 3

i=4: nums[4]=4
     - i=4 > maxReach=3 ✗（不可达！）
     - 返回 false
```

## 四、复杂度分析

### 时间复杂度
- **O(n)**：只需遍历数组一次
- 最佳情况下可能提前退出（maxReach 提前到达终点）

### 空间复杂度
- **O(1)**：只使用了一个额外变量 `maxReach`
- 不需要额外的数组或数据结构

## 五、关键点总结

### 5.1 易错点

1. **混淆"跳跃长度"和"目标位置"**
   - ❌ 错误理解：从位置 i 跳到位置 i+1
   - ✅ 正确理解：从位置 i 可以跳到任意位置 j，其中 `i < j <= i + nums[i]`

2. **边界条件**
   - 数组长度为 1 时，已经在终点，返回 `true`
   - 第一个元素为 0 且长度大于 1 时，无法移动，返回 `false`

3. **判断顺序**
   - 先判断 `i > maxReach`，再更新 `maxReach`
   - 顺序不能颠倒！

### 5.2 贪心选择的正确性证明

**定理**：如果位置 `i` 是可达的，那么所有 `j < i` 的位置都可达。

**证明**：
- 位置 `i` 可达意味着存在某个位置 `k < i`，使得 `k + nums[k] >= i`
- 由于 `k < i` 且 `k` 可达，同理可证所有中间位置都可达
- 由数学归纳法，从起点到 `i` 形成一条可达链

**贪心最优性**：
- 我们总是维护最远可达位置，因为只要能到达更远的地方，就有更多选择
- 不存在"故意少走"反而能到达终点的情况（如果能到远的，一定能到近的）

### 6.1 进阶题：跳跃游戏 II (LeetCode 45)

```python
def jump(self, nums: List[int]) -> int:
    """求到达终点的最小跳跃次数"""
    n = len(nums)
    if n <= 1:
        return 0
    
    jumps = 0       # 跳跃次数
    current_end = 0 # 当前这跳能到达的最远位置
    max_reach = 0   # 下一跳能到达的最远位置
    
    for i in range(n - 1):  # 不需要访问最后一个元素
        max_reach = max(max_reach, i + nums[i])
        
        if i == current_end:  # 需要再跳一次
            jumps += 1
            current_end = max_reach
            
            if current_end >= n - 1:
                break
    
    return jumps
```

## 七、面试技巧

### 7.1 面试常见问题

**Q: 这道题能用DP做吗？**
> A: 可以，但没必要。DP解法 `dp[i]` 表示能否到达位置 `i`，时间复杂度 O(n²)，空间复杂度 O(n)。贪心 O(n) 更优。

**Q: 如果需要输出路径呢？**
> A: 需要记录每个位置的"前驱"，可以用额外的数组存储是从哪个位置跳过来的，最后用回溯得到完整路径。

**Q: 如果数组很大（10^9级别），无法一次加载到内存怎么办？**
> A: 使用流式处理，只维护 `maxReach` 变量，不需要存储整个数组（除非需要回溯路径）。

### 7.2 代码书写技巧

1. **提前退出优化**：当 `maxReach >= n-1` 时立即返回，避免不必要的遍历
2. **边界处理**：先处理 `n <= 1` 的情况
3. **变量命名**：`maxReach` / `maxPos` / `farthest` 都可以，但要表意清晰

跳跃游戏是贪心算法的经典入门题，核心思想是：

维护最远可达位置，问题就迎刃而解。这道题考察的是对"可达性"的理解和贪心策略的应用，是面试中的高频考点。
