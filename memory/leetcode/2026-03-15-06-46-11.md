## 📋 题目描述

给定一个长度为 `n` 的整数数组 `height` 。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])` 。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

**说明**：你不能倾斜容器。

## 💡 解题思路

### 暴力解法（不推荐）

**思路**: 枚举所有可能的两个柱子组合，计算每对柱子能盛的水量，取最大值。

**时间复杂度**: O(n²) — 需要枚举 n*(n-1)/2 对柱子  
**空间复杂度**: O(1)

```python
def maxArea_brute_force(height):
    n = len(height)
    max_area = 0
    for i in range(n):
        for j in range(i + 1, n):
            # 面积 = 宽度 * 高度
            # 宽度 = j - i
            # 高度 = min(height[i], height[j]) — 受限于较短的柱子
            area = (j - i) * min(height[i], height[j])
            max_area = max(max_area, area)
    return max_area
```

---

### 最优解法：双指针

**核心思想**: 

容器的盛水量由两个因素决定：
1. **宽度**: 两根柱子之间的距离 `(right - left)`
2. **高度**: 两根柱子中较短的那根 `min(height[left], height[right])`

**面积公式**: `area = (right - left) * min(height[left], height[right])`

**关键观察** 🔑:

假设我们有两根柱子，`left` 在左，`right` 在右：
- 如果 `height[left] < height[right]`，那么 `left` 是当前容器的短板
- **此时移动 `right` 向内侧移动只会使宽度减小，而高度仍受限于 `height[left]`**，所以面积必然减小
- **因此，我们只能移动 `left` 指针，希望找到一个更高的柱子**

**算法步骤**:
1. 初始化两个指针：`left = 0`，`right = n - 1`
2. 计算当前面积并更新最大值
3. **移动高度较小的那根指针**向内侧移动
4. 重复步骤2-3直到 `left >= right`

**为什么移动短板是正确的？**

```
假设: height[left] = 2, height[right] = 8
当前面积 = (right - left) * 2

情况1: 移动 left → left+1
     - 新高度可能 > 2，面积可能增大
     
情况2: 移动 right → right-1  
     - 高度仍受限于 height[left] = 2
     - 宽度减小了
     - 面积必然减小！
```

## 📝 代码实现

### Python 实现

```python
from typing import List

class Solution:
    def maxArea(self, height: List[int]) -> int:
        """
        双指针法求解最大盛水容器
        
        时间复杂度: O(n) — 只需要遍历一次
        空间复杂度: O(1) — 只使用了常数额外空间
        """
        left, right = 0, len(height) - 1
        max_area = 0
        
        while left < right:
            # 计算当前容器的面积
            # 高度由较短的柱子决定
            current_height = min(height[left], height[right])
            current_width = right - left
            current_area = current_height * current_width
            
            # 更新最大面积
            max_area = max(max_area, current_area)
            
            # 关键决策：移动高度较小的指针
            # 因为移动短板才有可能找到更高的柱子，从而增大面积
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        
        return max_area
```

### Java 实现

```java
class Solution {
    public int maxArea(int[] height) {
        int left = 0;
        int right = height.length - 1;
        int maxArea = 0;
        
        while (left < right) {
            // 计算当前面积
            int currentHeight = Math.min(height[left], height[right]);
            int width = right - left;
            int currentArea = currentHeight * width;
            
            // 更新最大值
            maxArea = Math.max(maxArea, currentArea);
            
            // 移动高度较小的指针
            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }
        
        return maxArea;
    }
}
```

## 🎯 关键要点总结

### 1. 为什么双指针有效？

**核心原理**: 每次排除掉"不可能成为最优解"的组合

- 当我们决定移动短板时，实际上是在说：
  - "以当前左指针为边的所有容器，我们已经检查过了最大可能的那个"
  - "固定左指针、移动右指针向内，只会使面积越来越小"
  - "所以我们完全可以放弃左指针，寻找下一个可能的左边界"

### 2. 贪心策略的正确性证明

**贪心选择性质**: 每一步选择移动短板，都是安全的

**反证法**: 
假设最优解使用了 `(left, right)` 这对柱子，且 `height[left] < height[right]`。
如果我们没有检查这个组合，那一定是在某一步移动了 `left`。
而移动 `left` 的唯一原因是 `height[left]` 是当时左端点的所有可能中最大的（否则我们不会放弃它）。
矛盾！所以双指针一定能找到最优解。

### 3. 边界情况

```python
# 情况1: 只有两根柱子
height = [1, 1]  # 结果: 1

# 情况2: 单调递增
height = [1, 2, 3, 4, 5]  # 结果: 6 (1和5: 1*4=4, 2和5: 2*3=6)

# 情况3: 单调递减  
height = [5, 4, 3, 2, 1]  # 结果: 6 (5和2: 2*3=6, 5和1: 1*4=4)

# 情况4: 大量重复
height = [10000] * 100000  # 结果: 99990000
```

### 4. 易错点

❌ **错误**: 总是移动左指针  
✅ **正确**: 移动高度较小的指针

❌ **错误**: 计算面积时用 `max(height[left], height[right])`  
✅ **正确**: 用 `min`，因为水会从短板处溢出

## 🧠 面试高频考点

### Q1: 为什么时间复杂度是 O(n)？
> 双指针从数组两端出发，每次循环移动一个指针，直到相遇。指针最多移动 n 步，所以是 O(n)。

### Q2: 如果移动长板会怎样？
> 移动长板会导致宽度减小，而高度仍受限于短板（没有变化），所以面积必然减小。这会遗漏可能的最优解。

### Q3: 这种贪心策略在什么情况下会失效？
> 如果问题变成"求最小面积"，贪心策略就失效了，因为最小面积可能出现在任意位置。

### Q4: 空间复杂度能否优化？
> 已经是 O(1)，无法再优化。

## 📌 一句话总结

> **双指针从两端向中间夹逼，每次移动高度较小的指针，确保不遗漏任何可能的最大面积。**

这是一道考察"贪心+双指针"的经典题目，理解"为什么移动短板"比记住代码更重要。
