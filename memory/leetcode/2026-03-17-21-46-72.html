LeetCode #72 - 编辑距离

Hard 动态规划 · 字符串 · 经典面试题

📋 题目描述

给你两个字符串 word1 和 word2，请返回将 word1 转换成 word2 所使用的最少操作数。

你可以对一个单词进行如下三种操作：

  • 插入一个字符（Insert）
  • 删除一个字符（Delete）
  • 替换一个字符（Replace）
输入：word1 = "horse", word2 = "ros"
输出：3
解释：
horse → rorse (将 'h' 替换为 'r')
rorse → rose (删除 'r')
rose → ros (删除 'e')
输入：word1 = "intention", word2 = "execution"
输出：5
💡 核心思想
这题是编辑距离（Edit Distance），也称为莱文斯坦距离（Levenshtein Distance）。
状态定义：dp[i][j] 表示 word1[0:i] 转换到 word2[0:j] 所需的最小操作数。

⏱️ 复杂度分析

维度 值 说明
时间复杂度 O(m × n) m 和 n 分别是两个字符串的长度，需要填充 m×n 的 DP 表格
空间复杂度 O(m × n) 二维 DP 数组，可优化至 O(min(m,n)) 使用滚动数组

🔄 空间优化版本

💡 优化原理
每次只依赖上一行和当前行，可用两行数组代替完整矩阵，空间降至 O(n)。
