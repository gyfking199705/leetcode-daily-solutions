# LeetCode 25：K 个一组翻转链表

## 📌 题目描述

**难度**: Hard  
**链接**: https://leetcode.cn/problems/reverse-nodes-in-k-group/

### 题目要求
给你链表的头节点 `head` ，每 `k` 个节点一组进行翻转，请你返回修改后的链表。

`k` 是一个正整数，它的值小于或等于链表的长度。

如果节点总数不是 `k` 的整数倍，那么请将最后剩余的节点保持原有顺序。

你不能只是单纯的改变节点内部的值，而是需要实际进行节点交换。

### 示例

**示例 1：**
```
输入：head = [1,2,3,4,5], k = 2
输出：[2,1,4,3,5]
```

**示例 2：**
```
输入：head = [1,2,3,4,5], k = 3
输出：[3,2,1,4,5]
```

### 约束条件
- 链表中的节点数目为 `n`
- `1 <= k <= n <= 5000`
- `0 <= Node.val <= 1000`

---

## 💡 核心思路

这道题的本质是 **"分块处理 + 局部翻转 + 链表拼接"**

### 解题关键点
1. **分组定位**：每 k 个节点为一组，需要找到每组的起始和结束位置
2. **边界判断**：如果剩余节点不足 k 个，则保持原序不翻转
3. **链表反转**：实现一个局部反转函数，只反转指定范围内的节点
4. **指针拼接**：反转后需要正确连接前一组、当前组、后一组

### 两种主流解法

| 方法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|-----------|-----------|------|
| **迭代法** | O(n) | O(1) | 满足进阶要求，工程首选 |
| **递归法** | O(n) | O(n/k) | 代码简洁，易于理解 |

## 🔧 解法一：迭代法（推荐）

### 思路详解
使用虚拟头节点（dummy node）简化边界处理，通过迭代遍历链表：
1. 检查剩余节点是否足够 k 个
2. 保存当前组的前驱和后继节点
3. 反转当前组的 k 个节点
4. 重新连接三组指针
5. 移动到下一组继续处理

### 代码实现 (Python)

```python
# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def reverseKGroup(self, head: ListNode, k: int) -> ListNode:
        # 边界处理
        if not head or k == 1:
            return head
        
        # 创建虚拟头节点，简化头节点翻转的边界处理
        dummy = ListNode(0)
        dummy.next = head
        
        # prev_group_end: 当前组的前一个节点（翻转后要连接新头）
        prev_group_end = dummy
        
        while True:
            # 检查剩余节点是否足够 k 个
            kth_node = prev_group_end
            for _ in range(k):
                kth_node = kth_node.next
                if not kth_node:
                    # 不足 k 个，直接返回结果
                    return dummy.next
            
            # 记录下一组的起始节点
            next_group_start = kth_node.next
            
            # 当前组的起始节点（翻转后变成尾节点）
            group_start = prev_group_end.next
            
            # 断开当前组与后续节点的连接，便于反转
            kth_node.next = None
            
            # 反转当前组，返回新的头节点
            new_group_head = self.reverseList(group_start)
            
            # 重新连接三组：
            # 1. 前一组的尾 -> 当前组的新头
            prev_group_end.next = new_group_head
            # 2. 当前组的尾（原头）-> 下一组的头
            group_start.next = next_group_start
            
            # 移动到下一组的前驱位置
            prev_group_end = group_start
    
    def reverseList(self, head: ListNode) -> ListNode:
        """反转链表，返回新的头节点"""
        prev = None
        curr = head
        while curr:
            next_temp = curr.next
            curr.next = prev
            prev = curr
            curr = next_temp
        return prev
```

## 🔧 解法二：递归法

### 思路详解
利用递归的分治思想：
1. 先处理当前 k 个节点的翻转
2. 递归处理剩余的链表
3. 将两部分连接起来

### 代码实现 (Python)

```python
class Solution:
    def reverseKGroup(self, head: ListNode, k: int) -> ListNode:
        # 边界条件
        if not head or k == 1:
            return head
        
        # 步骤1：检查当前组是否有 k 个节点
        curr = head
        count = 0
        while curr and count < k:
            curr = curr.next
            count += 1
        
        # 不足 k 个，直接返回原头（保持原序）
        if count < k:
            return head
        
        # 步骤2：翻转当前组的 k 个节点
        prev = None
        cur = head
        count = 0
        while cur and count < k:
            temp = cur.next
            cur.next = prev
            prev = cur
            cur = temp
            count += 1
        
        # 步骤3：递归翻转剩余部分，并连接
        # head 现在是当前组的尾节点，cur 是剩余部分的头
        head.next = self.reverseKGroup(cur, k)
        
        # 返回当前组的新头（prev）
        return prev
```

## 📊 复杂度分析

### 时间复杂度
- **O(n)**：每个节点被访问常数次（一次用于分组判断，一次用于反转）
- n 为链表长度

### 空间复杂度
| 方法 | 空间复杂度 | 说明 |
|------|-----------|------|
| 迭代法 | **O(1)** | 只使用常数个指针变量，满足进阶要求 |
| 递归法 | **O(n/k)** | 递归栈深度为分组数，最坏情况下 k=1 时退化为 O(n) |

## 🎯 关键要点总结

### 1. 虚拟头节点（Dummy Node）
```python
dummy = ListNode(0)
dummy.next = head
```
- **作用**：统一处理头节点和普通节点的翻转逻辑
- **好处**：避免对头节点进行特殊判断，代码更简洁

### 2. 局部反转技巧
```python
def reverseList(self, head):
    prev, curr = None, head
    while curr:
        next_temp = curr.next
        curr.next = prev
        prev, curr = curr, next_temp
    return prev
```
- 三指针法：`prev`, `curr`, `next`
- 每次迭代将 `curr.next` 指向 `prev`

### 3. 边界条件检查
```python
# 检查剩余节点是否足够 k 个
kth_node = prev_group_end
for _ in range(k):
    kth_node = kth_node.next
    if not kth_node:
        return dummy.next  # 不足 k 个，结束
```

### 4. 指针拼接顺序
翻转后需要正确连接三个部分：
1. **前一组的尾** → **当前组的新头**
2. **当前组的尾（原头）** → **下一组的头**

## 📝 面试高频问题

### Q1: 为什么使用虚拟头节点？
**A**: 处理头节点翻转时，头节点会变化。使用 dummy 节点可以统一处理逻辑，无需对头节点特殊判断。

### Q2: 如何优化递归法的空间复杂度？
**A**: 使用迭代法可以将空间复杂度优化到 O(1)，因为递归栈的调用需要额外空间。

### Q3: 如果 k 等于链表长度，算法会发生什么？
**A**: 整个链表会被完全反转一次，相当于对整个链表执行反转操作。

### Q4: 如何处理 k=1 的情况？
**A**: k=1 时每个节点自成一组，翻转后还是原序，直接返回原链表即可。
