## 🎯 LeetCode 精讲 #55 - 跳跃游戏

---

### 📋 题目描述

**原题链接**: https://leetcode.cn/problems/jump-game/

**难度**: 中等  
**标签**: 贪心、数组、动态规划

---

#### 题目原文

> 给你一个非负整数数组 `nums`，你最初位于数组的 **第一个下标**。
>
> 数组中的每个元素代表你在该位置可以跳跃的 **最大长度**。
>
> 判断你是否能够到达最后一个下标。

---

#### 示例

**示例 1：**
```
输入：nums = [2,3,1,1,4]
输出：true
解释：可以先跳 1 步，从下标 0 到达下标 1，然后再从下标 1 跳 3 步到达最后一个下标。
```

**示例 2：**
```
输入：nums = [3,2,1,0,4]
输出：false
解释：无论怎样，总会到达下标为 3 的位置。但该位置的最大跳跃长度是 0，所以永远不可能到达最后一个下标。
```

---

#### 约束条件

- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 10^5`

---

### 💡 解题思路

#### 🔍 核心洞察：从终点倒推

这道题最容易想到的解法是从起点开始模拟跳跃，但这样做会遇到一个问题：**选择太多**。从位置0可以跳到位置1、2...，从每个位置又有多种选择，导致复杂度爆炸。

**关键洞察**：换个角度，从终点倒推——

> "哪些位置可以直接到达终点？"

如果一个位置 `i` 满足 `i + nums[i] >= target`，那么从 `i` 就能跳到终点或更远。

**更进一步的洞察**：

> "我们需要找到一条从起点到终点的路径吗？不需要！我们只关心 *能不能到达* 。"

这其实等价于：**从起点出发，能到达的最远位置是哪里？**

---

#### 🎯 方法一：贪心算法（正序）

**核心思想**：

维护一个变量 `max_reach`，表示从起点出发，当前能够到达的最远位置。

遍历数组，对于每个位置 `i`：
1. 如果 `i > max_reach`，说明当前位置已经超出了可达范围，直接返回 `false`
2. 更新 `max_reach = max(max_reach, i + nums[i])`
3. 如果 `max_reach >= len(nums) - 1`，说明已经可以到达终点，返回 `true`

**为什么这样是对的？**

想象你有一根绳子，绳子的长度就是你能到达的最远位置。每到一个新位置，你就有两种选择：
- 绳子不变（不从这跳）
- 绳子变长（从这跳，跳得更远）

只要绳子能覆盖到终点，就说明可达。

**图解过程**：

```
nums = [2, 3, 1, 1, 4]

下标:    0    1    2    3    4
数值:    2    3    1    1    4
        
初始:    ↑
        max_reach = 0

i=0:    2    3    1    1    4
        ↑
        max_reach = max(0, 0+2) = 2
        
i=1:    2    3    1    1    4
             ↑
        (i=1 <= max_reach=2, 可达)
        max_reach = max(2, 1+3) = 4
        
        4 >= 4 (终点), 返回 true!
```

#### 🎯 方法二：贪心算法（倒序）

**核心思想**：

从终点开始倒推，维护一个变量 `target`，表示当前需要到达的位置。

遍历数组（从后往前），对于每个位置 `i`：
- 如果 `i + nums[i] >= target`，说明从 `i` 可以到达 `target`，更新 `target = i`

最后，如果 `target == 0`，说明可以从起点到达终点。

**图解过程**：

```
nums = [2, 3, 1, 1, 4]

下标:    0    1    2    3    4
数值:    2    3    1    1    4
                    
初始:                   target = 4 (终点)

i=3:    2    3    1    1    4
                         ↑
        3 + 1 = 4 >= 4 ✓
        target = 3

i=2:    2    3    1    1    4
                    ↑
        2 + 1 = 3 >= 3 ✓
        target = 2

i=1:    2    3    1    1    4
             ↑
        1 + 3 = 4 >= 2 ✓
        target = 1

i=0:    2    3    1    1    4
        ↑
        0 + 2 = 2 >= 1 ✓
        target = 0

target == 0, 返回 true!
```

#### 🎯 方法三：动态规划

虽然贪心是最优解，但为了对比理解，我们也看看DP做法。

**状态定义**：
- `dp[i]`：是否能到达位置 `i`（布尔值）

**状态转移**：
- `dp[i] = OR{ dp[j] && (j + nums[j] >= i) }`，对所有 `j < i`

**初始化**：
- `dp[0] = True`（起点总能到达）

**时间复杂度**：O(n²)  
**空间复杂度**：O(n)

```python
def canJump_dp(nums):
    n = len(nums)
    dp = [False] * n
    dp[0] = True
    
    for i in range(1, n):
        for j in range(i):
            if dp[j] and j + nums[j] >= i:
                dp[i] = True
                break
    
    return dp[n-1]
```

**为什么贪心更优？**

DP是 "能否到达每个位置"，而贪心是 "最远能到哪里"。后者更直接地回答了问题，避免了冗余计算。

### 📝 参考代码

#### 最优解：贪心算法（正序）

```python
class Solution:
    def canJump(self, nums: list[int]) -> bool:
        """
        贪心算法：维护最远可达位置
        
        思路：
        1. 遍历数组，维护一个 max_reach 表示当前能到达的最远位置
        2. 如果当前位置 i 已经超出了 max_reach，说明不可达
        3. 否则更新 max_reach = max(max_reach, i + nums[i])
        4. 如果 max_reach 已经覆盖终点，直接返回 True
        
        时间复杂度：O(n)
        空间复杂度：O(1)
        """
        max_reach = 0  # 当前能到达的最远位置
        n = len(nums)
        
        for i in range(n):
            # 如果当前位置已经超出可达范围，说明不可达
            if i > max_reach:
                return False
            
            # 更新最远可达位置
            max_reach = max(max_reach, i + nums[i])
            
            # 已经可以到达终点，提前返回
            if max_reach >= n - 1:
                return True
        
        return True


# 测试代码
if __name__ == "__main__":
    sol = Solution()
    
    # 测试用例 1
    nums1 = [2, 3, 1, 1, 4]
    print(f"输入: {nums1}")
    print(f"输出: {sol.canJump(nums1)}")  # True
    
    # 测试用例 2
    nums2 = [3, 2, 1, 0, 4]
    print(f"输入: {nums2}")
    print(f"输出: {sol.canJump(nums2)}")  # False
    
    # 测试用例 3: 只有一个元素
    nums3 = [0]
    print(f"输入: {nums3}")
    print(f"输出: {sol.canJump(nums3)}")  # True
    
    # 测试用例 4: 第一个就是0
    nums4 = [0, 1]
    print(f"输入: {nums4}")
    print(f"输出: {sol.canJump(nums4)}")  # False
```

#### 替代解：贪心算法（倒序）

```python
class Solution:
    def canJump_reverse(self, nums: list[int]) -> bool:
        """
        贪心算法（倒序）：从终点倒推
        
        思路：
        1. 维护一个 target，表示当前需要到达的位置（初始为终点）
        2. 从后往前遍历，如果 i + nums[i] >= target，说明从 i 可以到达 target
        3. 更新 target = i
        4. 最后检查 target 是否为 0
        
        时间复杂度：O(n)
        空间复杂度：O(1)
        """
        n = len(nums)
        target = n - 1  # 当前需要到达的位置
        
        for i in range(n - 2, -1, -1):
            if i + nums[i] >= target:
                target = i
        
        return target == 0
```

### 🔑 关键要点

#### 1️⃣ 贪心的本质

这道题完美诠释了贪心的核心思想：**只看最远的，不看路径**。

不是 "怎么到达"，而是 "能不能到达"。一旦问题可以转化为范围覆盖，贪心往往是最优解。

#### 2️⃣ 边界条件

- `nums = [0]`：只有一个元素，起点即终点，返回 `True`
- `nums = [0, ...]`：第一个元素为0且长度>1，无法移动，返回 `False`
- `nums = [2, 0, 0]`：可以跳过0，返回 `True`

#### 3️⃣ 正序 vs 倒序

| 特性 | 正序 | 倒序 |
|------|------|------|
| 思路 | "我能到哪" | "谁能到终点" |
| 提前终止 | ✅ 可达终点时提前返回 | ❌ 必须遍历完 |
| 代码量 | 稍多 | 更简洁 |
| 可读性 | 更直观 | 需要理解倒推逻辑 |

#### 4️⃣ 面试常问

1. **为什么不用BFS/DFS？**
   - 可以，但时间复杂度O(n) vs O(n²)，贪心更优
   
2. **如果要求输出跳跃路径呢？**
   - 贪心不行，需要用BFS或DP记录路径
   
3. **如果要求最少跳跃次数呢？**
   - 见 LeetCode #45（跳跃游戏 II），也是贪心，但思路略有不同

### 🎯 面试高频考点

1. **为什么选择贪心而非DP？**
   - 贪心O(n) vs DP O(n²)，且贪心空间O(1)
   - 关键是 "最远可达" 这个单调性

2. **能否输出一条可行路径？**
   - 贪心不行，需要BFS或回溯
   - 倒序贪心可以：记录每次更新 target 的位置

3. **如果数组很大（10^9），如何优化？**
   - 贪心已经是O(n)，无法更优
   - 如果有很多连续的0，可以提前剪枝

4. **变种：每个位置有代价，求最小代价到达终点？**
   - 此时贪心失效，需要用Dijkstra算法
