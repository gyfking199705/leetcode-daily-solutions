#22. 括号生成

Generate Parentheses
Medium 回溯算法 / DFS 字符串 第46期

📋 题目描述

数字 n 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且 有效的 括号组合。

有效括号组合需满足：任意前缀中，左括号的数量都 大于等于 右括号的数量。

示例 1：

输入：n = 3

输出：["((()))","(()())","(())()","()(())","()()()"]

示例 2：

输入：n = 1

输出：["()"]

提示：1 <= n <= 8

💡 核心洞察

这是一道经典的回溯/DFS模板题。关键在于理解「剪枝条件」：

  • 剩余左括号数量 > 0 时，可以添加 '('
  • 已添加的右括号数量 < 已添加的左括号数量时，可以添加 ')'
  • 当左右括号都用完时，得到一个有效组合

🧠 解题思路

1️⃣ 决策树分析 — 理解搜索空间 ▼

对于 n=3，我们需要生成 3 对括号。每个位置有两个选择（理论上）：添加 '(' 或 ')'。

但通过约束条件剪枝，我们得到如下决策树：

""
↓
(
↓
((
()
↓
(((
(()
()) ✗
()(
↓
...继续展开直到生成所有合法组合

注意：当右括号数量 > 左括号数量时，该分支被剪枝（标记为 ✗）。

2️⃣ 算法执行过程 — 以 n=2 为例 ▼
Step 1: 初始状态
当前字符串: ""
已用左括号: 0
已用右括号: 0

选择：可以添加 '(' (剩余3个) 或 ')'？右括号已用0 < 左括号已用0？否，所以不能添加 ')'

Step 2: 添加 '('
当前字符串: "("
已用左括号: 1
已用右括号: 0

选择：可以添加 '(' 或 ')' (1<1? 否, 但 0<1 是, 可以添加 ')')

Step 3-A: 分支 - 添加 '('
当前字符串: "(("
已用左括号: 2
已用右括号: 0

左括号已用完(n=2)，只能添加右括号...

Step 4-A: 添加 ')' 两次 → 得到 "(())" ✅
结果: "(())"
Step 3-B: 分支 - 添加 ')'
当前字符串: "()"
已用左括号: 1
已用右括号: 1

可以添加 '(' 或 ')'？右括号1 < 左括号1？否，只能添加 '('

Step 4-B: 继续 → 得到 "()()" ✅
结果: "()()"
⚠️ 常见误区
  • 不要试图用「枚举所有组合再过滤」的方法 —— 时间复杂度会爆炸（2^(2n) 种可能）
  • 剪枝条件不要写反：是 已用右 < 已用左 才能加右括号，不是剩余右 < 剩余左
  • 注意递归终止条件：字符串长度达到 2*n 时才记录结果

📝 代码实现

Python 解法

1class Solution:


2    def generateParenthesis(self, n: int) -> List[str]:


3        result = []


4        


5        def backtrack(current, left_used, right_used):


6            # 终止条件：生成了 n 对括号


7            if len(current) == 2 * n:


8                result.append(current)


9                return


10            


11            # 选择1：添加左括号（如果还有剩余）


12            if left_used < n:


13                backtrack(current + "(", left_used + 1, right_used)


14            


15            # 选择2：添加右括号（如果右括号数量 < 左括号）


16            if right_used < left_used:


17                backtrack(current + ")", left_used, right_used + 1)


18        


19        backtrack("", 0, 0)


20        return result

Java 解法

1class Solution {


2    public List<String> generateParenthesis(int n) {


3        List<String> result = new ArrayList<>();


4        backtrack(result, new StringBuilder(), 0, 0, n);


5        return result;


6    }


7    


8    private void backtrack(List<String> result, StringBuilder current, 


9                           int left, int right, int n) {


10        if (current.length() == 2 * n) {


11            result.add(current.toString());


12            return;


13        }


14        


15        if (left < n) {


16            current.append('(');


17            backtrack(result, current, left + 1, right, n);


18            current.deleteCharAt(current.length() - 1); // 回溯


19        }


20        


21        if (right < left) {


22            current.append(')');


23            backtrack(result, current, left, right + 1, n);


24            current.deleteCharAt(current.length() - 1); // 回溯


25        }


26    }


27}

📊 复杂度分析

指标 复杂度 说明
时间复杂度 O(4ⁿ/√n) 第 n 个 Catalan 数，约为 4ⁿ/(n^(3/2)√π)
空间复杂度 O(4ⁿ/√n) 结果存储空间 + 递归调用栈 O(n)
结果数量 Catalan(n) n=8 时有 1430 种组合
📐 Catalan 数

有效括号组合数恰好等于第 n 个 Catalan 数：

C(n) = (2n)! / ((n+1)! × n!) = C(2n,n) / (n+1)

Catalan 数列：1, 2, 5, 14, 42, 132, 429, 1430...
