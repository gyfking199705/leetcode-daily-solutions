LeetCode #146

LRU 缓存

Medium
🎨 设计 🗂️ 哈希表 🔗 双向链表 ⚡ O(1) 操作

📋 题目描述

请你设计并实现一个满足 LRU (最近最少使用) 缓存 约束的数据结构。

实现 LRUCache 类：
  • LRUCache(int capacity) 以 正整数 作为容量 capacity 初始化 LRU 缓存
  • int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1
  • void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value；如果不存在，则向缓存中插入该组 key-value。如果插入操作导致关键字数量超过 capacity ，则应该 逐出 最久未使用的关键字。
函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。

📌 示例

输入
["LRUCache", "put", "put", "get", "put", "get", "put", "get", "get", "get"]
[[2], [1, 1], [2, 2], [1], [3, 3], [2], [4, 4], [1], [3], [4]]

输出
[null, null, null, 1, null, -1, null, -1, 3, 4]
解释
LRUCache lRUCache = new LRUCache(2);
lRUCache.put(1, 1); // 缓存是 {1=1}
lRUCache.put(2, 2); // 缓存是 {1=1, 2=2}
lRUCache.get(1); // 返回 1，缓存变为 {2=2, 1=1}
lRUCache.put(3, 3); // 逐出 key 2，缓存是 {1=1, 3=3}
lRUCache.get(2); // 返回 -1 (未找到)
lRUCache.put(4, 4); // 逐出 key 1，缓存是 {3=3, 4=4}
lRUCache.get(1); // 返回 -1 (未找到)
lRUCache.get(3); // 返回 3，缓存变为 {4=4, 3=3}
lRUCache.get(4); // 返回 4，缓存变为 {3=3, 4=4}

💡 解题思路

💡 核心洞察

LRU缓存的核心矛盾：需要同时支持 O(1) 查找 和 O(1) 移动/删除。
单一数据结构无法满足，因此采用 哈希表 + 双向链表 的组合策略。

🔍 为什么需要两种数据结构？

▼
数据结构 查找 插入/删除 维护访问顺序
数组 O(n) O(n) ❌ 困难
普通链表 O(n) O(1)* ✅ 容易
哈希表 O(1) O(1) ❌ 无顺序
哈希 + 双向链表 O(1) O(1) ✅ 完美

*链表插入删除是O(1)，但需要先找到节点，所以单独使用仍是O(n)

👆 最近使用 (MRU) 最久未使用 (LRU) 👇
🔗 协作原理：
哈希表负责 O(1) 定位 节点 → 双向链表负责 O(1) 调整顺序
头节点 = 最近使用 (Most Recently Used)
尾节点 = 最久未使用 (Least Recently Used) → 淘汰候选

⚙️ 操作步骤详解

1
Get 操作 - 获取值
① 哈希表查找key → 不存在返回-1
② 存在则取出节点，移动到链表头部（标记为最近使用）
③ 返回节点值
2
Put 操作 - 插入/更新
① key已存在？→ 更新值，移动到头部
② key不存在？→ 创建新节点，插入头部
③ 检查容量：超出则删除尾部节点（最久未使用）
3
MoveToHead - 移到头部
① 从当前位置删除节点（修改前后指针）
② 插入到头部（头节点的next位置）
时间复杂度：O(1)
4
RemoveTail - 淘汰尾部
① 获取尾节点（tail.prev）
② 从链表和哈希表中删除
③ 返回被删除的key（用于清理哈希表）

📊 复杂度分析

操作 时间复杂度 空间复杂度 说明
get O(1) O(1) 哈希查找O(1) + 链表移动O(1)
put O(1) O(1) 哈希插入O(1) + 链表操作O(1)
总体空间 - O(capacity) 哈希表 + 链表，最多capacity个节点

🎯 为什么使用双向链表而非单向链表？

删除节点时需要知道前驱节点。单向链表需要从头遍历找前驱（O(n)），双向链表直接通过 prev 指针获取（O(1)）。
