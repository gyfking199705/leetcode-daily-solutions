## 🎯 LeetCode 精讲 #215 - 数组中的第K个最大元素

---

### 📋 题目描述

给定整数数组 `nums` 和整数 `k`，请返回数组中第 `k` 个最大的元素。

请注意，你需要找的是数组排序后的第 `k` 个最大的元素，而不是第 `k` 个不同的元素。

**示例 1：**
```
输入: [3,2,1,5,6,4], k = 2
输出: 5
解释: 排序后数组为 [1,2,3,4,5,6]，第2大的元素是5
```

**示例 2：**
```
输入: [3,2,3,1,2,4,5,5,6], k = 4
输出: 4
解释: 排序后数组为 [1,2,2,3,3,4,5,5,6]，第4大的元素是4
```

**提示：**
- `1 <= k <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

---

### 💡 解题思路

#### 方法一：排序（最直观）

**核心思想**：直接排序，取倒数第k个元素

- 时间复杂度：O(n log n)
- 空间复杂度：O(1) 或 O(log n)（取决于排序算法）

```python
def findKthLargest(nums, k):
    nums.sort()
    return nums[-k]
```

这个方法简单直接，但不是最优解。

---

#### 方法二：最小堆 ⭐ 推荐（面试常考）

**核心思想**：维护一个大小为k的最小堆，堆顶就是第k大的元素

**详细解释**：

想象你有k个"名额"，你要找的是第k大的数。维护一个大小为k的小根堆：

1. **堆的性质**：小根堆的堆顶是所有元素中最小的
2. **维护过程**：
   - 遍历数组，先将前k个元素放入堆中
   - 对于剩下的每个元素：
     - 如果当前元素 **大于** 堆顶，说明这个元素有资格进入前k大
     - 弹出堆顶（当前k个中最小的），放入当前元素
   - 遍历结束后，堆中保留的就是数组中最大的k个元素
   - 堆顶就是这k个中最小的，即第k大的元素

**为什么这样是对的？**
- 堆中始终保持k个元素，是迄今为止见过的最大的k个数
- 堆顶是这k个数中最小的
- 任何比堆顶小的数都不可能进入前k大

---

#### 方法三：快速选择 ⭐⭐ 最优解（面试加分项）

**核心思想**：利用快速排序的partition思想，平均O(n)时间复杂度

**详细解释**：

快速排序的partition操作会将数组分为两部分：
- `nums[left...p-1]` < `pivot`（基准值）
- `nums[p]` = `pivot`
- `nums[p+1...right]` > `pivot`

我们需要找第k大的元素，即排序后索引为 `n-k` 的元素（0-based）。

**算法步骤**：

1. 随机选择一个pivot（避免最坏情况）
2. 进行partition，将pivot放到正确位置 `p`
3. 比较 `p` 与 `n-k`：
   - 如果 `p == n-k`：找到了！返回 `nums[p]`
   - 如果 `p < n-k`：目标在右半部分，递归右边
   - 如果 `p > n-k`：目标在左半部分，递归左边

**为什么平均是O(n)？**
- 每次partition大约将搜索范围减半
- 期望比较次数：n + n/2 + n/4 + ... ≈ 2n
- 因此是O(n)时间复杂度

### 📝 参考代码

#### Python - 最小堆解法

```python
import heapq
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        """
        最小堆解法：维护大小为k的堆
        时间复杂度：O(n log k)
        空间复杂度：O(k)
        """
        # 使用heapq创建最小堆（默认）
        # 方法：先将前k个元素入堆，然后维护堆的大小为k
        
        min_heap = nums[:k]  # 取前k个元素
        heapq.heapify(min_heap)  # 建堆，O(k)
        
        # 遍历剩下的元素
        for num in nums[k:]:
            # 如果当前元素大于堆顶，替换
            if num > min_heap[0]:
                heapq.heapreplace(min_heap, num)
                # heapreplace = pop then push，比先heappop再heappush效率高
        
        # 堆顶就是第k大的元素
        return min_heap[0]
```

#### Python - 快速选择解法（最优）

```python
import random
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        """
        快速选择算法：平均O(n)时间复杂度
        类似快速排序的partition思想
        """
        n = len(nums)
        # 第k大 = 排序后索引为 n-k 的元素（从小到大排序）
        return self.quickSelect(nums, 0, n - 1, n - k)
    
    def quickSelect(self, nums: List[int], left: int, right: int, target_idx: int) -> int:
        """
        在nums[left:right+1]中找到第target_idx小的元素
        """
        if left == right:
            return nums[left]
        
        # 随机选择pivot，避免最坏情况O(n²)
        pivot_idx = random.randint(left, right)
        
        # 将pivot放到最右边
        nums[pivot_idx], nums[right] = nums[right], nums[pivot_idx]
        
        # partition操作：将<pivot的放左边，>pivot的放右边
        store_idx = left
        for i in range(left, right):
            if nums[i] < nums[right]:  # 当前元素小于pivot
                nums[store_idx], nums[i] = nums[i], nums[store_idx]
                store_idx += 1
        
        # 将pivot放到正确位置
        nums[store_idx], nums[right] = nums[right], nums[store_idx]
        
        # 现在pivot在store_idx位置
        if store_idx == target_idx:
            return nums[store_idx]
        elif store_idx < target_idx:
            # 目标在右半部分
            return self.quickSelect(nums, store_idx + 1, right, target_idx)
        else:
            # 目标在左半部分
            return self.quickSelect(nums, left, store_idx - 1, target_idx)
```

#### Python - 快速选择（迭代版，避免递归栈溢出）

```python
import random
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        """
        快速选择 - 迭代实现
        """
        n = len(nums)
        target_idx = n - k  # 转换为第target_idx小（0-based）
        
        left, right = 0, n - 1
        
        while True:
            if left == right:
                return nums[left]
            
            # 随机选择pivot并partition
            pivot_idx = random.randint(left, right)
            nums[pivot_idx], nums[right] = nums[right], nums[pivot_idx]
            
            store_idx = left
            for i in range(left, right):
                if nums[i] < nums[right]:
                    nums[store_idx], nums[i] = nums[i], nums[store_idx]
                    store_idx += 1
            
            nums[store_idx], nums[right] = nums[right], nums[store_idx]
            
            if store_idx == target_idx:
                return nums[store_idx]
            elif store_idx < target_idx:
                left = store_idx + 1
            else:
                right = store_idx - 1
```

### ⏱️ 复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 排序 | O(n log n) | O(1) / O(log n) | 简单但非最优 |
| 最小堆 | O(n log k) | O(k) | k较小时效率高，面试推荐 |
| 快速选择（平均）| **O(n)** | O(1) | 理论最优，面试加分 |
| 快速选择（最坏）| O(n²) | O(n) | 随机化可避免 |

### 🔑 关键要点

1. **最小堆的直觉**：
   - 堆顶是"门槛"，只有比它大的才有资格进入前k
   - 遍历完数组，堆中就是最大的k个数
   - 堆顶是这k个中最小的，即第k大

2. **快速选择的核心**：
   - partition后，pivot的位置就是最终排序后的位置
   - 根据pivot位置与目标索引的关系，决定递归哪一半
   - 平均每次处理一半数据，总复杂度O(n)

3. **面试技巧**：
   - 先写最小堆解法，展示基础扎实
   - 再提快速选择，展示深入理解
   - 强调随机化避免最坏情况

4. **常见陷阱**：
   - 第k大 ≠ 第k小，注意索引转换（n-k）
   - 快速选择非原地实现时，注意空间复杂度
   - 边界条件：k=1（最大值）、k=n（最小值）

### 🎓 扩展思考

**如果数组非常大（无法全部放入内存），怎么办？**
- 使用**蓄水池抽样**或**外部排序**
- 或者使用**桶排序**思想（知道数据范围时）

**如果需要找中位数？**
- 中位数就是第 `n//2` 大（或第 `(n+1)//2` 大）
- 同样可以用快速选择O(n)解决

**如果要求第k大和第m大的元素？**
- 最小堆：一次遍历得到两个结果
- 快速选择：两次调用，或一次找较小的那个，另一次在相应区间找
