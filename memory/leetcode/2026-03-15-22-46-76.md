## 一、题目描述

**链接**: https://leetcode.cn/problems/minimum-window-substring/

### 题目

给定两个字符串 `s` 和 `t`，返回 `s` 中涵盖 `t` 所有字符的最小子串。如果不存在符合条件的子串，则返回空字符串 `""`。

**注意**：
- 如果 `s` 中存在这样的子串，我们保证它是唯一的答案
- `t` 中的字符可能包含重复字符，子串中对应字符数量必须不少于 `t` 中的数量

### 示例

```
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小子串 "BANC" 包含了字符串 t 的所有字符 'A'、'B' 和 'C'

输入：s = "a", t = "a"
输出："a"

输入：s = "a", t = "aa"
输出：""
解释：t 中两个字符 'a' 均应包含在 s 的子串中，因此不存在符合条件的子串
```

### 约束条件
- `m == s.length`, `n == t.length`
- `1 <= m, n <= 10^5`
- `s` 和 `t` 由英文字母组成

## 二、解题思路

### 核心思想：滑动窗口

这道题是**滑动窗口算法**的经典应用，核心逻辑可以用一句话概括：

> **右指针扩展窗口找"满足条件的窗口"，左指针收缩窗口找"最短的窗口"**

#### 算法框架

```
初始化：left = 0, right = 0

while right < s.length:
    # 1. 扩张窗口：将 s[right] 加入窗口
    window.add(s[right])
    right++
    
    # 2. 判断窗口是否满足条件（包含t的所有字符）
    while 窗口满足条件:
        # 3. 更新最小窗口记录
        记录当前窗口
        
        # 4. 收缩窗口：将 s[left] 移出窗口
        window.remove(s[left])
        left++
```

### 关键问题：如何判断窗口满足条件？

使用**双哈希表** + **有效计数器**：

1. **`need`**: 记录 `t` 中每个字符需要的次数
2. **`window`**: 记录当前窗口中每个字符的实际次数  
3. **`valid`**: 记录窗口中满足需求的字符种类数

当 `valid == need.size` 时，说明窗口已覆盖 `t` 的所有字符。

## 三、完整代码实现

### Python 实现

```python
class Solution:
    def minWindow(self, s: str, t: str) -> str:
        from collections import defaultdict
        
        need = defaultdict(int)  # t中字符的需求频次
        window = defaultdict(int)  # 当前窗口中字符的频次
        
        # 统计t中字符需求
        for c in t:
            need[c] += 1
        
        left, right = 0, 0
        valid = 0  # 窗口中满足need条件的字符个数
        
        # 记录最小覆盖子串的起始索引和长度
        start, length = 0, float('inf')
        
        while right < len(s):
            # c是将要移入窗口的字符
            c = s[right]
            right += 1
            
            # 更新窗口数据
            if c in need:
                window[c] += 1
                # 当窗口中该字符数量刚好满足需求时，valid++
                if window[c] == need[c]:
                    valid += 1
            
            # 判断左侧窗口是否需要收缩
            while valid == len(need):
                # 更新最小覆盖子串
                if right - left < length:
                    start = left
                    length = right - left
                
                # d是将要移出窗口的字符
                d = s[left]
                left += 1
                
                # 更新窗口数据
                if d in need:
                    # 移出前该字符刚好满足需求，移出后就不满足了
                    if window[d] == need[d]:
                        valid -= 1
                    window[d] -= 1
        
        return "" if length == float('inf') else s[start:start + length]
```

### Java 实现

```java
class Solution {
    public String minWindow(String s, String t) {
        Map<Character, Integer> need = new HashMap<>();
        Map<Character, Integer> window = new HashMap<>();
        
        // 统计t中字符需求
        for (char c : t.toCharArray()) {
            need.put(c, need.getOrDefault(c, 0) + 1);
        }
        
        int left = 0, right = 0;
        int valid = 0;
        
        // 记录最小覆盖子串的起始索引和长度
        int start = 0, len = Integer.MAX_VALUE;
        
        while (right < s.length()) {
            char c = s.charAt(right);
            right++;
            
            // 更新窗口数据
            if (need.containsKey(c)) {
                window.put(c, window.getOrDefault(c, 0) + 1);
                if (window.get(c).equals(need.get(c))) {
                    valid++;
                }
            }
            
            // 判断左侧窗口是否需要收缩
            while (valid == need.size()) {
                // 更新最小覆盖子串
                if (right - left < len) {
                    start = left;
                    len = right - left;
                }
                
                char d = s.charAt(left);
                left++;
                
                // 更新窗口数据
                if (need.containsKey(d)) {
                    if (window.get(d).equals(need.get(d))) {
                        valid--;
                    }
                    window.put(d, window.get(d) - 1);
                }
            }
        }
        
        return len == Integer.MAX_VALUE ? "" : s.substring(start, start + len);
    }
}
```

### C++ 实现

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map<char, int> need, window;
        
        // 统计t中字符需求
        for (char c : t) need[c]++;
        
        int left = 0, right = 0;
        int valid = 0;
        
        // 记录最小覆盖子串的起始索引和长度
        int start = 0, len = INT_MAX;
        
        while (right < s.size()) {
            char c = s[right];
            right++;
            
            // 更新窗口数据
            if (need.count(c)) {
                window[c]++;
                if (window[c] == need[c]) valid++;
            }
            
            // 判断左侧窗口是否需要收缩
            while (valid == need.size()) {
                // 更新最小覆盖子串
                if (right - left < len) {
                    start = left;
                    len = right - left;
                }
                
                char d = s[left];
                left++;
                
                // 更新窗口数据
                if (need.count(d)) {
                    if (window[d] == need[d]) valid--;
                    window[d]--;
                }
            }
        }
        
        return len == INT_MAX ? "" : s.substr(start, len);
    }
};
```

## 四、复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| **时间复杂度** | O(m + n) | m为s长度，n为t长度。左右指针各遍历s一次，最坏情况下每个字符被访问两次 |
| **空间复杂度** | O(k) | k为字符集大小，哈希表存储字符频次，最多存储ASCII字符集的256个字符 |

**详细分析**：
- 初始化构建 `need` 哈希表：O(n)
- 滑动窗口过程：每个字符最多被 `right` 和 `left` 各访问一次，共 O(2m) = O(m)
- 总时间复杂度：O(m + n)

## 五、关键要点与易错点

### ✅ 正确理解 `valid` 计数器

`valid` 不是"窗口中包含的t字符种类数"，而是"**满足数量要求**的字符种类数"。

- 只有当 `window[c] == need[c]` 时，`valid` 才增加
- 当 `valid == need.size()` 时，窗口才真正满足条件

### ⚠️ 注意重复字符的处理

如果 `t = "AAB"`，则子串中必须包含 **2个A** 和 **1个B** 才算满足条件。

### ⚠️ 收缩窗口的时机

本题是**窗口合法时收缩**（与"无重复字符的最长子串"相反）：
- 那道题是窗口不合法时收缩（出现重复）
- 本题是窗口合法时收缩（满足覆盖条件）

## 七、面试常问问题

**Q1: 如果字符集很大，如何优化空间？**
> 使用长度为128或256的数组代替HashMap（针对ASCII字符），可将空间优化到O(1)。

**Q2: 如何保证找到的是"最小"子串？**
> 每次满足条件时，先记录当前窗口，然后尝试收缩左边界。由于左指针是逐步右移的，第一次遇到满足条件的窗口不一定最小，需要持续收缩直到不满足条件为止。

**Q3: 时间复杂度能否进一步优化？**
> 不能。因为最坏情况下需要遍历整个字符串s，Ω(m)是下界。
