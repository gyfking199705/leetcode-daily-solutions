# 🎯 LeetCode 精讲 #3 - 无重复字符的最长子串

---

### 📋 题目描述

**来源**: LeetCode 第3题  
**难度**: Medium  
**链接**: https://leetcode.cn/problems/longest-substring-without-repeating-characters/

#### 题目原文
> 给定一个字符串 `s` ，请你找出其中不含有重复字符的 **最长子串** 的长度。

#### 示例

**示例 1:**
```
输入: s = "abcabcbb"
输出: 3
解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。
```

**示例 2:**
```
输入: s = "bbbbb"
输出: 1
解释: 因为无重复字符的最长子串是 "b"，所以其长度为 1。
```

**示例 3:**
```
输入: s = "pwwkew"
输出: 3
解释: 因为无重复字符的最长子串是 "wke"，所以其长度为 3。
         请注意，你的答案必须是 子串 的长度，"pwke" 是一个子序列，不是子串。
```

#### 约束条件
- `0 <= s.length <= 5 × 10⁴`
- `s` 由英文字母、数字、符号和空格组成

### 💡 解题思路

这道题是**滑动窗口（Sliding Window）**算法的经典入门题。核心思想是维护一个"窗口"，窗口内的字符都是不重复的，通过移动窗口的左右边界来寻找最长子串。

#### 核心概念：滑动窗口

```
字符串:  a b c a b c b b
         ↑
         left=right=0 (窗口为空)

步骤1: 加入 'a', 窗口 [a], 无重复 ✓
步骤2: 加入 'b', 窗口 [a,b], 无重复 ✓  
步骤3: 加入 'c', 窗口 [a,b,c], 无重复 ✓
步骤4: 加入 'a', 发现重复! 
       移动 left 到第一个 'a' 的下一个位置
       窗口 [b,c,a], 无重复 ✓
...以此类推
```

#### 算法步骤详解

1. **初始化**: 
   - `left = 0` (窗口左边界)
   - `maxLen = 0` (记录最大长度)
   - `charSet = {}` (哈希集合，存储窗口内的字符)

2. **遍历右边界** (`right` 从 0 到 n-1):
   - 如果 `s[right]` 不在 `charSet` 中:
     - 将其加入 `charSet`
     - 更新 `maxLen = max(maxLen, right - left + 1)`
   - 如果 `s[right]` 已在 `charSet` 中:
     - 说明窗口内出现了重复，需要收缩左边界
     - 不断移除 `s[left]` 并右移 `left`，直到 `s[right]` 不在集合中
     - 然后将 `s[right]` 加入集合

3. **返回** `maxLen`

#### 优化：哈希表记录位置

上面的方法在遇到重复时需要逐个移除左边字符，时间复杂度最坏为 O(n²)。

**优化思路**: 用哈希表记录每个字符**最后出现的位置**，发现重复时直接将 `left` 跳到该位置之后。

```
优化后的跳跃:

字符串:  a b c a b c
索引:    0 1 2 3 4 5

当 right=3 时，'a' 已经在位置0出现过
直接设置 left = max(left, lastPos['a'] + 1) = max(0, 1) = 1

窗口瞬间从 [a,b,c] 变为 [b,c,a]，无需逐个移除!
```

### 📝 代码实现

#### 方法一：哈希集合（基础版）

```python
def lengthOfLongestSubstring(s: str) -> int:
    char_set = set()  # 存储窗口内的字符
    left = 0          # 左指针
    max_len = 0       # 最大长度
    
    for right in range(len(s)):
        # 如果当前字符已在窗口中，收缩左边界
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1
        
        # 将当前字符加入窗口
        char_set.add(s[right])
        # 更新最大长度
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

**复杂度分析**:
- 时间复杂度: O(n) — 每个字符最多被访问两次（右指针一次，左指针一次）
- 空间复杂度: O(min(m, n)) — m 是字符集大小，n 是字符串长度

#### 方法二：哈希表优化版（推荐）⭐

```python
def lengthOfLongestSubstring(s: str) -> int:
    # 记录字符最后出现的位置
    char_index = {}
    left = 0
    max_len = 0
    
    for right in range(len(s)):
        char = s[right]
        
        # 如果字符已存在且在窗口内，直接跳到该位置之后
        if char in char_index and char_index[char] >= left:
            left = char_index[char] + 1
        
        # 更新字符位置
        char_index[char] = right
        # 更新最大长度
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

**复杂度分析**:
- 时间复杂度: O(n) — 只需一次遍历
- 空间复杂度: O(min(m, n)) — 哈希表存储

#### 方法三：数组优化（针对 ASCII 字符）

```python
def lengthOfLongestSubstring(s: str) -> int:
    # ASCII 共128个字符，用数组替代哈希表
    # -1 表示该字符未出现过
    last_pos = [-1] * 128
    left = 0
    max_len = 0
    
    for right in range(len(s)):
        char_code = ord(s[right])
        
        # 如果字符已出现且在窗口内
        if last_pos[char_code] >= left:
            left = last_pos[char_code] + 1
        
        # 更新位置
        last_pos[char_code] = right
        max_len = max(max_len, right - left + 1)
    
    return max_len
```

**复杂度分析**:
- 时间复杂度: O(n)
- 空间复杂度: O(1) — 固定大小128的数组

### ⚠️ 易错点与面试陷阱

1. **子串 vs 子序列**
   - 子串：连续的字符序列
   - 子序列：不要求连续，保持相对顺序即可
   - 本题要求子串，必须是连续的！

2. **`left` 更新的条件**
   ```python
   # 错误写法
   if char in char_index:
       left = char_index[char] + 1
   
   # 正确写法
   if char in char_index and char_index[char] >= left:
       left = char_index[char] + 1
   ```
   必须判断字符是否在**当前窗口内**，否则会把 `left` 拉回到已经处理过的位置！

3. **空字符串处理**
   - 如果 `s = ""`，应返回 0
   - 代码中 `max_len` 初始化为 0 即可正确处理

4. **Unicode 字符**
   - 如果字符串包含中文等非 ASCII 字符，数组优化方法失效
   - 需要使用哈希表方法
