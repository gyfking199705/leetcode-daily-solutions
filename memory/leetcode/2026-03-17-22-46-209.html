LeetCode 209 - 长度最小的子数组

Minimum Size Subarray Sum

📋 题目描述

给定一个含有 n 个正整数的数组和一个正整数 target 。

找出该数组中满足其总和大于等于 target 的长度最小的 连续子数组 [numsl, numsl+1, ..., numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。

核心思路

使用滑动窗口（Sliding Window）技术。用两个指针 left 和 right 维护一个窗口，当窗口内元素和 ≥ target 时，尝试收缩窗口以找到更短的有效子数组。

示例

输入：target = 7, nums = [2,3,1,2,4,3]
输出：2
解释：子数组 [4,3] 是该条件下的长度最小的子数组，和为 7。

📊 复杂度分析

指标 值 说明
时间复杂度 O(n) 每个元素最多被访问两次（右指针扩展一次，左指针收缩一次）
空间复杂度 O(1) 只使用常数额外空间（left, right, current_sum, min_len）
最优解 ✓ 是 对于正整数数组，滑动窗口是此问题的最优解法

💡 关键点解析

1. 为什么滑动窗口适用于正整数数组？

因为都是正数，当右指针右移时窗口和一定单调递增；当左指针右移时窗口和一定单调递减。这种单调性保证了我们不会遗漏最优解。

2. 收缩窗口的时机

当 current_sum ≥ target 时，说明当前窗口是「有效」的。此时应该尝试收缩左边界，看看能否找到更短的有效子数组。

3. 为什么用 while 而不是 if 收缩？

因为收缩一个元素后，新的窗口可能仍然满足条件（例如 [3,4] 和为7，收缩3后 [4] 不足7）。使用 while 可以持续收缩直到窗口不再有效。

📝 代码实现

Python - 滑动窗口解法
class Solution:
    def minSubArrayLen(self, target: int, nums: List[int]) -> int:
        n = len(nums)
        left = 0
        current_sum = 0
        min_len = float('inf')
        
        for right in range(n):
            current_sum += nums[right]  # 扩展右边界
            
            # 当窗口满足条件时，尝试收缩左边界
            while current_sum >= target:
                min_len = min(min_len, right - left + 1)
                current_sum -= nums[left]
                left += 1
        
        return min_len if min_len != float('inf') else 0
