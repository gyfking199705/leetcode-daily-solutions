LeetCode #240 - 搜索二维矩阵 II

Medium 二分查找优化 / 从右上角搜索

📋 题目描述

编写一个高效的算法来搜索 m × n 矩阵 matrix 中的一个目标值 target。该矩阵具有以下特性：

  • 每行的元素从左到右升序排列。
  • 每列的元素从上到下升序排列。

示例：

matrix = [
  [1,   4,  7, 11, 15],
  [2,   5,  8, 12, 19],
  [3,   6,  9, 16, 22],
  [10, 13, 14, 17, 24],
  [18, 21, 23, 26, 30]
]
target = 5  → 返回 true
target = 20 → 返回 false

🎬 算法执行过程（点击步骤或代码行查看对应关系）

从右上角开始搜索：当前值 > target 则左移，当前值 < target 则下移。利用矩阵的行列有序性一次排除一行或一列。

📊 复杂度分析

复杂度类型 值 说明
时间复杂度 O(m + n) 每次比较排除一行或一列，最多遍历 m+n 个元素
空间复杂度 O(1) 只使用常数额外空间

对比：暴力搜索 O(mn)，每行二分 O(m log n)，本算法达到最优 O(m+n)。

💡 核心思路解析

为什么从右上角开始？

右上角元素 15 有一个特殊性质：

  • 它的左边所有元素都比它小（同一行，升序）
  • 它的下边所有元素都比它大（同一列，升序）

这种「一边是小的，一边是大的」特性让我们可以做二分决策：

  • 如果 target < 当前值 → target 不可能在当前列的下方 → 向左移动
  • 如果 target > 当前值 → target 不可能在当前行的左方 → 向下移动

能否从左下角开始？

可以！左下角元素 18 也有类似性质：右边都更大，上边都更小。两种方法完全对称。
