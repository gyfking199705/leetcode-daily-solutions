LeetCode #139 - 单词拆分

Word Break | 动态规划经典入门题

📋 题目描述 Medium

给你一个字符串 s 和一个字符串列表 wordDict 作为字典。请你判断是否可以利用字典中出现的单词拼接出 s。

注意：不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用。

示例 1

输入: s = "leetcode", wordDict = ["leet", "code"]

输出: true

解释: 返回 true 因为 "leetcode" 可以由 "leet" 和 "code" 拼接成

示例 2

输入: s = "applepenapple", wordDict = ["apple", "pen"]

输出: true

解释: 返回 true 因为 "applepenapple" 可以由 "apple" + "pen" + "apple" 拼接成

示例 3

输入: s = "catsandog", wordDict = ["cats", "dog", "sand", "and", "cat"]

输出: false

📊 复杂度分析

维度 值 说明
时间复杂度 O(n × m × k) n=字符串长度, m=字典单词数, k=平均单词长度。优化版可做到O(n²)
空间复杂度 O(n) 只需要一个长度为n+1的dp数组

💡 解题思路详解

核心思想：动态规划

定义 dp[i] 表示字符串 s[0:i]（前i个字符）是否可以被拆分成字典中的单词。

状态转移方程

dp[i] = True 如果存在某个 j < i，使得：
1. dp[j] == True（前j个字符可以被拆分）
2. s[j:i] 在字典中

初始化

dp[0] = True，表示空字符串可以被「拆分」（作为基础状态）

优化技巧

  • 将 wordDict 转换为 set，实现 O(1) 的查找
  • 只需检查长度不超过最大单词长度的子串
  • 提前返回：如果某个位置为True，直接跳到下一位置
