LeetCode #287 - 寻找重复数

Medium 数组 双指针 二分查找 Floyd判圈

📋 题目描述

给定一个包含 n + 1 个整数的数组 nums，其数字都在 [1, n] 范围内（包括 1 和 n），可知至少存在一个重复的整数。

假设 nums 只有 一个重复的整数，返回这个重复的数。

限制条件：

  • 不能更改原数组（假设数组是只读的）
  • 只能使用额外的 O(1) 的空间
  • 时间复杂度小于 O(n²)
示例 1

输入: nums = [1,3,4,2,2]

输出: 2

示例 2

输入: nums = [3,1,3,4,2]

输出: 3

💡 核心洞察

这道题的精妙之处在于：把数组看作链表！数组值作为"下一个节点的索引"，重复的数字会形成环。于是问题转化为：找环的入口——这正是经典的 Floyd 判圈算法。

🔍 算法原理解析

为什么 Floyd 判圈算法适用？

将数组 nums 看作一个函数 f(i) = nums[i]，从任意位置出发，按照这个映射关系一定会形成环（因为有重复数字）。

Floyd 判圈算法的两个阶段：

  1. 相遇阶段：快慢指针（速度2:1）从起点出发，必在环内某点相遇
  2. 找入口阶段：一个指针从起点，一个从相遇点，同速前进，相遇点即环入口（重复数）

数学证明：设环前长度为 a，环长度为 b，相遇时慢指针走了 a + x，快指针走了 a + x + kb。由于 2(a+x) = a+x+kb，得 a+x = kb，即 x = kb - a。因此从起点走 a 步，从相遇点走 kb-x = a 步，会在入口相遇。
