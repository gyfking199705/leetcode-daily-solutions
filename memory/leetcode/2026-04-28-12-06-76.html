76. 最小覆盖子串

LeetCode 精讲 · 第245期 · 2026-04-28 12:06 (Asia/Shanghai)

📝 题目描述

给你一个字符串 s 、一个字符串 t 。返回 s 中涵盖 t 所有字符的最小子串。如果不存在涵盖 t 所有字符的子串，则返回空字符串 ""。


注意：

  • 对于 t 中重复字符，咱们寻找的子串中该字符数量必须不少于 t 中该字符数量。
  • 如果 s 中存在这样的子串，我们保证它是唯一的答案。

🎯 示例

参数 值
s "ADOBECODEBANC"
t "ABC"
输出 "BANC"
解释：最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。
参数 值
s "a"
t "aa"
输出 ""（空字符串）
陷阱：t 中有两个 'a'，但 s 中只有一个 'a'，因此没有符合条件的子串。

💡 核心思路

这道题是滑动窗口（Sliding Window）算法的经典应用，难度为 Hard，关键在于同时控制窗口扩张和窗口收缩两个过程。

核心观察：如果一个子串 s[left..right] 已经包含了 t 中所有字符，那么当我们右移 left 缩小窗口时，只要窗口仍然包含 t 的所有字符，就得到了一个更优（更短）的候选答案。

算法框架（双指针）

1初始化 left=0, right=0
2right 向右扩展窗口，直到窗口包含 t 中所有字符
3尝试 left 右移收缩窗口，直到窗口不再满足条件
4记录这个过程中的最小窗口
5重复 2-4 直到 right 到达末尾

如何快速判断"窗口包含 t 中所有字符"？

使用哈希表维护字符频率：

  • need[char]：字符 char 在 t 中出现次数
  • window[char]：字符 char 在当前窗口中出现次数
  • 额外维护一个 valid 计数器：表示窗口中满足 need 条件的字符个数
valid 计数器的精妙之处：当 window[c] == need[c] 时，字符 c 刚好"达标"，valid++。如果 window[c] > need[c]，valid 不变（超量了）。这样 valid 只需要跟 need 中不同字符的种类数比较，而不是每次遍历整个哈希表。

Python 解法
from collections import Counter

class Solution:
    def minWindow(self, s: str, t: str) -> str:
        # need: t 中各字符的需求
        need = Counter(t)
        # window: 当前窗口中各字符的计数
        window = Counter()
        
        left, right = 0, 0
        # valid: 窗口中满足 need 条件的字符种数
        valid = 0
        # 记录最小覆盖子串的起始位置和长度
        start, length = 0, float('inf')
        
        while right < len(s):
            # c 是将移入窗口的字符
            c = s[right]
            right += 1
            # 更新窗口内数据
            if c in need:
                window[c] += 1
                if window[c] == need[c]:
                    valid += 1
            
            # 判断左侧窗口是否要收缩
            while valid == len(need):
                # 更新最小覆盖子串
                if right - left < length:
                    start = left
                    length = right - left
                
                # d 是将移出窗口的字符
                d = s[left]
                left += 1
                # 更新窗口内数据
                if d in need:
                    if window[d] == need[d]:
                        valid -= 1
                    window[d] -= 1
        
        return "" if length == float('inf') else s[start:start+length]
Java 解法
class Solution {
    public String minWindow(String s, String t) {
        int[] need = new int[128];
        int[] window = new int[128];
        
        for (char c : t.toCharArray()) {
            need[c]++;
        }
        
        int left = 0, right = 0;
        int valid = 0;
        int needCnt = 0;
        for (int n : need) {
            if (n > 0) needCnt++;
        }
        
        int start = 0, len = Integer.MAX_VALUE;
        
        while (right < s.length()) {
            char c = s.charAt(right++);
            if (need[c] > 0) {
                window[c]++;
                if (window[c] == need[c]) {
                    valid++;
                }
            }
            
            while (valid == needCnt) {
                if (right - left < len) {
                    start = left;
                    len = right - left;
                }
                
                char d = s.charAt(left++);
                if (need[d] > 0) {
                    if (window[d] == need[d]) {
                        valid--;
                    }
                    window[d]--;
                }
            }
        }
        
        return len == Integer.MAX_VALUE ? "" : s.substring(start, start + len);
    }
}
C++ 解法
string minWindow(string s, string t) {
    unordered_map<char, int> need, window;
    for (char c : t) need[c]++;
    
    int left = 0, right = 0;
    int valid = 0;
    int start = 0, len = INT_MAX;
    
    while (right < s.size()) {
        char c = s[right++];
        if (need.count(c)) {
            window[c]++;
            if (window[c] == need[c]) valid++;
        }
        
        while (valid == need.size()) {
            if (right - left < len) {
                start = left;
                len = right - left;
            }
            char d = s[left++];
            if (need.count(d)) {
                if (window[d] == need[d]) valid--;
                window[d]--;
            }
        }
    }
    
    return len == INT_MAX ? "" : s.substr(start, len);
}

⏱️ 复杂度分析

指标 值 说明
时间复杂度 O(|s| + |t|) left 和 right 都最多遍历 s 一次，初始化 need 需要遍历 t
空间复杂度 O(|Σ|) 哈希表大小，|Σ| 为字符集大小。英文/ASCII 时是 O(128) ≈ O(1)
关键理解：虽然有两个 while 循环嵌套，但 left 和 right 指针都只会向右移动，各自最多遍历 s 一次。因此时间复杂度是 O(2|s|) = O(|s|)，而不是 O(n²)。

🎓 面试连环追问

Q1: 如果有多个最小窗口，如何返回所有？

维护一个列表记录所有等于最小长度的窗口起始位置。在更新最小长度时，如果新窗口长度等于当前最小长度，也加入列表；如果更小，清空列表并记录新的。

Q2: t 中字符不要求连续怎么办？

这其实就是本题的设定——t 中字符在子串中不需要连续，只要都出现且数量足够即可。如果需要连续，就变成了子串匹配问题（可以用 KMP 等算法）。

Q3: 如果字符集是 Unicode 呢？

代码框架不变，哈希表可以处理任意字符。但注意 Java/C++ 中如果用固定大小数组（128或256）就不够了，必须使用真正的哈希表/字典结构。

Q4: 如何优化为 O(1) 空间？

对于 ASCII 字符集，可以用固定大小的数组代替哈希表，空间就是 O(128) = O(1)。如果字符集无限制，则无法在理论上做到 O(1) 空间。

最小覆盖子串 = 滑动窗口 + 双指针 + 哈希计数 + valid 优化。 右指针扩张找可行解，左指针收缩找最优解。valid 计数器避免每次遍历哈希表，是降维优化的关键。
