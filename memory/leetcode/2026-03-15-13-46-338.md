# LeetCode 338. 比特位计数

**日期**: 2026-03-15  
**标签**: 动态规划, 位运算  
**难度**: Easy

---

## 📋 题目描述

给你一个整数 `n`，对于每个在范围 `0 <= i <= n` 中的 `i`，计算其二进制表示中 `1` 的个数（即汉明重量），返回一个长度为 `n + 1` 的数组 `ans` 作为答案。

### 示例 1
```
输入: n = 2
输出: [0,1,1]
解释:
0 --> 0 (1个1)
1 --> 1 (1个1)  
2 --> 10 (1个1)
```

### 示例 2
```
输入: n = 5
输出: [0,1,1,2,1,2]
解释:
0 --> 0    (0个1)
1 --> 1    (1个1)
2 --> 10   (1个1)
3 --> 11   (2个1)
4 --> 100  (1个1)
5 --> 101  (2个1)
```

### 约束条件
- `0 <= n <= 10^5`
- 必须在 `O(n)` 时间复杂度和 `O(1)` 空间复杂度内解决此问题（输出数组不计入空间复杂度）

---

## 💡 解题思路

### 方法一：暴力法（不推荐）

对每个数字统计其二进制中1的个数，时间复杂度 `O(n log n)`。

```python
def countBits(n):
    def count_ones(x):
        count = 0
        while x:
            count += x & 1
            x >>= 1
        return count
    
    return [count_ones(i) for i in range(n + 1)]
```

### 方法二：Brian Kernighan 算法（次优解）

利用 `x & (x - 1)` 可以消除 `x` 最右边的1这一性质。

```python
def countBits(n):
    def count_ones(x):
        count = 0
        while x:
            x &= x - 1  # 消除最右边的1
            count += 1
        return count
    
    return [count_ones(i) for i in range(n + 1)]
```

**时间复杂度**: `O(n log n)` — 每个数字平均需要 `log n` 次操作  
**空间复杂度**: `O(1)` — 不包括输出数组

---

### 方法三：动态规划（最优解）✅

#### 核心思想

观察数字的二进制表示，可以发现规律：

```
0:  0000  → 0个1
1:  0001  → 1个1  = dp[0] + 1
2:  0010  → 1个1  = dp[1]
3:  0011  → 2个1  = dp[1] + 1
4:  0100  → 1个1  = dp[2]
5:  0101  → 2个1  = dp[2] + 1
6:  0110  → 2个1  = dp[3]
7:  0111  → 3个1  = dp[3] + 1
8:  1000  → 1个1  = dp[4]
```

#### 两种DP状态转移方程

**思路1：最高有效位（Most Significant Bit）**

```
dp[i] = dp[i - high_bit] + 1
```

其中 `high_bit` 是不超过 `i` 的最大2的幂次。

例如：
- `i = 5` (101)，`high_bit = 4` (100)
- `dp[5] = dp[5-4] + 1 = dp[1] + 1 = 1 + 1 = 2` ✓

**思路2：最低有效位（Least Significant Bit）—— 更简洁**

```
dp[i] = dp[i >> 1] + (i & 1)
```

解释：
- `i >> 1`：右移一位，相当于 `i // 2`
- `i & 1`：取最低位，判断奇偶

例如：
- `i = 5` (101)
- `i >> 1 = 2` (10)
- `i & 1 = 1`
- `dp[5] = dp[2] + 1 = 1 + 1 = 2` ✓

**思路3：消除最低位**

```
dp[i] = dp[i & (i - 1)] + 1
```

解释：
- `i & (i - 1)` 消除 `i` 最右边的1
- 所以 `i` 比 `i & (i - 1)` 多一个1

## 📝 完整代码

### Python 实现（推荐思路2）

```python
class Solution:
    def countBits(self, n: int) -> list[int]:
        # dp[i] 表示数字 i 的二进制中1的个数
        dp = [0] * (n + 1)
        
        for i in range(1, n + 1):
            # dp[i] = dp[i // 2] + (i % 2)
            # 等价于：dp[i] = dp[i >> 1] + (i & 1)
            dp[i] = dp[i >> 1] + (i & 1)
        
        return dp
```

### Python 实现（思路3 - 消除最低位）

### Python 实现（思路1 - 最高有效位）

```python
class Solution:
    def countBits(self, n: int) -> list[int]:
        dp = [0] * (n + 1)
        high_bit = 0  # 当前最高有效位
        
        for i in range(1, n + 1):
            # 如果 i 是2的幂次，更新 high_bit
            if i & (i - 1) == 0:
                high_bit = i
            
            dp[i] = dp[i - high_bit] + 1
        
        return dp
```

## 🔑 关键要点总结

### 1. DP的核心洞察

| 思路 | 状态转移 | 关键观察 |
|-----|---------|---------|
| 最高有效位 | `dp[i] = dp[i - high_bit] + 1` | 找到不超过i的最大2的幂 |
| 右移位 | `dp[i] = dp[i >> 1] + (i & 1)` | 右移一位后最低位单独处理 |
| 消最低位 | `dp[i] = dp[i & (i-1)] + 1` | `i & (i-1)` 消除最右的1 |

### 2. 位运算技巧

```python
# 判断是否为2的幂次
is_power_of_2 = (x & (x - 1)) == 0

# 获取最低位的1
lowest_bit = x & (-x)

# 消除最低位的1
clear_lowest = x & (x - 1)

# 获取最低位值（0或1）
lowest_bit_value = x & 1
```

### 3. 为什么 DP[i >> 1] 有效？

```
i = 13  →  1101 (3个1)
i >> 1 = 6  →  0110 (2个1)

观察：
- 1101 去掉最低位1 → 110 (6)
- 1101 去掉最低位0 → 110 (6)

所以 dp[i] = dp[i >> 1] + (i的最低位)
```

## 📝 面试考点

### 高频追问

1. **Q**: 为什么 `i & (i - 1)` 能消除最右边的1？  
   **A**: `i - 1` 会将最右边的1借位变成0，右边的0都变成1，所以与运算后最右边的1变成0，右边不变。

2. **Q**: 三种DP方法各有什么优劣？  
   **A**: 
   - 右移法最简洁，一行代码
   - 消除最低位法最直观，符合 Brian Kernighan 思想
   - 最高有效位法有助于理解数字的二进制结构

3. **Q**: 如果要求空间复杂度严格 O(1)（包括输出），怎么优化？  
   **A**: 无法优化，因为必须返回 n+1 个结果。题目说明输出数组不计入空间复杂度。

4. **Q**: 位运算在实际编程中的应用？  
   **A**: 权限系统（用位表示多个权限）、布隆过滤器、状态压缩DP、图形学颜色处理等。
