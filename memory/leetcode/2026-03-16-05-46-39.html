🔥 LeetCode #39 - 组合总和

Medium 数组 回溯 组合

📋 题目描述

给你一个 无重复元素 的整数数组 candidates 和一个目标整数 target，找出 candidates 中可以使数字和为目标数 target 的所有 不同组合，并以列表形式返回。你可以按 任意顺序 返回这些组合。

candidates 中的 同一个 数字可以 无限制重复被选取 。如果至少一个数字的被选数量不同，则两种组合是不同的。

示例 1
输入: candidates = [2,3,6,7], target = 7 输出: [[2,2,3],[7]] 解释: 2 和 3 可以形成一组候选，2 + 2 + 3 = 7 。注意 2 可以使用多次。 7 也是一个候选， 7 = 7 。 仅有这两种组合。
示例 2
输入: candidates = [2,3,5], target = 8 输出: [[2,2,2,2],[2,3,3],[3,5]]
示例 3
输入: candidates = [2], target = 1 输出: []
💡 核心约束
• 1 ≤ candidates.length ≤ 30
• 2 ≤ candidates[i] ≤ 40
• candidates 所有元素互不相同
• 1 ≤ target ≤ 40

💡 解题思路

思路一：回溯算法（推荐） ▼

这是一道经典的组合问题，可以用回溯算法（Backtracking）来解决。核心思想是：

  1. 从 candidates 的第一个元素开始尝试
  2. 每次选择可以选择当前元素（因为可以重复），也可以跳过当前元素选择后面的
  3. 记录当前路径的和，当和等于 target 时找到一个解
  4. 当和超过 target 时，剪枝返回
关键技巧：为了避免重复的组合，我们规定只能向后选择（即从当前索引开始），不能回头选前面的元素。

🌳 回溯决策树可视化

以 candidates = [2,3,5], target = 8 为例：

开始(剩余=8, 路径=[]) ├── 选2 (剩余=6, 路径=[2]) │ ├── 选2 (剩余=4, 路径=[2,2]) │ │ ├── 选2 (剩余=2, 路径=[2,2,2]) │ │ │ ├── 选2 (剩余=0) → ✓ 找到解 [2,2,2,2] │ │ │ └── 跳过 3,5 (剩余不足) │ │ ├── 选3 (剩余=1, 无法继续) │ │ └── 选5 (剩余=-1, 超过) │ ├── 选3 (剩余=3, 路径=[2,3]) │ │ └── 选3 (剩余=0) → ✓ 找到解 [2,3,3] │ └── 选5 (剩余=1, 无法继续) └── 选3 (剩余=5, 路径=[3]) └── 选5 (剩余=0) → ✓ 找到解 [3,5]
优化：剪枝技巧 ▼

通过以下方式可以剪掉无效分支，大幅提升效率：

  1. 排序剪枝：先将 candidates 排序，当当前数字 > 剩余 target 时，直接返回
  2. 提前终止：当 path 的和已经超过 target 时，不再继续递归
排序的优势：排序后一旦发现 candidates[i] > target，后面所有更大的数字都不用尝试了，直接剪枝。

📝 代码实现

Python 实现

class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:
        res = []
        path = []
        
        # 排序，用于剪枝优化
        candidates.sort()
        
        def backtrack(start: int, remain: int):
            # 找到有效解
            if remain == 0:
                res.append(path[:])  # 拷贝当前路径
                return
            
            # 从 start 开始尝试每个候选数字
            for i in range(start, len(candidates)):
                # 剪枝：如果当前数字已超过剩余目标，后面的都不用试了
                if candidates[i] > remain:
                    break
                
                # 做选择
                path.append(candidates[i])
                
                # 递归：注意这里传 i（可以重复选自己）而不是 i+1
                backtrack(i, remain - candidates[i])
                
                # 撤销选择（回溯）
                path.pop()
        
        backtrack(0, target)
        return res

Java 实现

class Solution {
    private List<List<Integer>> res = new ArrayList<>();
    private List<Integer> path = new ArrayList<>();
    
    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        Arrays.sort(candidates);  // 排序用于剪枝
        backtrack(candidates, 0, target);
        return res;
    }
    
    private void backtrack(int[] candidates, int start, int remain) {
        if (remain == 0) {
            res.add(new ArrayList<>(path));
            return;
        }
        
        for (int i = start; i < candidates.length; i++) {
            if (candidates[i] > remain) break;  // 剪枝
            
            path.add(candidates[i]);
            backtrack(candidates, i, remain - candidates[i]);  // 传 i，不是 i+1
            path.remove(path.size() - 1);  // 回溯
        }
    }
}
🔑 关键点
1. 递归时传 i 而不是 i+1，因为数字可以重复使用
2. 用 path[:] 或 new ArrayList<>(path) 拷贝当前路径，避免引用问题
3. 排序后的剪枝能显著提升效率

📊 复杂度分析

指标 复杂度 说明
时间复杂度 O(N^(T/M)) N 是 candidates 数量，T 是 target，M 是 candidates 最小值。最坏情况下需要遍历所有组合。
空间复杂度 O(T/M) 递归栈深度最多为 T/M（一直选最小值的情况），加上存储结果的 O(K)（K 是结果数量）。
💡 复杂度解读
最坏情况下，如果 candidates = [1]，target = 30，则时间复杂度接近 O(2^30)，但实际题目约束（target ≤ 40）保证了算法在合理时间内完成。排序剪枝可以大幅剪除无效分支。

🎯 核心要点总结

  • 回溯模板：选择 → 递归 → 撤销选择
  • 去重技巧：规定搜索顺序，从当前位置向后搜
  • 可重复选：递归传 i 而非 i+1
  • 剪枝优化：排序后 candidates[i] > remain 时 break
  • 路径拷贝：找到解时复制当前路径，避免引用污染
