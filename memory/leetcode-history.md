## 2026-04-27 - 第241期（cron任务）
- **题号**: 11
- **标题**: 盛最多水的容器
- **方法**: 双指针 / 贪心
- **难度**: Medium
- **核心**: 经典双指针贪心题。面积=底×高=(right-left)×min(h[left],h[right])。关键观察：移动较长一侧不可能获得更大面积（底边变小，高由短板决定），所以每次都移动较短侧的指针。正确性证明：固定短板，无论长板怎么移，底边只会变小，因此当前以短板为界的面积已是最大可能，可以安全丢弃短板。时间O(n)空间O(1)。面试常考正确性证明。
- **报告文件**: `memory/leetcode/2026-04-27-12-06-11.html`

## 2026-04-26 - 第238期（cron任务）
- **题号**: 42
- **标题**: 接雨水
- **方法**: 双指针 / 动态规划 / 单调栈
- **难度**: Hard
- **核心**: 经典Hard题，考多种解法。核心观察：每个位置能接的雨水 = min(左边最高柱, 右边最高柱) - 当前高度。双指针法从两侧向中间推进，矮的一侧先计算（因为矮侧决定水位），时间O(n)空间O(1)为最优解。DP法预处理左右最大值数组，思路直观。单调栈按层计算凹槽，适合理解结构。注意边界：长度小于3无法接水。

## 2026-04-26 - 第237期（cron任务）
- **题号**: 33
- **标题**: 搜索旋转排序数组
- **方法**: 二分查找（变形）
- **难度**: Medium
- **核心**: 旋转排序数组的经典二分变形。数组被旋转后分成两段各自升序的区间。通过比较 nums[mid] 和 nums[right] 判断哪一半是有序的，再判断 target 是否在该有序段内，从而决定收缩方向。时间复杂度 O(log n)，空间复杂度 O(1)。关键：元素互不相同，因此 mid 和 right 不会相等，总能判断有序段。

## 2026-04-10 - 第236期（cron任务）
- **题号**: 56
- **标题**: 合并区间
- **方法**: 排序 + 贪心算法
- **难度**: Medium
- **核心**: 区间问题经典解法。先按起点排序，然后遍历维护当前区间。重叠条件：下一个区间的起点 <= 当前区间的终点。重叠时合并（更新终点为两者最大值），不重叠时将当前区间加入结果。时间复杂度O(n log n)（主要来自排序），空间复杂度O(n)。
- **报告文件**: `memory/leetcode/2026-04-10-06-00-56.html`

## 2026-04-06 - 第235期（cron任务）
- **题号**: 72
- **标题**: 编辑距离
- **方法**: 动态规划
- **难度**: Hard
- **核心**: 经典DP题，定义dp[i][j]为word1[0..i-1]转word2[0..j-1]的最小操作数。状态转移：字符相同则dp[i][j]=dp[i-1][j-1]；字符不同则取删除、插入、替换三种操作的最小值+1。时间复杂度O(mn)，空间复杂度O(mn)可优化至O(min(m,n))。
- **报告文件**: `memory/leetcode/2026-04-06-00-00-72.html`
